# Summary

본 장에서는 자바의 대표적인 객체지향 특징 중 하나인 **상속(Inheritance)**의 기본 개념과 구현 방법을 학습합니다. 상속 관계에서 부모-자식 간의 멤버 접근 범위 제어, 서브 클래스 생성 시 생성자의 동작 메커니즘과 호출 순서, 그리고 부모 클래스의 동작을 자식 클래스에서 재정의하는 **메서드 오버라이딩(Method Overriding)**에 대해 자세히 다룹니다. 추가적으로 상속을 확장 또는 제어하는 예약어인 `super`와 `final`의 상세한 활용법을 알아봅니다.

---

# Why it matters

* **코드 재사용성 극대화**: 공통적인 속성과 동작을 상위 클래스에 한 번만 정의하고 이를 상속받음으로써 불필요한 코드의 중복(양산)을 방지하고 생산성을 향상시킵니다.
* **유지보수와 구조의 체계화**: 연관된 클래스들을 계층적 구조로 구성하여 프로그램의 복잡성을 낮추며, 상위 클래스의 변경이 하위 클래스에 자연스럽게 전파되도록 설계할 수 있습니다.

# Key Ideas

## 1. 상속(Inheritance)의 개념

### 클래스의 재사용 방식 변화
* **기존 방식 (객체지향 이전)**: 클래스를 직접 복사하여 일부 내용을 변경하고 코드를 추가하는 방식으로 재사용하였습니다. 이 방식은 비슷한 코드를 가진 클래스가 무수히 양산되어 관리가 어려워지는 원인이 되었습니다.
* **상속 방식**: 이미 존재하는 클래스의 기능을 그대로 물려받고 필요한 기능만 확장하는 방식으로 문제를 해결합니다.
* **수식**:
  $$\text{하위 클래스(Sub Class)} = \text{상위 클래스(Super Class)} + \text{하위 클래스에 추가된 기능과 속성}$$

### 상속의 효과
1. **클래스 검색 및 체계화 용이**: 클래스 계층 구조를 형성하여 원하는 클래스를 쉽게 분류하고 발견할 수 있습니다.
2. **클래스 확장 용이**: 기존의 검증된 클래스를 바탕으로 새로운 클래스를 용이하게 설계할 수 있습니다.
3. **클래스 수정 및 유지보수 용이**: 공통된 부분은 상위 클래스에 집중되므로 코드 수정 시 상위 클래스만 변경하면 모든 하위 클래스에 자동으로 적용됩니다.

### 자바 상속의 핵심 특징
1. **다중 상속(Multiple Inheritance) 비허용**: 자바는 클래스 간의 다중 상속을 지원하지 않습니다. 즉, `extends` 뒤에는 단 하나의 부모 클래스만 지정할 수 있는 **단일 상속(Single Inheritance)** 체계입니다.
2. **최상위 클래스 `java.lang.Object`**: 자바의 모든 클래스는 명시적으로 상속을 선언하지 않더라도 자동으로 `Object` 클래스로부터 상속을 받습니다. 즉, 자바의 모든 인스턴스는 `Object` 클래스의 기본 메서드(`equals()`, `toString()`, `getClass()` 등)를 사용할 수 있습니다.

## 2. 상속과 접근 지정자

상속 관계에서 상위 클래스에 정의된 멤버(변수, 메서드)가 하위 클래스에서 어떻게 노출되는지 제어하는 접근 제한자의 동작 방식은 매우 중요합니다.

* **`private` 멤버**: 슈퍼 클래스의 `private` 멤버는 하위 클래스를 포함한 모든 외부 클래스에서 접근할 수 없습니다. 오직 해당 슈퍼 클래스의 내부 메서드를 통해서만 간접적으로 제어할 수 있습니다.
* **`default` 멤버**: 동일한 패키지 내에 속해 있을 때만 하위 클래스에서 접근을 허용합니다. 패키지가 다르다면 상속을 받았더라도 접근할 수 없습니다.
* **`protected` 멤버**: 동일한 패키지뿐만 아니라, **다른 패키지에 존재하더라도 자식 클래스(서브 클래스)라면 접근을 허용**합니다. 상속 관계를 위한 가장 핵심적인 접근 제한자입니다.
* **`public` 멤버**: 패키지에 무관하게 모든 곳에서 직접 접근을 허용합니다.

### 접근 제한자에 따른 서브 클래스의 접근 여부 요약

| 접근 제한자 | 동일 클래스 | 동일 패키지 (상속 여부 무관) | 다른 패키지 - 상속된 클래스 | 다른 패키지 - 상속되지 않은 클래스 |
| :--- | :---: | :---: | :---: | :---: |
| **`public`** | O | O | O | O |
| **`protected`** | O | O | **O** | X |
| **`default`** | O | O | X | X |
| **`private`** | O | X | X | X |

## 3. 상속과 생성자 (Constructor)

### 생성자 호출 및 실행 순서
서브 클래스의 인스턴스를 생성할 때, JVM은 메모리에 자식 객체뿐만 아니라 부모 객체의 정보도 함께 적재해야 합니다. 따라서 다음과 같은 순서로 생성자가 작동합니다.
1. **서브 클래스의 생성자 호출**
2. 서브 클래스 생성자의 첫 줄에서 자동으로 **슈퍼 클래스의 생성자가 호출**됨
3. **슈퍼 클래스의 생성자 실행 완료**
4. **서브 클래스의 생성자 실행 완료**
> **[핵심 원칙]**: 메모리상에 상위 클래스의 인스턴스 영역이 먼저 빌드되고 초기화된 후에, 하위 클래스의 초기화 작업이 실행됩니다.

### `super()` 키워드를 통한 부모 생성자 호출
* 서브 클래스의 생성자에서 슈퍼 클래스의 생성자를 명시적으로 호출할 때 `super(...)`를 사용합니다.
* **기본 생략**: 서브 클래스 생성자 첫 줄에 `super()`가 명시되어 있지 않으면, 컴파일러가 자동으로 매개변수가 없는 `super();` 코드를 삽입하여 부모의 기본 생성자를 호출합니다.
```java
class Car {
    Car() { System.out.println("슈퍼 클래스 생성자 호출"); }
}
class Sedan extends Car {
    Sedan() {
        // 컴파일러에 의해 super(); 자동 삽입됨
        System.out.println("서브 클래스 생성자 호출");
    }
}
```

#### 부모 클래스에 기본 생성자가 없는 경우의 문제와 해결
* 만약 슈퍼 클래스에 매개변수가 있는 생성자만 정의되어 있고 기본 생성자가 없다면, 컴파일러가 서브 클래스에 자동으로 삽입하는 `super()` 호출은 에러를 발생시킵니다.
* **해결 방법**: 서브 클래스 생성자의 첫 줄에 개발자가 직접 매개변수 값에 대응하는 **`super(매개변수...)`를 명시적으로 호출**해야 컴파일 에러를 방지할 수 있습니다.
```java
class Animal {
    String name;
    int age;
    
    // 기본 생성자가 없고, 매개변수 생성자만 정의됨
    Animal(String name, int age) {
        this.name = name;
        this.age = age;
    }
}

class Dog extends Animal {
    Dog(String name, int age) {
        super(name, age); // 명시적으로 부모 생성자 호출 (필수)
    }
}
```

## 4. 메서드 오버라이딩 (Method Overriding)

### 정의
* 상위 클래스에 이미 정의된 메서드를 하위 클래스의 특성에 맞게 동일한 시그니처로 다시 정의하여 덮어쓰는(치환하는) 기법입니다.
* **목적**: 상위 클래스 타입의 참조 변수로 하위 클래스 객체를 제어할 때 하위 클래스에서 재정의된 구체적인 동작을 유발(실행 시간 다형성 구현)하기 위함입니다.

### 오버라이딩의 성립 조건
1. **메서드 이름이 동일**해야 합니다.
2. **매개변수의 타입, 개수, 순서(메서드 시그니처)가 완전히 일치**해야 합니다.
3. **반환 타입이 동일**해야 합니다. (공변 반환 타입 지원으로 하위 클래스 타입으로 반환하는 것은 허용됨)
4. 접근 제한자는 부모 클래스의 메서드보다 **더 좁은 범위로 변경할 수 없습니다.** (예: 부모 메서드가 `protected`이면 자식은 `protected` 또는 `public`이어야 함)

### 오버라이딩(Overriding) vs 오버로딩(Overloading)

| 비교 항목 | 오버라이딩 (Overriding) | 오버로딩 (Overloading) |
| :--- | :--- | :--- |
| **정의** | 부모의 메서드를 자식 클래스에서 재정의 | 한 클래스 내에서 같은 이름의 메서드를 다중 정의 |
| **관계** | 상속 관계(클래스 간)에서 발생 | 동일 클래스(또는 상속받은 범위 내)에서 발생 |
| **시그니처** | 이름, 매개변수 개수/타입 모두 일치 | 이름은 같으나 매개변수의 개수/타입은 다름 |
| **반환 타입** | 완전히 동일해야 함 | 반환 타입은 무관함 |
| **바인딩 시점** | **동적 바인딩 (실행 시간, Runtime)** | **정적 바인딩 (컴파일 시간, Compile time)** |
| **개념적 성격** | **치환 (Replacement)**: 부모 메서드를 덮어씀 | **중첩 (Accumulation)**: 여러 메서드가 공존함 |

## 5. 예약어 super와 final

### 1) super 예약어의 두 가지 형태
자식 클래스 내부에서 `super` 키워드는 다음과 같이 사용됩니다.
* **`super.멤버변수` 또는 `super.메서드()`**:
  * 자식 클래스에서 변수나 메서드를 오버라이딩하여 부모의 원본 멤버가 가려졌을 때, 부모 클래스의 멤버에 명시적으로 접근하기 위해 사용합니다.
* **`super(...)`**:
  * 자식 클래스의 생성자 내부에서 부모 클래스의 생성자를 명시적으로 호출할 때 사용합니다. 반드시 생성자의 첫 줄에 와야 합니다.

### 2) final 예약어의 세 가지 용도
`final` 키워드는 "변경 불가능함"을 나타내며, 지정하는 위치에 따라 다른 제약 사항을 가집니다.

* **변수 선언 시 (`final int MAX_SPEED = 150;`)**:
  * 변수를 상수로 만듭니다. 초기화 이후에는 값을 절대 변경(재대입)할 수 없습니다.
* **메서드 선언 시 (`final void upSpeed(...)`)**:
  * 자식 클래스에서 해당 메서드를 **오버라이딩(재정의)할 수 없도록 제한**합니다. 메서드의 핵심 로직을 자식 클래스에서 왜곡하지 못하게 보호할 때 사용합니다.
* **클래스 선언 시 (`final class Car`)**:
  * 해당 클래스를 **상속(extends)할 수 없도록 제한**합니다. 즉, 자식 클래스를 만들 수 없는 리프(Leaf) 클래스가 됩니다. (예: `java.lang.String`은 final 클래스로 설계되어 상속 및 왜곡이 불가능함)
