# 1. 자바 프로그램의 기본 구조

자바 프로그램은 엄격한 객체 지향 및 클래스 기반 구조를 따릅니다.

* **클래스 기반 작성**: 자바의 모든 소스 코드는 클래스(Class) 내부에 정의되어야 합니다. 클래스 외부에는 `import` 문과 `package` 선언만 올 수 있습니다.
* **파일명 일치 규칙**: `public class`로 선언된 클래스의 이름은 소스 파일의 이름과 대소문자까지 정확히 일치해야 합니다. (예: `public class Hello` $\rightarrow$ 파일명은 `Hello.java`여야 함).
* **메서드(Method) 정의**: C/C++의 함수(Function)에 대응하는 개념으로, 자바에서는 전역 함수가 존재하지 않으며 반드시 클래스의 멤버로서 메서드가 정의됩니다.
* **진입점(Entry Point)**: 프로그램을 실행하기 위해서는 `public static void main(String[] args)` 메서드가 반드시 필요합니다.
* **문장의 끝**: 모든 실행문은 세미콜론 `;`으로 끝나야 합니다.
* **주석(Comment)**:
  * 단일 행 주석: `//`로 시작하여 해당 행 끝까지 주석 처리합니다.
  * 다중 행 주석: `/*`와 `*/` 사이에 위치한 모든 내용을 주석 처리합니다.

# 2. 식별자 (Identifier)와 키워드 (Keyword)

### 1) 식별자 작명 규칙 (Identifier Rules)
식별자는 클래스, 변수, 상수, 메서드 등에 개발자가 부여하는 이름입니다. 자바 컴파일러는 다음의 규칙을 엄격하게 검증합니다.

* **사용 가능한 문자**: 알파벳 대소문자, 숫자, 언더바(`_`), 달러 기호(`$`)
  * 특수문자 중 `@`, `#`, `!` 등과 공백(Space) 및 탭(Tab)은 식별자로 사용할 수 없습니다.
* **숫자 시작 제한**: 식별자의 첫 번째 문자로 숫자는 사용할 수 없습니다. (예: `3Chapter` (X), `Chapter3` (O))
* **유니코드 지원**: 한글 식별자도 문법적으로 사용 가능하지만, 현업 개발 관례 및 호환성 유지를 위해 사용을 권장하지 않습니다.
* **예약어 및 리터럴 사용 불가**: 자바 예약어(키워드) 및 불린 리터럴(`true`, `false`), 널 리터럴(`null`)은 식별자로 사용할 수 없습니다.
* **대소문자 구분 (Case-sensitive)**: `Test`와 `test`는 완전히 서로 다른 식별자로 취급됩니다.
* **길이 제한 없음**: 식별자의 길이는 물리적으로 제한이 없습니다.

### 2) 자바 예약어 (Keywords)
자바 언어 명세에 고유한 의미가 정의된 단어들로, 개발자가 다른 용도로 재정의할 수 없습니다.

| 분류 | 키워드 목록 |
| :--- | :--- |
| **기본 타입** | `boolean`, `byte`, `char`, `short`, `int`, `long`, `float`, `double` |
| **제어문** | `if`, `else`, `switch`, `case`, `default`, `for`, `do`, `while`, `break`, `continue`, `return` |
| **객체 지향** | `class`, `interface`, `enum`, `extends`, `implements`, `new`, `instanceof`, `this`, `super` |
| **접근/범위** | `public`, `protected`, `private`, `package`, `import`, `static`, `final`, `abstract` |
| **예외 처리** | `try`, `catch`, `finally`, `throw`, `throws`, `assert` |
| **기타** | `void`, `synchronized`, `volatile`, `transient`, `native`, `strictfp`, `const`\*, `goto`\* |

> [!NOTE]
> `const`와 `goto`는 C/C++과의 호환성 등의 이유로 자바 키워드로 예약되어 있으나, 실제 자바 문법에서는 사용되지 않는 키워드(Unused Keywords)입니다. 상수를 정의할 때는 `const` 대신 `final` 키워드를 사용해야 합니다.

### 3) 작명 관례 (Naming Conventions)
자바 생태계에서 권장하는 명명 규약입니다.

* **클래스 이름**: 대문자로 시작하는 파스칼 케이스(PascalCase)를 따릅니다. (예: `HelloWorld`, `AutoVendingMachine`)
* **변수 및 메서드 이름**: 소문자로 시작하는 카멜 케이스(camelCase)를 따릅니다. (예: `myAge`, `isSingle`, `getAge()`)
* **상수 이름**: 전체를 대문자로 작성하며, 단어 사이는 언더바로 연결하는 스네이크 케이스(UPPER_SNAKE_CASE)를 따릅니다. (예: `final static double PI = 3.141592;`)

# 3. 데이터 타입 (Data Types)

자바의 모든 데이터 타입은 크게 **기본 타입**과 **레퍼런스 타입** 두 가지로 분류됩니다.

### 1) 기본 타입 (Primitive Types)
* **특징**: 데이터의 실제 값(Value)이 메모리의 스택(Stack) 영역에 직접 저장됩니다. 
* C/C++과 다르게 **컴파일러와 운영체제, CPU 아키텍처에 무관하게 데이터 타입의 크기가 고정**되어 있어 뛰어난 이식성을 보장합니다.

| 분류 | 타입 | 크기 | 표현 범위 | 주요 특징 |
| :--- | :--- | :--- | :--- | :--- |
| **논리형** | `boolean` | 1비트 (혹은 JVM 의존) | `true`, `false` | 조건문 및 논리 연산의 결과값으로 활용됨. |
| **문자형** | `char` | 2바이트 (16비트) | Unicode (`\u0000` ~ `\uffff`) | 2바이트 유니코드를 사용하므로 전 세계 모든 문자를 단일 원소로 표현 가능. 단일 인용부호 `' '`를 사용. |
| **정수형** | `byte` | 1바이트 (8비트) | $-128 \sim 127$ | 파일 입출력, 네트워크 데이터 전송 등 바이트 단위 작업에 유용. |
| | `short` | 2바이트 (16비트) | $-32,768 \sim 32,767$ | C/C++과의 호환성을 위해 유지됨. |
| | `int` | 4바이트 (32비트) | $-2^{31} \sim 2^{31}-1$ | 자바 정수 연산의 기본 단위. JDK 8부터 `Integer` 클래스의 Helper 메서드를 통해 부호 없는 32비트 정수($0 \sim 2^{32}-1$)로 처리 가능. |
| | `long` | 8바이트 (64비트) | $-2^{63} \sim 2^{63}-1$ | 아주 큰 정수를 다룰 때 사용. 리터럴 뒤에 접미사 `L` 또는 `l`을 붙여야 함. JDK 8부터 부호 없는 64비트 정수($0 \sim 2^{64}-1$)로 처리 가능. |
| **실수형** | `float` | 4바이트 (32비트) | $\pm 1.4\text{E}-45 \sim \pm 3.4\text{E}38$ | IEEE 754 표준을 따름. 접미사 `F` 또는 `f`가 필수임. |
| | `double` | 8바이트 (64비트) | $\pm 4.9\text{E}-324 \sim \pm 1.7\text{E}308$ | 자바 실수 연산의 기본 단위. 더 정밀도가 높음. |

### 2) 레퍼런스 타입 (Reference Types)
* **특징**: 실제 객체나 배열은 메모리의 힙(Heap) 영역에 생성되며, 스택에 저장되는 레퍼런스 변수는 실제 데이터가 위치한 **힙 메모리의 주소(Reference)**만을 담고 있습니다.
* 용도는 배열 레퍼런스, 클래스 레퍼런스, 인터페이스 레퍼런스 3가지로 나뉩니다.
* **null 리터럴**: 레퍼런스 타입 변수가 어떠한 객체도 참조하고 있지 않음을 나타내는 값입니다. 기본 타입 변수에는 `null`을 대입할 수 없으며 컴파일 에러가 발생합니다. (예: `int n = null;` (X), `String str = null;` (O))

### 3) 문자열 (String)과 리터럴
* 문자열은 자바의 기본 타입이 아니라 JDK에서 제공하는 **`String` 클래스**를 통해 객체로 표현됩니다.
* 이중 인용부호 `" "`를 이용하여 문자열 리터럴을 나타내며, 문자열과 다른 타입의 데이터를 `+` 연산자로 연결하면 상대 타입이 자동으로 문자열로 형변환되어 결합됩니다.
  * 예: `"JDK" + 1.8` $\rightarrow$ `"JDK1.8"`
  * 예: `"(" + 3 + "," + 5 + ")"` $\rightarrow$ `"(3,5)"`
* **특수 문자 리터럴(Escape Sequence)**: 백슬래시(`\`)로 시작하는 특수 문자입니다.
  * `\b` (백스페이스), `\t` (탭), `\n` (라인피드), `\f` (폼피드), `\r` (캐리지 리턴), `\"` (이중 인용부호), `\'` (단일 인용부호), `\\` (백슬래시)

# 4. 타입 변환 (Type Casting)

자바에서는 연산 시 피연산자들의 타입을 일치시키는 과정에서 타입 변환이 발생합니다.

### 1) 자동 타입 변환 (Implicit/Promotion Type Conversion)
* **원리**: 표현 범위가 좁은 데이터 타입이 범위가 더 넓은(큰) 타입에 대입되거나 연산될 때, 컴파일러가 암묵적으로 타입을 변환해 줍니다. 데이터 손실이 일어나지 않으므로 안전합니다.
* **규칙**: `byte` $\rightarrow$ `short`/`char` $\rightarrow$ `int` $\rightarrow$ `long` $\rightarrow$ `float` $\rightarrow$ `double`
  * 예: `long m = 25;` $\rightarrow$ 32비트 정수 리터럴 `25`가 64비트 `long` 타입 변수에 들어가면서 자동으로 64비트로 확장됩니다.
  * 예: `double d = 3.14 * 10;` $\rightarrow$ 연산식 내 `3.14`가 `double`이므로 피연산자인 정수 `10`이 실수 `10.0`으로 자동 승격되어 실수 연산이 진행됩니다.

### 2) 강제 타입 변환 (Explicit/Demotion Type Casting)
* **원리**: 표현 범위가 넓은 데이터 타입을 좁은 타입으로 변환하고자 할 때 사용하며, 괄호 `(target_type)` 캐스트 연산자를 사용하여 개발자가 명시적으로 변환을 지시해야 합니다.
* **위험성**: 메모리 크기의 축소로 인해 상위 비트들이 절단되어 **값 손실(Data Loss)**이 발생할 수 있습니다.

#### [정수형 캐스팅 시 비트 절단 메커니즘]
`int` 타입 변수 `n`의 값 `300`을 `byte` 타입으로 강제 변환하는 과정:
1. `300`은 16진수로 `0x0000012C` (32비트)입니다.
2. 이를 1바이트(8비트)인 `byte`로 강제 변환(`(byte)n`)하면 하위 1바이트인 `0x2C`만 남고 상위 3바이트는 모두 버려집니다.
3. 2진수 `00101100` (`0x2C`)은 10진수로 `44`에 해당합니다. 즉, $300 \pmod{256} = 44$의 결과와 동일하며 심각한 값의 변화가 유발됩니다.

#### [실수형에서 정수형 캐스팅 시 소수점 버림 메커니즘]
`double` 또는 `float` 실수를 `int`와 같은 정수형으로 강제 변환하면 실수형의 가수(Fraction) 부분 중 **소수점 이하 부분 전체가 강제로 폐기**됩니다.
```java
double d = 1.9;
int num = (int)d; // num의 값은 1이 됨 (소수점 이하 0.9 데이터 손실)
```

# 5. 자바의 키 입력: System.in과 Scanner

### 1) System.in의 원리와 한계
* **System.in**은 키보드 장치로부터 데이터를 직접 입력받는 자바의 표준 바이트 입력 스트림(InputStream)입니다.
* **한계**: 입력된 키 값을 가공하지 않은 **바이트 데이터(raw byte data)**로 리턴하기 때문에, 문자나 숫자 등으로 변환하기 위해 추가적인 비트 연산 및 타입 캐스팅 코드를 구현해야 하는 한계가 존재합니다.

### 2) Scanner 클래스를 통한 토큰화 파싱
`java.util.Scanner` 클래스는 `System.in`으로부터 바이트 데이터를 넘겨받아 다양한 타입으로 손쉽게 분할 파싱하여 변환해 주는 고수준 텍스트 파서입니다.

* **공백 구분자(Delimiter)**: Scanner는 입력된 텍스트를 공백 문자를 기준으로 분할하여 읽어들입니다.
  * 구분자 대상: 탭(`\t`), 폼피드(`\f`), 캐리지 리턴(`\r`), 스페이스(` `), 뉴라인(`\n`)
* **토큰(Token) 파싱 방식**:
  * 공백으로 구분된 단위(토큰)를 읽어옵니다. (예: `"Kim Seoul 20 65.1 true"` 입력 시 공백으로 구분된 5개의 토큰 생성)
  * `next()`: 다음 토큰을 `String` 타입으로 읽어옵니다.
  * `nextInt()`, `nextDouble()`, `nextBoolean()` 등: 다음 토큰을 읽어 각각의 데이터 타입으로 파싱합니다. 만약 다음 토큰이 호출한 데이터 타입 형식과 다를 경우 `InputMismatchException`이 발생합니다.
  * `nextLine()`: 공백 문자 구분 기호를 무시하고, 줄 바꿈 문자(`\n`)를 만날 때까지의 모든 전체 라인을 한 번에 문자열로 가져옵니다.

# 6. 식과 연산자 (Expressions & Operators)

### 1) 연산자 우선순위와 결합 법칙 (Precedence & Associativity)
여러 연산자가 하나의 식에 혼재할 때 아래의 규칙에 따라 계산이 진행됩니다.
* **우선순위(Precedence)**: 괄호 `()`가 최우선순위를 가지며, 단항 $\rightarrow$ 산술 $\rightarrow$ 시프트 $\rightarrow$ 비교 $\rightarrow$ 논리 $\rightarrow$ 삼항 $\rightarrow$ 대입 순서로 진행됩니다.
* **결합성(Associativity)**: 동일한 우선순위를 가지는 연산자의 경우 연산 방향이 결정됩니다.
  * **왼쪽에서 오른쪽 (L-to-R)**: 대부분의 이항 연산자 (`+`, `-`, `*`, `/` 등)
  * **오른쪽에서 왼쪽 (R-to-L)**: 대입 연산자 (`=`, `+=`), 단항 증감 연산자 (`++`, `--`), 캐스트 연산자, 부호/부정 연산자 (`!`, `~`)

### 2) 증감 연산자 (Increment / Decrement Operators)
`++`, `--` 연산자는 피연산자의 위치에 따라 연산 실행 시점이 다릅니다.

* **전위 연산자 (Prefix: `++a`, `--a`)**:
  * 식 내에서 먼저 변수의 값을 1 증가/감소시킨 후, 바뀐 값을 전체 연산식에 대입하여 최종 연산을 수행합니다.
* **후위 연산자 (Postfix: `a++`, `a--`)**:
  * 변수가 가진 현재 값을 먼저 연산식에 대입하여 평가를 마친 뒤, 다음 문장으로 넘어가기 전 변수의 값을 1 증가/감소시킵니다.
  ```java
  int d = 3;
  int a = d++; // a = 3, d = 4 (후위)
  int b = ++d; // b = 5, d = 5 (전위)
  ```

### 3) 산술 연산자 (`/`, `%` 응용)
* 나누기(`/`): 피연산자가 모두 정수형인 경우 몫(Integer Division)만 구하며 소수점 이하는 버려집니다. (`5 / 2 = 2`)
* 나머지(`%`): 정수 나눗셈의 나머지를 구하며, 주기적 패턴 분석이나 자릿수 분리에 널리 활용됩니다.
  * **자릿수 분리**: `69 / 10 = 6` (십의 자리), `69 % 10 = 9` (일의 자리)
  * **홀짝 판별**: `n % 2 == 1`이면 홀수, `0`이면 짝수입니다.

### 4) 논리 연산자와 단락 평가 (Short-Circuit Evaluation)
논리 연산은 불린 값의 연산을 수행합니다.
* `&&` (AND), `||` (OR), `!` (NOT), `^` (XOR - 두 피연산자의 불린 값이 다를 때만 `true`)
* **단락 평가(Short-Circuit Evaluation)**:
  * `&&` 연산에서 앞의 피연산자가 `false`이면, 뒤의 피연산자가 무엇이든 전체 결과는 `false`가 확정되므로 뒤의 수식을 평가하지 않고 바로 생략합니다.
  * `||` 연산에서 앞의 피연산자가 `true`이면, 뒤의 수식을 평가하지 않고 바로 `true`를 리턴합니다.
  * 이 메커니즘을 사용하면 `null` 체크나 인덱스 초과 방지 코드를 안전하게 한 라인에 작성할 수 있습니다. (예: `if (str != null && str.length() > 0)`)

### 5) 조건 삼항 연산자 (`? :`)
* `condition ? expr1 : expr2` 구조의 삼항(Ternary) 연산자입니다.
* `condition`이 `true`이면 `expr1`의 결과값을 반환하고, `false`이면 `expr2`를 반환합니다. `if-else` 구조를 간결하게 한 줄로 리팩토링할 때 매우 유용합니다.

# 7. 조건 제어 흐름: 조건문 (Conditionals)

### 1) if, if-else, 다중 if-else, 중첩 if
* 조건식의 결과(반드시 `boolean` 타입)에 따라 코드 블록의 실행 여부를 결정합니다.
* 실행문이 단 한 문장일 경우에는 중괄호 `{}`를 생략할 수 있으나, 가독성과 향후 코드 유지보수 중 발생할 버그 방지를 위해 항상 중괄호를 명시하는 것이 모범 사례(Best Practice)입니다.

### 2) switch 문과 타입 제약
* **동작 원리**: 표현식(식)을 평가하여 그 결과값과 일치하는 `case` 레이블로 직접 점프하여 문장을 수행합니다.
* **break의 역할**: 실행이 끝난 후 `switch` 문 바깥으로 제어권을 반환합니다. 만약 `break` 문이 누락되면 다음 `case` 레이블의 실행문들이 연달아 수행되는 **Fall-through** 현상이 발생합니다.
* **case 레이블 값의 제약**:
  * `case` 뒤에 올 수 있는 값은 **컴파일 시점에 상수로 확정되는 리터럴(Literal) 또는 상수**만 가능합니다.
  * **허용되는 타입**: `char`, `byte`, `short`, `int`의 정수 계열 기본 타입, `String` 문자열(자바 7부터 지원), 그리고 열거형(`enum`) 상수.
  * **허용되지 않는 타입**: `float`, `double`과 같은 실수형 리터럴은 부동 소수점 오차 정밀도 문제로 인해 `case` 레이블로 사용할 수 없습니다. 또한 변수나 조건 수식(`case a > 3:` 등)도 기술할 수 없으며 컴파일 에러를 유발합니다.
