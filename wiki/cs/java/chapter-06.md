# Summary

본 장에서는 객체지향 프로그래밍(OOP, Object-Oriented Programming)의 기본 개념과 자바 프로그래밍의 핵심 단위인 클래스(Class)와 객체(Object)에 대해 다룹니다. 또한 클래스를 구성하는 3대 요소인 멤버 변수, 메서드, 생성자의 개념을 파악하고, 정보 은닉(Information Hiding)을 구현하기 위한 접근 제한자(Access Modifier)의 특징과 범위를 학습합니다.

---

# Why it matters

* **구조적 프로그래밍의 한계 극복**: 절차지향적 프로그래밍 방식과 달리 데이터와 그 데이터를 처리하는 동작을 하나의 단위인 '객체'로 묶음으로써 소프트웨어의 생산성, 유연성, 재사용성을 극대화합니다.
* **유지보수성 향상**: 클래스를 활용해 모듈화된 설계를 진행하고 접근 제한자를 통해 외부로부터 데이터를 안전하게 보호(캡슐화)함으로써 대형 소프트웨어 시스템 개발의 복잡성을 낮추고 관리의 용이성을 향상시킵니다.

# Key Ideas

## 1. 객체지향 프로그래밍(OOP) 개요

### 객체지향 프로그래밍의 개념
* **정의**: 데이터(속성)와 이를 다루는 메서드(동작)를 결합한 '객체(Object)'를 기본 단위로 하여, 객체 간의 상호작용을 통해 프로그램을 구성하는 개발 패러다임입니다.
* **장점**:
  * **개발 속도 향상**: 기존에 작성된 클래스의 코드를 재사용(상속 등)하여 개발 생산성을 높입니다.
  * **소프트웨어 유지·관리 향상**: 관련 있는 데이터와 메서드가 단일 위치(객체)에 통합되므로 오류가 발생했을 때 수정하기 용이합니다.
  * **소프트웨어 개발 생산성 향상 및 비용 절감**: 유지보수가 쉬워지므로 전체적인 개발 및 유지관리 비용이 줄어듭니다.

### 객체지향 프로그래밍의 6대 구성 요소

| 구성 요소 | 영어 명칭 | 설명 |
| :--- | :--- | :--- |
| **클래스** | Class | 같은 종류의 집단에 속한 속성과 행동을 정의한 설계도(틀)입니다. |
| **객체** | Object | 클래스를 기반으로 메모리에 생성된 실체(인스턴스)입니다. |
| **캡슐화** | Encapsulation | 데이터(속성)와 행동(메서드)을 하나의 단위로 묶고 불필요한 내부 세부 사항을 감추는 기법입니다. |
| **상속** | Inheritance | 이미 존재하는 상위 클래스의 멤버(변수, 메서드)를 하위 클래스가 물려받아 재사용하는 기법입니다. |
| **다형성** | Polymorphism | 하나의 변수, 메서드, 혹은 객체가 상황에 따라 여러 가지 다양한 형태로 동작할 수 있게 하는 기법입니다. (예: 오버로딩, 오버라이딩) |
| **추상화** | Abstraction | 구체적인 세부 사항은 숨기고, 객체들의 공통적인 본질적 속성과 행동만을 추출하여 정의하는 기법입니다. |

## 2. 클래스(Class)와 객체(Object)

### 개념 비교
* **클래스(Class)**: 객체를 만들어내기 위한 설계도 혹은 틀입니다. 메모리에 직접 로드되지 않으며 속성(변수)과 행동(메서드)을 정의합니다.
* **객체(Object)**: 클래스의 모양 그대로 메모리 공간에 할당되어 실제로 존재하는 실체입니다. 프로그램 실행 중에 생성되며 **인스턴스(Instance)**라고도 부릅니다.
  > **[붕어빵 틀과 붕어빵 비유]**
  > * **붕어빵 틀**: 클래스 (먹을 수 없음, 속성 정의, 붕어빵을 만들기 위한 도구)
  > * **붕어빵**: 객체/인스턴스 (먹을 수 있음, 팥/슈크림 등 실제 데이터 보유, 실행되는 실체)

### 클래스의 형태
```java
[접근제한자] class 클래스명 {
    // 멤버 변수 (속성)
    자료형 변수명;
    
    // 생성자 (초기화)
    클래스명() { ... }
    
    // 멤버 메서드 (행동)
    반환타입 메서드명() { ... }
}
```
* 모든 클래스에 반드시 변수와 메서드가 동시에 존재해야 하는 것은 아닙니다. 변수만 있거나 메서드만 포함된 클래스도 존재하며, 수행할 작업에 따라 멤버를 다르게 구성합니다.

### 객체(인스턴스) 생성
* 자바에서 객체를 생성할 때는 `new` 키워드를 사용합니다.
```java
클래스명 객체명 = new 클래스명();
```
* `new` 연산자는 메모리(Heap)에 객체를 저장할 공간을 생성하고, 생성자를 호출하여 초기화한 후 그 객체의 참조값(주소)을 반환합니다.

## 3. 클래스의 구성 요소와 생성자

### 1) 멤버 변수(Member Variable) 또는 필드(Field)
* 객체의 상태 정보를 저장하는 공간입니다.
* 객체 생성 후 마침표(`.`) 구문을 사용하여 접근할 수 있습니다.
```java
Student student = new Student();
student.id = 20260619; // 멤버 변수 id에 접근 및 값 대입
```

### 2) 멤버 메서드(Member Method)
* 객체의 동작을 구현하고 내부 데이터를 조작하는 곳입니다.
* **정적 메서드(Static Method)**: `static` 키워드가 선언된 메서드로, 객체를 생성하지 않고 클래스명을 통해 바로 호출할 수 있습니다.
* **인스턴스 메서드(Instance Method)**: `static` 키워드가 없는 메서드로, 반드시 객체를 생성한 후에만 호출하여 사용할 수 있습니다.

### 3) 생성자(Constructor)
* 객체가 생성될 때(`new` 연산자 사용 시) 자동으로 호출되어 객체를 초기화하는 특수한 유형의 메서드입니다.

#### 생성자 작성 규칙
* **생성자명은 클래스명과 동일**해야 합니다.
* 반환 자료형(Return Type)이 없으며, `void`조차 붙이지 않습니다.
* 생성자는 `abstract`, `final`, `static`으로 선언할 수 없습니다.
* 접근 제한자(public, protected, default, private)를 붙여 생성자에 접근할 수 있는 범위를 제어할 수 있습니다.

#### 생성자의 종류

* **기본 생성자 (Default Constructor)**:
  * 매개변수가 없는 생성자입니다.
  * 클래스 내부에 생성자를 하나도 작성하지 않으면, 자바 컴파일러가 아무런 동작을 하지 않는 기본 생성자를 자동으로 추가해 줍니다.
  ```java
  클래스명() { }
  ```
  * **주의**: 매개변수가 있는 생성자가 단 하나라도 작성되어 있다면, 컴파일러는 기본 생성자를 자동으로 만들어주지 않습니다.

* **일반 생성자 (Parameter Constructor)**:
  * 매개변수를 통해 객체 생성 시점에 값을 전달받아 멤버 변수를 초기화하는 역할을 수행합니다.

* **생성자 오버로딩 (Constructor Overloading)**:
  * 이름은 같지만 매개변수의 개수, 자료형, 순서가 다른 여러 개의 생성자를 정의하는 기법입니다. 이를 통해 다양한 방법으로 객체를 생성할 수 있습니다.

#### `this` 키워드와 `this()`

* **`this` 레퍼런스**:
  * 객체 자신을 가리키는 참조 변수입니다.
  * 주로 **멤버 변수와 메서드의 매개변수(로컬 변수) 이름이 동일**할 때, 이를 구분하기 위해 멤버 변수 앞에 붙여 사용합니다.
  ```java
  void inputData(int id, String name) {
      this.id = id;       // 좌측은 멤버 변수 id, 우측은 매개변수 id
      this.name = name;   // 좌측은 멤버 변수 name, 우측은 매개변수 name
  }
  ```
* **`this()` 메서드**:
  * 클래스 내부의 다른 생성자를 호출할 때 사용하는 특수 메서드입니다.
  * 중복되는 초기화 코드를 제거하고 생성자 결합도를 높이기 위해 사용됩니다.
  * **제약 조건**: 반드시 **생성자 내부의 첫 번째 문장(첫 줄)**으로만 작성해야 합니다.
  ```java
  class Student {
      int id;
      String name;
      
      Student() {
          this("홍길동"); // Student(String name) 생성자 호출
      }
      Student(String name) {
          this(20221005, name); // Student(int id, String name) 생성자 호출
      }
      Student(int id, String name) {
          this.id = id;
          this.name = name;
      }
  }
  ```

## 4. 접근 제한자(Access Modifier)와 정보 은닉

### 정보 은닉과 캡슐화 규칙
* 객체의 내부 구조나 핵심 데이터(멤버 변수)를 외부로부터 숨기고(private), 신뢰할 수 있는 메서드(public Getter / Setter)만을 통하여 객체의 안전한 상태 변화를 제공하는 것을 의미합니다.
* **캡슐화 규칙**:
  1. 클래스의 멤버 변수는 `private` 접근 제한자로 선언하여 직접적인 외부 접근을 차단합니다.
  2. 클래스의 변수 값을 읽거나 수정하기 위해 `public` 접근 제한자로 선언된 Getter() 및 Setter() 메서드를 노출시킵니다.
  ```java
  class Encapsulation {
      private String name; // 정보 은닉
      
      public String getName() { return name; } // Getter
      public void setName(String name) { this.name = name; } // Setter
  }
  ```

### 접근 제한자의 종류와 접근 범위
자바의 접근 제한자는 총 4가지 범위를 제공합니다.

| 접근 제한자 | 동일 클래스 | 동일 패키지 | 자식 클래스 (상속) | 전체 (다른 패키지) | 설명 |
| :--- | :---: | :---: | :---: | :---: | :--- |
| **`public`** | O | O | O | O | 패키지 관계없이 모든 클래스에서 접근이 허용됩니다. |
| **`protected`** | O | O | O (다른 패키지 가능) | X | 동일 패키지 내부, 혹은 다른 패키지에 있더라도 자식 클래스인 경우 접근이 허용됩니다. |
| **`default`** (생략 시) | O | O | X | X | **package-private**로 불리며, 오직 동일 패키지 내부의 클래스들에서만 접근이 허용됩니다. |
| **`private`** | O | X | X | X | 오직 자기 자신 클래스의 내부 멤버들에게만 접근이 허용됩니다. |

### 클래스 수준의 접근 제한
* 클래스 선언부 앞에도 접근 제한자를 지정할 수 있습니다.
  * **`public class`**: 다른 패키지를 포함한 모든 곳에서 객체를 생성하고 사용할 수 있는 클래스입니다. (소스 파일 이름과 public 클래스 이름은 반드시 일치해야 하며, 소스 파일 내 단 하나만 존재 가능)
  * **`default class` (접근제한자 생략)**: 동일 패키지 내의 클래스들만 접근할 수 있는 클래스입니다.
