## 45. 네트워크 - IoT(사물인터넷) 2대 핵심 프로토콜 (MQTT vs CoAP)

### 문제
다음 설명 중 괄호 ( ) 안에 들어갈 알맞은 IoT 관련 용어를 쓰시오.
* ( ① )은/는 IoT 장치, 텔레메트리 장치 등에서 최적화되어 사용할 수 있도록 개발된 프로토콜로, 브로커를 사용한 Publish/Subscribe 방식의 라이트 메시징을 전송하는 프로토콜로, 저전력 센서, 스위치, 밸브 등의 기기에 대한 표준적 인터넷 환경을 지원하고, 프로토콜 리소스 점유 최소화, 한정된 자원 시스템 지원하는 특징이 있다.
* ( ② )은/는 M2M 노드들 사이에서 이벤트에 대한 송수신을 비동기적으로 전송하는 REST 기반의 프로토콜이자 제약이 있는(Constrained) 장치들을 위한 특수한 인터넷 애플리케이션 프로토콜이다.

### 정답 및 풀이 결과
* **작성한 답**: 미작성 (IoT 프로토콜 기초 개념 혼동 및 상세 매커니즘 분석 질의 요청)
* **모범 답안**:
  * **①: MQTT (Message Queuing Telemetry Transport)**
  * **②: CoAP (Constrained Application Protocol)**
* **결과**: **이론 완벽 정리 완료**

### 핵심 해설
* **MQTT (Message Queuing Telemetry Transport)**:
  * **정의**: 대역폭이 좁고 자원이 한정된 저전력 센서나 소형 디바이스 환경에 특화되어 설계된 TCP 기반의 가벼운 메시징 프로토콜입니다.
  * **핵심 동작**: 중앙에 **브로커(Broker)**를 두고, 데이터를 보내는 발행자(Publisher)와 데이터를 받는 구독자(Subscriber)가 특정 주제(Topic)를 매개로 통신하는 **`Publish / Subscribe`** 모델을 가집니다.
  * **지문 단서**: "브로커", "Publish/Subscribe", "텔레메트리" $\rightarrow$ **`MQTT`**의 고유 단서어입니다.
* **CoAP (Constrained Application Protocol)**:
  * **정의**: 메모리나 연산 장치 성능이 극도로 제한된(Constrained) 소형 디바이스들 간의 통신을 위해 IETF에서 표준화한 UDP 기반의 경량 프로토콜입니다.
  * **핵심 동작**: 기존의 무거운 인터넷 프로토콜인 HTTP/TCP를 소형 기기에서 쓸 수 없기에, HTTP의 **`REST`** 구조(GET, POST, PUT, DELETE)를 그대로 채용하되 UDP 기반으로 압축 전송하여 리소스 소모를 최소화합니다.
  * **지문 단서**: "REST 기반", "제약이 있는(Constrained)" $\rightarrow$ **`CoAP`**의 고유 단서어입니다.

## 46. SQL - 데이터 삽입 조작어 (INSERT INTO VALUES) 작성

### 문제
다음 [학생] 테이블 스키마에 대한 명세이다. [학생] 테이블에서 학번이 ‘202101’이고, 성명은 ‘임꺽정’, 과목은 ‘프로그래밍’, 전화번호는 ‘010-1234-5678’인 학생의 정보를 입력하는 SQL 쿼리를 작성하시오. (단, 전화번호의 ‘010너234-5678’은 OCR 오타로 ‘010-1234-5678’로 교정하여 풀이한다.)

[학생] 테이블 스키마
* `학번`: INTEGER
* `성명`: VARCHAR(10)
* `과목명`: VARCHAR(20)
* `전화번호`: CHAR(13)

### 정답 및 풀이 결과
* **작성한 답**: `insert into 학생 values (202101, '임꺽정', '프로그래밍', '010-1234-5678')` (타입별 따옴표 매핑 및 INSERT 구문 완벽 매칭)
* **모범 답안**:
  * **`INSERT INTO 학생 VALUES (202101, '임꺽정', '프로그래밍', '010-1234-5678');`** (또는 소문자 기입 및 컬럼명을 명시하는 버전도 정답)
* **결과**: **정답 (100% 매칭 및 문법 완벽 작성)**

### 핵심 해설
* **데이터 삽입 조작어(INSERT) 표준 문법**:
  * 형식 1 (전체 컬럼 매핑 시 컬럼명 생략):
    * `INSERT INTO 테이블명 VALUES (값1, 값2, ...);` (작성자님이 선택한 깔끔하고 정석적인 문법입니다.)
  * 형식 2 (특정 컬럼만 지정하거나 가독성을 증대할 때):
    * `INSERT INTO 테이블명 (컬럼1, 컬럼2, ...) VALUES (값1, 값2, ...);`
* **자료형에 따른 값의 표현 규칙 (중요)**:
  * **숫자 타입 (`INTEGER`)**: 숫자 `202101`은 싱글 쿼테이션(작은따옴표 `'`) 없이 정수형 그대로 대입해야 문법적으로 완벽합니다.
  * **문자 타입 (`VARCHAR`, `CHAR`)**: 문자열값인 `'임꺽정'`, `'프로그래밍'`, `'010-1234-5678'`은 반드시 작은따옴표 `'`로 감싸서 문자 리터럴로 식별시켜야 에러가 나지 않습니다.
* **오타 보정 분석**:
  * 지문 상의 `010너234-5678` 오타를 표준적인 전화번호 포맷인 `010-1234-5678`로 매끄럽게 교정하여 정확하게 대입하셨습니다.
