## 1. 소프트웨어 아키텍처 - SOA (Service Oriented Architecture)

### 문제

### 정답 및 풀이 결과

* **결과**: 이론 정리 완료

### 핵심 해설
* **SOA**: 비즈니스 프로세스에 맞춰 서비스를 조합하여 시스템을 개발하는 아키텍처. 느슨한 결합(Loosely Coupled)으로 인해 변경에 유연하고 재사용성이 높음.

## 2. Java 재귀 함수 호출 연산

### 문제
아래 자바 코드를 실행했을 때 출력되는 결과를 쓰시오.
```java
public class Soojebi {
    public static void main(String[ ] args) {
        System.out.println(calc(2,2));
    }
    static int calc(int a, int b) {
        if (a<=0) return b;
        if (b<=0) return a;
        return calc(a-2, b)+calc(a, b-1);
    }
}
```

### 정답 및 풀이 결과
* **작성한 답**: 미작성 (강사 풀이용 질문)
* **모범 답안**: **`5`**
* **결과**: 이론 정리 완료

### 핵심 해설
* **재귀 함수 호출 트리 추적**:
  * `calc(2, 2)` -> `calc(0, 2) + calc(2, 1)`
  * `calc(0, 2)`는 `a <= 0` 조건에 따라 `b`인 `2`를 반환.
  * `calc(2, 1)` -> `calc(0, 1) + calc(2, 0)` 호출.
  * `calc(0, 1)`은 `a <= 0` 조건에 따라 `b`인 `1`을 반환.
  * `calc(2, 0)`은 `b <= 0` 조건에 따라 `a`인 `2`를 반환.
  * 따라서 `calc(2, 1)` = `1 + 2 = 3`.
  * 최종적으로 `calc(2, 2)` = `2 + 3 = 5`.

## 3. C언어 구조체와 포인터를 활용한 Stack 구현

### 문제
아래 C 코드를 실행했을 때 출력되는 결과를 쓰시오.
```c
#include <stdio.h>
#define SIZE 3
typedef struct {
    int data[SIZE];
    int top;
} Soojebi;
void push(Soojebi *s, int val) {
    s->data[++(s->top)]=val;
}
int pop(Soojebi *s) {
    return s->data[(s->top)--];
}
int main() {
    Soojebi s={{0}, -1};
    int a, b;
    push(&s, 1);
    push(&s, 2);
    pop(&s);
    push(&s, 3);
    a=pop(&s);
    b=pop(&s);
    printf("%d %d", b, a);
    return 0;
}
```

### 정답 및 풀이 결과
* **작성한 답**: 미작성 (강사 풀이용 질문)
* **모범 답안**: **`1 3`**
* **결과**: 이론 정리 완료

### 핵심 해설
* **스택 동작 상태 추적**:
  * 초기 상태: `s.data = {0, 0, 0}`, `s.top = -1`
  * `push(&s, 1)`: `top`을 먼저 `0`으로 증가시킨 후 `data[0] = 1`. 상태: `top = 0`, `data = {1, 0, 0}`
  * `push(&s, 2)`: `top`을 `1`로 증가시킨 후 `data[1] = 2`. 상태: `top = 1`, `data = {1, 2, 0}`
  * `pop(&s)`: 현재 `top = 1`인 `data[1] (2)`을 반환한 후 `top = 0`으로 감소. 상태: `top = 0` (값 2는 무시됨)
  * `push(&s, 3)`: `top`을 `1`로 증가시킨 후 `data[1] = 3`. 상태: `top = 1`, `data = {1, 3, 0}`
  * `a = pop(&s)`: 현재 `top = 1`인 `data[1] (3)`을 읽어 `a`에 저장한 후 `top = 0`으로 감소. (`a = 3`)
  * `b = pop(&s)`: 현재 `top = 0`인 `data[0] (1)`을 읽어 `b`에 저장한 후 `top = -1`로 감소. (`b = 1`)
  * `printf("%d %d", b, a)` 실행: `b`와 `a` 순서대로 출력하므로 `1 3` 출력.

## 4. C언어 2차원 배열과 포인터의 물리적 연속성 연산

### 문제
아래 C 코드를 실행했을 때 출력되는 결과를 쓰시오.
```c
#include <stdio.h>
int main(){
    int a[3][3]={1, 2, 3, 4, 5, 6, 7, 8, 9};
    int *p=a[1]+2;
    int *q=&a[0][1];

    printf("%d", p[1]);
    printf("%d", q[2]);
    return 0;
}
```

### 정답 및 풀이 결과
* **작성한 답**: 미작성 (강사 풀이용 질문)
* **모범 답안**: **`74`**
* **결과**: 이론 정리 완료

### 핵심 해설
* **2차원 배열의 메모리 연속 배치**:
  * 2차원 배열 `int a[3][3]`은 논리적으로는 3행 3열이지만, 메모리상에는 `1, 2, 3, 4, 5, 6, 7, 8, 9`가 연속된 1차원 공간에 할당된다.
* **포인터 `p` 분석**:
  * `a[1]`은 `a[1][0]`의 시작 주소(값 `4`가 있는 위치)이다.
  * `p = a[1] + 2`는 `&a[1][0] + 2` 이므로 `&a[1][2]` (값 `6`이 있는 위치)를 가리킨다.
  * `p[1]`은 포인터 `p`로부터 한 칸 더 뒤인 주소(`*(p + 1)`)를 역참조하므로, `a[1][2]`의 다음 물리 메모리 주소인 `a[2][0]`의 값 **`7`**을 가리킨다.
* **포인터 `q` 분석**:
  * `q = &a[0][1]`은 값 `2`가 있는 위치를 가리킨다.
  * `q[2]`는 포인터 `q`로부터 두 칸 뒤인 주소(`*(q + 2)`)를 역참조하므로, `a[0][1]` $\rightarrow$ `a[0][2]` $\rightarrow$ `a[1][0]`에 해당하여 값 **`4`**를 가리킨다.
* **출력 결과**:
  * `%d` 형식으로 연속해 출력하므로 `7`과 `4`가 공백이나 줄바꿈 없이 출력되어 **`74`**가 된다.

## 5. 애플리케이션 테스트 관리 - 정적 테스트 검토 기법

### 문제
다음은 정적 테스트에 대한 설명이다. 괄호 ( ) 안에 들어갈 용어를 쓰시오.
* 동료 검토(Peer Review)는 2〜3명이 진행하는 리뷰의 형태로 요구사항 명세서 작성자가 요구사항 명세서를 설명하고, 이해관계자들이 설명을 들으면서 결함을 발견하는 형태로 진행하는 검토 기법이다.
* ( ① )은/는 소프트웨어 요구, 설계, 원시 코드 등의 저작자 외의 다른 전문가 또는 팀이 검사하여 오류를 찾아내는 공식적 검토 방법이다.
* ( ② )은/는 검토 자료를 회의 전에 배포해서 사전 검토한 후 짧은 시간 동안 회의를 진행하는 형태로 리뷰를 통해 오류를 검출하고 문서로 만드는 비공식적인 검토 기법이다.

### 정답 및 풀이 결과
* **작성한 답**: 미작성 (강사 풀이용 질문)
* **모범 답안**:
  * **①: 인스펙션 (Inspection / 검사)**
  * **②: 워크스루 (Walkthrough)**
* **결과**: 이론 정리 완료

### 핵심 해설
* **정적 테스트(Static Test)**: 프로그램을 실행하지 않고, 명세서나 소스 코드를 대상으로 분석하는 테스트 기법.
* **대표적인 검토(Review) 기법 3가지**:
  1. **동료 검토 (Peer Review)**: 2~3명이 진행하며, 작성자가 설명하고 이해관계자가 들으면서 결함을 발견하는 형태.
  2. **인스펙션 (Inspection)**: 계획-사전준비-개인검토-검토회의-재작업-후속작업 등의 절차를 거치는 **가장 공식적이고 엄격한** 검토 방법. 저작자 외의 다른 전문가/팀이 참여.
  3. **워크스루 (Walkthrough)**: 요구사항 명세서 등을 **회의 전에 배포하여 사전 검토**한 후, 회의를 진행하며 오류를 검출하고 문서화하는 **비공식적** 검토 방법.

## 6. 데이터베이스 - SQL LIKE 연산자 와일드카드 활용

### 문제
다음은 [사전] 테이블이다. [결과] 테이블처럼 'symmetry'이라는 문자열 앞에 한 글자가 추가된 문자열을 찾기 위한 쿼리를 작성하시오.(symmetry, anti-symmetry와 같이 symmetry 문자열 앞에 한 글자가 추가되지 않는 문자열은 조회되지 않아야 한다.)

[사전] 테이블
* 단어 / 뜻
* symmetry / 대칭
* asymmetry / 비대칭
* anti-symmetry / 반대칭
* Rsymmetry / R대칭
* Tsymmetry / T대칭

[결과]
* 단어 / 뜻
* asymmetry / 비대칭
* Rsymmetry / R대칭
* Tsymmetry / T대칭

### 정답 및 풀이 결과
* **작성한 답**: 미작성 (강사 풀이용 질문)
* **모범 답안**:
  * **`SELECT * FROM 사전 WHERE 단어 LIKE '_symmetry';`**
  * (또는 `SELECT 단어, 뜻 FROM 사전 WHERE 단어 LIKE '_symmetry';`)
* **결과**: 이론 정리 완료

### 핵심 해설
* **SQL `LIKE` 연산자 패턴 매칭**:
  * **`_` (언더바 / 언더스코어)**: **정확히 임의의 한 글자**와 매칭됩니다.
  * **`%` (퍼센트)**: **0개 이상의 임의의 문자열**과 매칭됩니다.
* **조건 매칭 분석**:
  * `_symmetry` 패턴은 앞에 '정확히 한 글자'가 오고 뒤에 'symmetry'가 오는 단어만 선택합니다.
  * `asymmetry` ('a' + 'symmetry') $\rightarrow$ **매칭 성공**
  * `Rsymmetry` ('R' + 'symmetry') $\rightarrow$ **매칭 성공**
  * `Tsymmetry` ('T' + 'symmetry') $\rightarrow$ **매칭 성공**
  * `symmetry` (앞에 글자 없음) $\rightarrow$ **매칭 실패** (최소 한 글자 필요)
  * `anti-symmetry` (앞에 'anti-'로 5글자) $\rightarrow$ **매칭 실패**

## 7. C언어 함수 포인터 (Function Pointer) 대입 연산

### 문제
다음은 C언어 코드이다. 밑줄에 알맞은 코드를 작성하시오.
[출력결과]
91

```c
#include <stdio.h>
int add(int i, int j){
    return i+j;
}
int sub(int i, int j){
    return i-j;
}
int main(){
    int (*pf)(int, int);
    ① ;
    printf("%d", pf(5, 4));
    ② ;
    printf("%d", pf(5, 4));
    return 0;
}
```

### 정답 및 풀이 결과
* **작성한 답**: 미작성 (강사 풀이용 질문)
* **모범 답안**:
  * **①: pf = add** (또는 `pf = &add`)
  * **②: pf = sub** (또는 `pf = &sub`)
* **결과**: 이론 정리 완료

### 핵심 해설
* **함수 포인터 (Function Pointer)**:
  * 함수의 시작 주소를 가리키는 포인터 변수입니다.
  * 선언 형식: `반환타입 (*포인터변수명)(매개변수타입1, 매개변수타입2, ...);`
  * 대입 형식: 함수 포인터 변수에 함수의 이름을 대입하여 함수를 가리키게 합니다. (함수의 이름 자체가 함수의 시작 주소를 나타내는 포인터 상수이기 때문입니다.)
* **출력 결과 분석**:
  * 출력 결과가 `91`입니다. 줄바꿈이 없으므로 첫 번째 출력은 `9`, 두 번째 출력은 `1`이 되어야 합니다.
  * `5`와 `4`를 매개변수로 전달했을 때 `9`가 나오는 함수는 `add(5, 4) = 5 + 4 = 9`이므로, ①에서는 `pf`에 `add` 함수의 주소를 대입해야 합니다. (`pf = add;`)
  * `5`와 `4`를 매개변수로 전달했을 때 `1`이 나오는 함수는 `sub(5, 4) = 5 - 4 = 1`이므로, ②에서는 `pf`에 `sub` 함수의 주소를 대입해야 합니다. (`pf = sub;`)

## 11. 요구사항 확인 - 정형 명세 기법 (Formal Specification)

* 정형 명세 언어인 Z-스키마, Petri Nets, 상태 차트 활용
* 표현이 간결, 명확성 및 검증이 용이
* 기법의 이해가 어려움

* **작성한 답**: 미작성 (강사 풀이용 질문)

* **결과**: 이론 정리 완료

### 핵심 해설
* **요구사항 명세 기법의 두 가지 흐름**:
  * **정형 명세 기법 (Formal Specification)**:
    * **수학적 모델, 집합론, 논리식** 등 수학적 원리와 기호를 기반으로 요구사항을 작성합니다.
    * 모호함이 없어 일관성이 높고 시스템 검증이 쉬우나, 수학적 표현 방식이기 때문에 개발자나 사용자가 이해하기 어렵다는 단점이 있습니다.
    * 사용 기법: **Z, Petri Net(페트리 네트), 상태 차트, VDM** 등
  * **비정형 명세 기법 (Informal Specification)**:
    * **자연어, 상태도, 유스케이스 다이어그램** 등 사람이 이해하기 쉬운 언어나 표, 그림으로 명세합니다.
    * 작성과 이해가 쉬워 고객과의 의사소통에 유리하지만, 명세가 모호할 수 있고 일관성 검증이 어렵습니다.
    * 사용 기법: **DFD(데이터 흐름도), DD(자료 사전), 상태 전이도(STD)** 등

## 12. C언어 비트 연산자 (Bitwise AND, OR)

### 문제
아래 C 코드를 실행했을 때 출력되는 결과를 쓰시오.
```c
#include <stdio.h>
int main(){
    int a=30, b=15;
    printf("%d", a&b);
    printf("%d", a|b);
    return 0;
}
```

### 정답 및 풀이 결과
* **작성한 답**: 미작성 (강사 풀이용 질문)
* **모범 답안**: **`1431`**
* **결과**: 이론 정리 완료

### 핵심 해설
* **비트 연산자 (Bitwise Operator) 개념**:
  * 데이터를 비트(0과 1) 단위로 논리 연산을 수행합니다.
  * `&` (Bitwise AND): 두 비트가 모두 1일 때만 1을 반환.
  * `|` (Bitwise OR): 두 비트 중 하나라도 1이면 1을 반환.
* **이진수 변환 및 연산 추적**:
  * 십진수 `30` $\rightarrow$ 이진수 `0001 1110`
  * 십진수 `15` $\rightarrow$ 이진수 `0000 1111`
  * **AND (`a & b`)**:
    ```text
      0001 1110  (30)
    & 0000 1111  (15)
    -----------
      0000 1110  $\rightarrow$ 십진수 $8+4+2 = 14$
    ```
  * **OR (`a | b`)**:
    ```text
      0001 1110  (30)
    | 0000 1111  (15)
    -----------
      0001 1111  $\rightarrow$ 십진수 $16+8+4+2+1 = 31$
    ```
* **최종 출력**:
  * 첫 번째 출력: `14`
  * 두 번째 출력: `31`
  * 줄바꿈이나 공백 지시어가 없으므로 나란히 붙여서 **`1431`**이 출력됩니다.

## 14. 데이터베이스 - 부분 함수 종속성과 제2정규형 (2NF)

### 문제
다음 설명에서 괄호 ( ) 안에 알맞은 용어를 쓰시오.

고객명 / 서비스 이름 / 서비스 가격 / 서비스 이용 기간
* 홍길동 / 헬스 / 70000 / 1달
* 홍길동 / 수영 / 100000 / 2달
* 장길산 / 수영 / 100000 / 2달

고객명 / 서비스 이름 / 서비스 이용 기간
* 홍길동 / 헬스 / 1달
* 홍길동 / 수영 / 2달
* 장길산 / 수영 / 2달

서비스 이름 / 서비스 가격
* 헬스 / 70000
* 수영 / 100000

* 〈고객명, 서비스 이름〉이〈서비스 이용 기간〉에 영향을 주고,〈서비스 이름〉이〈서비스 가격〉에 영향을 주는 관계를 ( ① )(이)라고 한다.
  * 고객명, 서비스 이름 $\rightarrow$ 서비스 이용 기간
  * 서비스 이름 $\rightarrow$ 서비스 가격
* 〈고객명, 서비스 이름, 서비스 가격, 서비스 이용 기간〉을 한 테이블에 두는 것은 ( ① )(으)로 인해 ( ② )을/를 만족하지 못한다.
* 부분 관계인〈서비스 이름, 서비스 가격〉관계를 별도의 테이블로 두면 ( ① )이/가 제거되어 ( ② )을/를 만족한다.

### 정답 및 풀이 결과
* **작성한 답**: 미작성 (강사 풀이용 질문)
* **모범 답안**:
  * **①: 부분 함수 종속 (Partial Functional Dependency / 부분 함수적 종속성)**
  * **②: 제2정규형 (2NF / 제 2 정규형)**
* **결과**: 이론 정리 완료

### 핵심 해설
* **부분 함수 종속 (Partial Functional Dependency)**:
  * 기본키가 복합키(둘 이상의 속성)로 구성되어 있을 때, 비기본 속성이 기본키 전체에 종속되지 않고 **기본키의 일부 속성에만 종속**되는 관계입니다.
  * 본 문제에서 기본키는 `{고객명, 서비스 이름}`의 복합키입니다.
  * `{고객명, 서비스 이름}` $\rightarrow$ `{서비스 이용 기간}` 은 기본키 전체가 결정하는 완전 함수 종속입니다.
  * 반면 `{서비스 이름}` $\rightarrow$ `{서비스 가격}` 은 복합키의 일부인 `{서비스 이름}`이 `{서비스 가격}`을 결정하므로 **부분 함수 종속**에 해당합니다.
* **정규화 단계와 조건**:
  * **1정규형 (1NF)**: 모든 도메인이 원자값(Atomic Value)으로만 구성됨.
  * **2정규형 (2NF)**: 기본키의 일부 속성에 종속되는 **부분 함수적 종속성을 제거**하여, 모든 비기본 속성이 기본키에 **완전 함수적 종속**을 만족하도록 함.
  * **3정규형 (3NF)**: 기본키가 아닌 일반 속성들 간의 **이행적 함수적 종속성 제거** ($A \rightarrow B$ 이고 $B \rightarrow C$ 일 때 $A \rightarrow C$ 관계 성립 제거).

## 15. C언어 문자열 포인터 연산과 아스키코드(ASCII) 변위 계산

### 문제
아래 C 코드를 실행했을 때 출력되는 결과를 쓰시오.
```c
#include <stdio.h>
int main(){
    char a[6]="hello";
    printf("%s", a+2);
    printf("%c", a[1]+2);
    printf("%c", *a+1);
    printf("%c", *(a+1));
    printf("%c", a[1]);
    return 0;
}
```

### 정답 및 풀이 결과
* **작성한 답**: 미작성 (강사 풀이용 질문)
* **모범 답안**: **`llogieee`**
* **결과**: 이론 정리 완료

### 핵심 해설
* **포인터 덧셈 vs 아스키코드 값 덧셈**:
  * **포인터 연산 (`a+2`)**: 문자열 `"hello"`의 3번째 글자 `'l'`이 있는 시작 주소를 가리킵니다. `%s` 포맷으로 출력하므로 해당 위치부터 문자열 끝까지인 **`llo`**가 출력됩니다.
  * **아스키코드 값 연산 (`a[1]+2`)**: `a[1]`인 `'e'`(아스키코드 101)에 2를 더하므로 `'g'`(103)가 됩니다. `%c` 포맷이므로 문자 **`g`**가 출력됩니다.
  * **연산자 우선순위 (`*a+1`)**: 역참조 연산자 `*`가 덧셈 `+`보다 우선순위가 높으므로 `(*a) + 1`로 연산됩니다. `*a`는 `a[0]`인 `'h'`(104)이며, 1을 더하면 `'i'`(105)가 됩니다. 따라서 문자 **`i`**가 출력됩니다.
  * **주소 연산 후 역참조 (`*(a+1)`)**: 주소 `a`에 1을 더한 주소(`&a[1]`)를 역참조하므로 `'e'`가 됩니다. 문자 **`e`**가 출력됩니다.
  * **배열 직접 참조 (`a[1]`)**: `'e'`가 출력됩니다.
* **최종 출력 취합**:
  * 줄바꿈이나 공백이 없으므로 각 단계별 출력 `llo`, `g`, `i`, `e`, `e`가 연달아 출력되어 **`llogieee`**가 최종 결과입니다.

## 16. 데이터베이스 - DDL ALTER TABLE을 이용한 컬럼 크기 수정

## 데이터베이스 핵심 - SQL 분류 (DDL, DML, DCL, TCL) 및 트랜잭션 ACID

### 1. SQL 언어의 분류 (★필수 암기★)

데이터베이스 조작 및 정의를 위한 SQL은 기능에 따라 크게 3가지(TCL을 분리할 경우 4가지)로 분류됩니다.

| 분류      | 영문 풀네임                                     | 대상                           | 주요 명령어                                 | 역할                     | 암기 팁                          |
| :------ | :----------------------------------------- | :--------------------------- | :------------------------------------- | :--------------------- | :---------------------------- |
| **DDL** | Data Definition Language<br>(데이터 정의어)      | **스키마, 테이블,<br>뷰, 인덱스 등 객체** | `CREATE`, `ALTER`, `DROP`, `TRUNCATE`  | 데이터베이스 구조를 생성, 변경, 삭제함 | **C.A.D.T**<br>(캐디가 드롭하고 턴한다) |
| **DML** | Data Manipulation Language<br>(데이터 조작어)    | **테이블 내부의<br>실제 행 (데이터)**    | `SELECT`, `INSERT`, `UPDATE`, `DELETE` | 데이터를 조회, 삽입, 수정, 삭제함   | **S.I.U.D**<br>(시드머니를 조작한다)   |
| **DCL** | Data Control Language<br>(데이터 제어어)         | **보안, 권한,<br>무결성, 회복**       | `GRANT`, `REVOKE`                      | 사용자 권한 부여 및 회수를 관리함    | **G.R**<br>(그랜절로 권한 요청)       |
| **TCL** | Transaction Control Language<br>(트랜잭션 제어어) | **트랜잭션 제어**                  | `COMMIT`, `ROLLBACK`, `SAVEPOINT`      | 트랜잭션의 연산을 확정하거나 취소함    | **C.R.S**<br>(코스를 되돌린다)       |

> [!NOTE]
> **DROP vs TRUNCATE vs DELETE 차이**
> * `DROP` (DDL): 테이블 구조 자체를 통째로 삭제 (공간 반환, Rollback 불가)
> * `TRUNCATE` (DDL): 테이블 구조는 두고 데이터만 전체 삭제 (공간 반환, Rollback 불가, 속도 빠름)
> * `DELETE` (DML): 조건(`WHERE`)에 따라 행 데이터 삭제 (공간 반환 없음, Rollback 가능)

---

### 2. 트랜잭션(Transaction)의 4대 특성 (ACID)

트랜잭션이 안전하게 수행되기 위해 보장해야 하는 4대 성질로, 실기에서 약어 및 정의 매칭으로 무조건 출제됩니다.

1. **원자성 (Atomicity)**
   * 트랜잭션 내 연산은 전부 반영되거나 아예 반영되지 않아야 함 (**All or Nothing**).
   * 관련 명령어: `COMMIT`, `ROLLBACK`
2. **일관성 (Consistency)**
   * 트랜잭션이 성공적으로 완료되면 언제나 일관성 있는 데이터베이스 상태로 보존되어야 함 (예: 시스템 무결성 제약조건 준수).
3. **고립성 / 독립성 (Isolation)**
   * 동시에 여러 트랜잭션이 실행될 때, 어느 하나의 트랜잭션도 다른 트랜잭션의 중간 연산에 간섭할 수 없음.
4. **영속성 / 지속성 (Durability)**
   * 트랜잭션이 완료된 결과는 시스템에 장애가 발생하더라도 영구적으로 보존되어야 함.

## 17. 소프트웨어 설계 - 디자인 패턴 (Abstract Factory, Prototype)

### 문제
다음이 설명하는 디자인 패턴을 영어로 쓰시오.
* ① 구체적인 클래스에 의존하지 않고 서로 연관되거나 의존적인 객체들의 조합을 만드는 인터페이스를 제공하는 패턴으로 이 패턴을 통해 생성된 클래스에서는 사용자에게 인터페이스(API)를 제공하고, 구체적인 구현은 ConcreteProduct 클래스에서 이루어지는 특징을 갖는 디자인 패턴
* ② 처음부터 일반적인 원형을 만들어 놓고, 그것을 복사한 후 필요한 부분만 수정하여 사용하는 패턴으로, 생성할 객체의 원형을 제공하는 인스턴스에서 생성할 객체들의 타입이 결정되도록 설정하며 객체를 생성할 때 갖추어야 할 기본 형태가 있을 때 사용되는 디자인 패턴

### 정답 및 풀이 결과
* **작성한 답**: 미작성 (강사 풀이용 질문)
* **모범 답안**:
  * **①: Abstract Factory**
  * **②: Prototype**
* **결과**: 이론 정리 완료

### 핵심 해설
* **Abstract Factory (추상 팩토리) 패턴**:
  * 구체적인 클래스를 지정하지 않고 관련성 있거나 의존적인 객체들의 패밀리를 생성할 수 있는 인터페이스를 제공하는 생성(Creational) 패턴입니다.
  * 구체적인 팩토리 클래스(ConcreteFactory)에서 실제 생성될 제품 객체(ConcreteProduct)를 생성하여 리턴합니다.
* **Prototype (프로토타입) 패턴**:
  * 원형(Prototype)이 되는 인스턴스를 지정하고, 이 인스턴스를 복제(Clone)하여 새로운 객체를 생성하는 생성 패턴입니다.
  * 객체 생성 비용이 매우 크거나 복잡한 초기 형태를 가진 인스턴스를 효율적으로 여러 개 만들 때 유용합니다.

## 18. Java 디자인 패턴 - 싱글톤(Singleton) 패턴의 인스턴스 공유 연산

### 문제
아래 자바 코드를 실행했을 때 출력되는 결과를 쓰시오. (단, 2라인의 `ni』ll`은 오타인 `null`로 가정하여 풀이한다.)
```java
class Soojebi{
    static private Soojebi instance=null;
    private int count=0;
    static public Soojebi get(int x){
        if(instance==null){
            instance=new Soojebi();
        }
        instance.count += x;
        return instance;
    }
    public void count(){ count++;}
    public int getCount(){return instance.count;}
}

public class Soojebi2{
    public static void main(String[ ] args){
        Soojebi s1 =Soojebi.get(2);
        s1.count();
        Soojebi s2=Soojebi.get(3);
        s2.count();

        System.out.print(s1.getCount());
    }
}
```

### 정답 및 풀이 결과
* **작성한 답**: 미작성 (강사 풀이용 질문)
* **모범 답안**: **`7`**
* **결과**: 이론 정리 완료

### 핵심 해설
* **싱글톤 (Singleton) 패턴**:
  * 클래스의 인스턴스를 단 하나만 생성하고, 그 생성된 단일 인스턴스에 전역적으로 접근할 수 있도록 보장하는 패턴입니다.
  * `s1`과 `s2`는 서로 다른 변수이지만, 메모리에서 가리키는 대상은 동일한 `Soojebi` 클래스의 유일한 싱글톤 인스턴스(`instance`)입니다.
* **코드 실행 과정 상세 추적**:
  1. `Soojebi s1 = Soojebi.get(2);` 실행:
     * 최초 호출이므로 `instance`가 새로 생성됨 (`instance.count` 초기화값 0).
     * `instance.count += 2`가 실행되어 `count = 2`가 됨.
     * `s1`은 이 단일 인스턴스를 가리킵니다.
  2. `s1.count();` 실행:
     * `count`가 1 증가하여 `count = 3`이 됨.
  3. `Soojebi s2 = Soojebi.get(3);` 실행:
     * 이미 `instance`가 존재하므로 생성문을 패스함.
     * `instance.count += 3`이 실행되어 기존 `3`에 `3`이 더해져 `count = 6`이 됨.
     * `s2` 또한 동일한 인스턴스를 가리킵니다.
  4. `s2.count();` 실행:
     * `count`가 1 증가하여 `count = 7`이 됨.
  5. `System.out.print(s1.getCount());` 실행:
     * `s1`이 가리키는 인스턴스의 최종 `count`값인 **`7`**이 출력됩니다.

## 19. Java 상속 - 오버라이딩(Overriding)과 동적 바인딩(Dynamic Binding)의 메서드 연쇄 호출

### 문제
아래 자바 코드를 실행했을 때 출력되는 결과를 쓰시오.
```java
class SuperObj {
    public void show(){
        print();
    }
    public void print() {
        print();
        System.out.print("Super");
    }
}

class SubObj extends SuperObj {
    public void show() {
        super.print();
    }
    public void print() {
        System.out.print("Sub");
    }
}

public class Soojebi {
    public static void main(String[ ] args){
        SuperObj s=new SubObj();
        s.show();
    }
}
```

### 정답 및 풀이 결과
* **작성한 답**: 미작성 (강사 풀이용 질문)
* **모범 답안**: **`SubSuper`**
* **결과**: 이론 정리 완료

### 핵심 해설
* **상속과 동적 바인딩 (Dynamic Binding)**:
  * 부모 클래스형 참조 변수 `s`가 자식 클래스 인스턴스 `new SubObj()`를 참조할 때, 오버라이딩된 메서드가 호출되면 런타임 시 실제 객체(자식 클래스)의 메서드가 실행됩니다.
* **코드 실행 과정 상세 추적**:
  1. `s.show();` 실행:
     * `s`가 가리키는 실제 인스턴스는 `SubObj`이므로 `SubObj` 클래스의 `show()`가 호출됩니다.
  2. `SubObj`의 `show()` 내부:
     * `super.print();`가 호출되는데, `super` 키워드로 부모의 영역을 가리켰으므로 부모 클래스인 `SuperObj`의 `print()`가 실행됩니다.
  3. `SuperObj`의 `print()` 내부:
     * `print();` 호출문을 만납니다. 이는 객체 자신을 가리키는 `this.print();`와 같습니다.
     * 여기서 `this`가 가리키는 실제 런타임 인스턴스는 여전히 자식 객체인 **`SubObj`**입니다.
     * 따라서 동적 바인딩에 의해 **자식 클래스(`SubObj`)에 오버라이딩된 `print()`**가 실행되어 화면에 **`Sub`**가 출력됩니다.
     * 자식의 `print()` 호출이 끝난 후 다시 부모의 `print()`로 돌아와 그 다음 라인인 `System.out.print("Super");`가 수행되어 화면에 **`Super`**가 출력됩니다.
  4. 최종 출력값:
     * `Sub`와 `Super`가 연이어 출력되므로 **`SubSuper`**가 최종 정답입니다.

     * 성능(속도) 향상을 위해 인덱스(Index) 설계, 파티션(Partition) 설계, 역정규화(Denormalization) 등을 수행합니다.

## 21. 네트워크 - IP 주소 및 서브넷 마스크 연산 (네트워크 주소, 브로드캐스트 주소)

### 문제
어떤 호스트의 IP 주소와 서브넷 마스크가 다음과 같이 주어졌을 때, 질문에 알맞은 답을 쓰시오.
* 호스트 IP 주소: `192.168.1.130`
* 서브넷 마스크: `255.255.255.192`

[질문]
1. 이 호스트가 속한 네트워크 주소를 CIDR 표기법으로 쓰시오.
2. 이 호스트가 속한 네트워크의 브로드캐스트(Broadcast) 주소를 쓰시오.

### 정답 및 풀이 결과
* **작성한 답**: `192.168.1.128 / 26` (네트워크 주소), `192.168.1.191` (브로드캐스트 주소)
* **모범 답안**:
  * **1. `192.168.1.128/26`**
  * **2. `192.168.1.191`**
* **결과**: **정답 (100% 매칭)**

### 핵심 해설
* **네트워크 주소 계산**:
  * IP 주소의 4번째 옥텟 `130` $\rightarrow$ 이진수 `1000 0010`
  * 서브넷 마스크의 4번째 옥텟 `192` $\rightarrow$ 이진수 `1100 0000`
  * 비트 AND 연산: `1000 0010` & `1100 0000` = `1000 0000` (십진수 **`128`**)
  * 서브넷 마스크의 1비트 총 개수는 8+8+8+2 = **`26`**개 이므로 CIDR 접미사는 **`/26`**이 됩니다.
  * 따라서 네트워크 주소는 **`192.168.1.128/26`**입니다.
* **브로드캐스트 주소 계산**:
  * 네트워크 주소에서 호스트 비트 영역(뒤의 6비트)을 모두 1로 채웁니다.
  * `1000 0000` $\rightarrow$ `1011 1111` (십진수 **`191`**)
  * 따라서 브로드캐스트 주소는 **`192.168.1.191`**이 됩니다.

## 22. C언어 문자열 - 문자열 길이 계산 및 puts 주소 기반 부분 출력

### 문제
아래 C 코드를 실행했을 때 출력되는 결과를 쓰시오. (단, 7라인의 `a—5`는 오타인 `a-5`로 가정하여 풀이한다.)
```c
#include <stdio.h>
int main(){
    char str1[ ]="SOOJEBI";
    char str2[ ]="WORLD";
    int a=0;
    while (str1[a] != '\0') ++a;
    puts(&str2[a-5]);
    return 0;
}
```

### 정답 및 풀이 결과
* **작성한 답**: `RLD` (puts(&str2[a-5])는 str2[2]를 기점으로 끝까지 출력하라는 의미인지 질의 포함)
* **모범 답안**: **`RLD`**
* **결과**: **정답 (100% 매칭 및 개념 완벽 분석)**

### 핵심 해설
* **사용자 해석 검증**:
  * **"puts(&str2[a-5])는 str2[2]를 기점으로 끝까지 다 출력하라는 의미가 맞다"**는 판단은 정확히 100% 사실입니다.
* **코드 실행 과정 상세 추적**:
  1. `str1` = `"SOOJEBI"` 이므로 널 문자(`\0`)를 제외한 길이는 `7`입니다.
  2. `while (str1[a] != '\0') ++a;` 루프가 돌면서 변수 `a`는 `0`에서 시작하여 인덱스 `7`에 위치한 `\0`를 발견할 때까지 매 루프마다 `1`씩 증가합니다.
     * 결과적으로 루프가 끝났을 때 `a` = **`7`**이 됩니다.
  3. `puts(&str2[a-5]);` 의 변수를 치환하면 `puts(&str2[7-5]);` $\rightarrow$ `puts(&str2[2]);` 가 됩니다.
  4. `str2` = `"WORLD"`의 인덱스 매칭:
     * `str2[0]` = `'W'`, `str2[1]` = `'O'`, `str2[2]` = **`'R'`**, `str2[3]` = `'L'`, `str2[4]` = `'D'`, `str2[5]` = `'\0'`
  5. `&str2[2]`는 문자 `'R'`의 시작 메모리 주소를 나타냅니다.
  6. **`puts`** 함수는 문자열 포인터(시작 주소)를 인자로 받아 해당 주소부터 **널 문자(`\0`)를 만날 때까지** 문자들을 출력하고 **자동 개행(`\n`)**을 하는 함수입니다.
  7. 따라서 `'R'`부터 문자열 끝까지인 **`RLD`**가 정상 출력되고 개행됩니다.

## 23. Java 제어문 - 라벨 지정 continue (Labeled continue) 연산

### 문제
아래 자바 코드를 실행했을 때 출력되는 결과를 쓰시오. (단, 5라인의 `j너`는 오타인 `j=1`로 가정하여 풀이한다.)
```java
public class Soojebi {
    public static void main(String[ ] args) {
        int sum=0;
        x: for (int i=1; i<= 3; i++) {
            for (int j=1; j<= 3; j++) {
                if (i+j==3) continue x;
                else sum += i * j;
            }
        }
        System.out.println(sum);
    }
}
```

### 정답 및 풀이 결과
* **작성한 답**: `19` (sum = 1 + 3 + 6 + 9 = 19, (1,2)와 (2,1)에서 스킵이 일어나 해당 루프 분기가 넘어감에 대한 상세 분석 포함)
* **모범 답안**: **`19`**
* **결과**: **정답 (100% 매칭 및 동작 논리 완벽 증명)**

### 핵심 해설
* **라벨 지정 제어문 (Labeled Control Statements)**:
  * 자바에서는 중첩 루프 구조에서 특정 루프 앞에 라벨(예: `x:`)을 지정하고, `continue 라벨명;` 또는 `break 라벨명;`을 호출하여 원하는 상위 루프로 단번에 흐름을 이동시킬 수 있습니다.
  * `continue x;`가 실행되면, 내부 루프를 완전히 중단하고 라벨 `x`가 지정된 **외부 `for` 루프의 증감식(`i++`)**으로 제어가 즉시 이동합니다.
* **디버깅 표 (Trace Table)**:
  * **`i = 1`**:
    * `j = 1` $\rightarrow i+j=2 \neq 3 \rightarrow$ `sum += 1 * 1` $\rightarrow$ **`sum = 1`**
    * `j = 2` $\rightarrow i+j=3 \rightarrow$ **`continue x;`** 호출 $\rightarrow$ 내부 루프 중단 후 즉시 `i = 2` 분기로 점프 (즉, `j = 3` 연산은 아예 수행되지 않고 패스됨!)
  * **`i = 2`**:
    * `j = 1` $\rightarrow i+j=3 \rightarrow$ **`continue x;`** 호출 $\rightarrow$ 즉시 `i = 3` 분기로 점프 (`j = 2`, `j = 3` 연산 패스!)
  * **`i = 3`**:
    * `j = 1` $\rightarrow i+j=4 \neq 3 \rightarrow$ `sum += 3 * 1` $\rightarrow$ **`sum = 1 + 3 = 4`**
    * `j = 2` $\rightarrow i+j=5 \neq 3 \rightarrow$ `sum += 3 * 2` $\rightarrow$ **`sum = 4 + 6 = 10`**
    * `j = 3` $\rightarrow i+j=6 \neq 3 \rightarrow$ `sum += 3 * 3` $\rightarrow$ **`sum = 10 + 9 = 19`**
  * 최종적으로 `sum`값인 **`19`**가 화면에 출력됩니다.

### 💡 ★강사 특별 조언: Labeled continue & Labeled break 완전 정복★

많은 수험생들이 일반 `continue`와 라벨 지정 `continue`를 헷갈려서 틀립니다. 이 개념을 시각적으로 확실하게 정리하고 넘어가겠습니다.

#### 1. 일반 `continue` vs Labeled `continue` 흐름도

* **일반 `continue;`** (자신이 속한 가장 가까운 루프를 제어)
  ```text
  for (int i=1; i<=3; i++) {       <───────┐
      for (int j=1; j<=3; j++) {   <──┐    │
          if (조건) continue; ────────┘    │  (내부 루프 j++로 이동)
          sum += i * j;                      │
      }                                      │
  }                                          │
  ```
* **라벨 지정 `continue x;`** (지정된 라벨 루프를 제어)
  ```text
  x: for (int i=1; i<=3; i++) {    <──────────────┐ (외부 루프 i++로 바로 이동!)
      for (int j=1; j<=3; j++) {   <──┐           │
          if (조건) continue x; ──────────────────┘ (내부 루프 j의 잔여 연산은 즉시 증발!)
          sum += i * j;
      }
  }
  ```

#### 2. Labeled `break`와의 결정적 차이점

라벨을 타겟으로 삼는 점은 같지만, 동작의 범위가 다릅니다.

* **`continue x;`**:
  * 외부 루프 `x`를 **완전히 끝내는 것이 아니라**, 현재 분기(`i=1`일 때의 남은 과정)만 건너뛰고 **다음 바퀴(`i=2`)를 이어서 계속 돌립니다.**
* **`break x;`**:
  * 외부 루프 `x`의 블록 바깥으로 탈출하라는 의미입니다. 즉, 중첩 루프 전체를 **그 즉시 영구 종료**하고 아래쪽의 출력문으로 내려갑니다.

#### 3. 암기 팁 & 시험장 대처법
* 1단계: 코드에서 `continue`나 `break` 뒤에 알파벳(라벨명)이 붙어 있는지 확인한다.
* 2단계: 라벨이 붙어있다면, 화살표를 그려 해당 라벨이 걸려있는 **바깥쪽 `for`문의 머리(증감식)**로 제어선을 직접 연결한다.
* 3단계: `continue`인 경우 증감식을 거쳐 바깥 루프를 계속 진행하고, `break`인 경우 바깥 루프의 중괄호(`}`) 바깥으로 제어선을 빼낸다.

## 24. C언어 구조체 - 구조체 배열 연산 및 버블 정렬(Bubble Sort) 내림차순 정렬

### 문제
아래 C 코드를 실행했을 때 출력되는 결과를 쓰시오. (단, 1라인의 `ffinclude`는 `#include`로, 10라인의 `struct p=x[3]`은 `struct p x[3]`으로, 18라인의 `2—i`는 `2-i`로 간주하여 풀이한다.)
```c
#include <stdio.h>
struct p {
    char name;
    int s;
    int w;
    float p;
};

int main(){
    struct p x[3] = {{'A', 3, 2}, {'B', 10, 7}, {'C', 7, 3}};
    struct p temp;
    int i, j;

    for(i=0; i<3; i++)
        x[i].p = (x[i].w + x[i].s) / (float)(x[i].w);

    for(i=0; i<2; i++){
        for(j=0; j<2-i; j++){
            if(x[j].p < x[j+1].p){
                temp = x[j];
                x[j] = x[j+1];
                x[j+1] = temp;
            }
        }
    }

    for(i=0; i<3; i++)
        printf("%c", x[i].name);

    return 0;
}
```

### 정답 및 풀이 결과
* **작성한 답**: `CAB` (float 나눗셈 연산 및 버블 정렬 내림차순 원리를 정확히 계산하여 짚음)
* **모범 답안**: **`CAB`**
* **결과**: **정답 (100% 매칭 및 분석 성공)**

### 핵심 해설
* **데이터 실수 나눗셈 연산**:
  * C언어에서 정수 간 나눗셈(`/`)은 소수점을 절삭하지만, 나누는 수인 `x[i].w`를 `(float)`로 실수 캐스팅했기 때문에 정확한 실수 나눗셈 연산이 수행됩니다.
  * `A`: `(2 + 3) / 2.0` = `5 / 2.0` = **`2.5`**
  * `B`: `(7 + 10) / 7.0` = `17 / 7.0` = **`2.428...`**
  * `C`: `(3 + 7) / 3.0` = `10 / 3.0` = **`3.333...`**
* **버블 정렬 조건식**:
  * `if (x[j].p < x[j+1].p)`는 앞의 값(`x[j]`)이 뒤의 값(`x[j+1]`)보다 **작을 때** 서로 맞바꿉니다.
  * 이것은 큰 값이 앞으로 오게 하는 **내림차순(Descending) 정렬**을 수행하는 조건식입니다.
  * 세 값의 크기 서열: `C (3.333...)` > `A (2.5)` > `B (2.428...)`
  * 내림차순 정렬 완료 후의 배열 순서: `C` $\rightarrow$ `A` $\rightarrow$ `B`
* **출력**:
  * 루프를 돌면서 `x[i].name`을 출력하므로 최종 출력 문자열은 **`CAB`**가 됩니다.

## 25. 애플리케이션 테스트 - 검증(Verification)과 확인(Validation)의 정의

### 문제
개발자 혹은 시험자의 시각으로 소프트웨어가 명세화된 기능을 올바르게 수행하는지 알아보는 과정으로 소프트웨어 개발 과정을 테스트하는 것은 ( ① ) (이)라고 하고, 사용자 시각으로 올바른 소프트웨어가 개발되었는지 입증하는 과정으로 소프트웨어 결과를 테스트하는 것은 ( ② ) (이)라고 한다.

### 정답 및 풀이 결과
* **작성한 답**: `확인` (①), `검증` (②) $\rightarrow$ **순서 반대로 오답** (알파/베타 테스트에서 정정 질의 포함)
* **모범 답안**:
  * **①: 검증 (Verification)**
  * **②: 확인 (Validation)**
* **결과**: **오답 정정 및 완벽 분석 완료**

### 핵심 해설
* **검증 (Verification) - "Are we building the product right?"**:
  * **개발자(Developer)** 혹은 시험자의 시각에서 접근합니다.
  * 소프트웨어가 명세서(Specification)에 적힌 설계대로 오류 없이 **올바르게 구현되고 있는지** 개발 과정(Process)을 테스트합니다.
* **확인 (Validation) - "Are we building the right product?"**:
  * **사용자(Customer)** 혹은 고객의 시각에서 접근합니다.
  * 최종 만들어진 결과물(Product)이 사용자가 요구했던 본래의 목적과 부합하여 **진짜 올바른 소프트웨어가 맞는지** 입증합니다.
* **알파 / 베타 테스트와의 구분**:
  * 알파/베타 테스트는 확인(Validation) 테스트 범주에 속하는 **인수 테스트(Acceptance Test)**의 구체적인 하위 기법입니다.
  * 알파 테스트는 통제된 환경(개발실 내부)에서 사용자와 개발자가 함께 수행하는 테스트이며, 베타 테스트는 통제되지 않은 환경(실제 운영 환경)에서 일반 사용자가 수행하는 테스트입니다.

## 26. SQL - 집계 함수(SUM)와 NULL 값의 연산 처리 규칙

### 문제
다음 [점수] 테이블에 대해 쿼리를 수행한 결과는 [결과] 테이블과 같다. ①, ②, ③에 들어갈 값을 쓰시오.

[점수] 테이블
| 이름 | DB | 프로그래밍 | 알고리즘 |
| :--- | :--- | :--- | :--- |
| 이완용 | 20 | NULL | 50 |
| 송병준 | NULL | 30 | 0 |
| 민영휘 | NULL | 10 | 20 |

[쿼리]
```sql
SELECT SUM(DB), SUM(프로그래밍), SUM(알고리즘) FROM 점수;
```

[결과]
| SUM(DB) | SUM(프로그래밍) | SUM(알고리즘) |
| :--- | :--- | :--- |
| ① | ② | ③ |

### 정답 및 풀이 결과
* **작성한 답**: `①: 20`, `②: 40`, `③: 70` (SUM 연산 시 NULL은 제외/무시된다는 규칙을 정확히 짚음)
* **모범 답안**:
  * **①: 20**
  * **②: 40**
  * **③: 70**
* **결과**: **정답 (100% 매칭 및 분석 성공)**

### 핵심 해설
* **SQL 집계 함수와 NULL 처리 원칙**:
  * SQL 표준 집계 함수(`SUM`, `AVG`, `COUNT`, `MAX`, `MIN`)는 컬럼 내에 **`NULL` 값이 존재하는 행(Row)을 연산 대상에서 자동으로 제외(무시)**합니다.
  * 만약 사칙연산(`+`, `-`, `*`, `/`) 등 일반적인 연산이라면 `값 + NULL`은 `NULL`이 되지만, 집계 함수는 `NULL`을 0 취급하는 것이 아니라 **아예 배제하고 존재하는 수치 데이터만 합산**합니다.
* **디버깅 세부 계산**:
  * **① `SUM(DB)`**: `20 + NULL + NULL` $\rightarrow$ NULL 제외 $\rightarrow$ **`20`**
  * **② `SUM(프로그래밍)`**: `NULL + 30 + 10` $\rightarrow$ NULL 제외 $\rightarrow$ `30 + 10` = **`40`**
  * **③ `SUM(알고리즘)`**: `50 + 0 + 20` $\rightarrow$ **`70`** (`0`은 유효한 정수이므로 정상 합산됩니다.)

## 27. C언어 문자열 - 포인터 offset 연산 및 strcat 문자열 결합

### 문제
아래 C 코드를 실행했을 때 출력되는 결과를 쓰시오. (단, 2라인의 `ffinclude`는 `#include`로, 6라인의 `三`은 `=`으로, 8라인의 `—`는 `-`로 간주하여 풀이한다.)
```c
#include <stdio.h>
#include <string.h>
int main(int argc, char *argv[]){
    char str1[11]="ABCDE";
    char str2[6]="12345";
    char* p1 = str1+1;
    char* p2 = str2+3;
    str1[1]=p2[-2];
    str2[3]=p1[1];
    strcat(str1, str2);
    printf("%s", p1+2);
    return 0;
}
```

### 정답 및 풀이 결과
* **작성한 답**: `DE123C5` (str1[1]을 4로 오산한 지점이 있었으나, printf 출력 영역이 오산 지점을 비껴가게 됨을 인지하고 정확한 최종 결과 출력)
* **모범 답안**: **`DE123C5`**
* **결과**: **정답 (100% 매칭 및 최종 결과 완벽 도출)**

### 핵심 해설
* **포인터 지정 위치 상세 분석**:
  * `str1` = `"ABCDE"`, `str2` = `"12345"`
  * `p1 = str1 + 1` $\rightarrow$ `str1[1]`인 `'B'`의 주소 지칭.
  * `p2 = str2 + 3` $\rightarrow$ `str2[3]`인 `'4'`의 주소 지칭.
* **대입 연산 세부 계산**:
  * `str1[1] = p2[-2]`
    * `p2[-2]`는 `*(p2 - 2)` $\rightarrow$ `*(str2 + 3 - 2)` $\rightarrow$ `str2[1]`에 해당하는 문자 **`'2'`**를 가리킵니다.
    * 따라서 `str1[1]`은 `'B'`에서 **`'2'`**로 변경되어 `str1`은 `"A2CDE"`가 됩니다. (※ 작성한 답에서 '4'로 짚은 실수가 있었으나, 아래 출력 범위 덕분에 정답에 영향이 없었습니다.)
  * `str2[3] = p1[1]`
    * `p1[1]`은 `*(p1 + 1)` $\rightarrow$ `*(str1 + 1 + 1)` $\rightarrow$ `str1[2]`에 해당하는 문자 **`'C'`**를 가리킵니다.
    * 따라서 `str2[3]`은 `'4'`에서 **`'C'`**로 변경되어 `str2`는 `"123C5"`가 됩니다.
* **strcat 결합**:
  * `strcat(str1, str2)` $\rightarrow$ `"A2CDE"` 뒤에 `"123C5"`를 붙여 `str1`은 최종 **`"A2CDE123C5"`**가 됩니다.
* **출력 주소 연산**:
  * `printf("%s", p1 + 2)`
    * `p1`은 `str1 + 1`이므로, `p1 + 2`는 `str1 + 1 + 2` $\rightarrow$ **`str1 + 3`** 주소를 의미합니다.
    * `str1`의 인덱스 3에 위치한 문자는 네 번째 글자인 **`'D'`**입니다.
    * 따라서 `str1 + 3` 주소부터 널 문자(`\0`)를 만날 때까지인 **`DE123C5`**가 최종 화면에 출력됩니다.

## 28. 파이썬 - while 루프 및 산술/대입 연산자 복합 디버깅

### 문제
아래 파이썬 코드를 실행했을 때 출력되는 결과를 쓰시오. (단, 3라인의 `7：`은 `7:`으로 간주하여 풀이한다.)
```python
i = 0
sum = 0
while i < 7:
    i = i + 1
    if i % 3 == 0:
        sum -= i
    elif i % 3 == 1:
        sum += i
    else:
        sum *= i

print(sum)
```

### 정답 및 풀이 결과
* **작성한 답**: `16` (루프 1회전부터 7회전까지의 변수 i와 sum의 변화 과정을 표 형식으로 완벽 추적)
* **모범 답안**: **`16`**
* **결과**: **정답 (100% 매칭 및 디버깅 정확도 100%)**

### 핵심 해설
* **코드 실행 과정 상세 디버깅 (Trace Table)**:
  * 초기 상태: `i = 0`, `sum = 0`

  | 회전 수 | `i < 7` 판별 | `i` 증가 (`i = i + 1`) | `i % 3` 연산 | 조건 분기 및 `sum` 대입 연산 | 최종 `sum` 값 |
  | :--- | :--- | :--- | :--- | :--- | :--- |
  | **1회전** | `0 < 7` (참) | **`1`** | `1` | `sum += 1` $\rightarrow$ `0 + 1` | **`1`** |
  | **2회전** | `1 < 7` (참) | **`2`** | `2` | `sum *= 2` $\rightarrow$ `1 * 2` | **`2`** |
  | **3회전** | `2 < 7` (참) | **`3`** | `0` | `sum -= 3` $\rightarrow$ `2 - 3` | **`-1`** |
  | **4회전** | `3 < 7` (참) | **`4`** | `1` | `sum += 4` $\rightarrow$ `-1 + 4` | **`3`** |
  | **5회전** | `4 < 7` (참) | **`5`** | `2` | `sum *= 5` $\rightarrow$ `3 * 5` | **`15`** |
  | **6회전** | `5 < 7` (참) | **`6`** | `0` | `sum -= 6` $\rightarrow$ `15 - 6` | **`9`** |
  | **7회전** | `6 < 7` (참) | **`7`** | `1` | `sum += 7` $\rightarrow$ `9 + 7` | **`16`** |
  | **종료** | `7 < 7` (거짓) | - | - | 루프 탈출 | **`16`** |

  * 루프를 빠져나온 후 `print(sum)`에 의해 최종값인 **`16`**이 출력됩니다.

## 29. Java 객체지향 - 추상 클래스(Abstract Class)와 추상 메서드(Abstract Method) 선언 지시어

### 문제
아래 자바 코드에서 빈칸 01라인과 02라인에 공통으로 들어갈 알맞은 예약어(지시어)를 쓰시오.
```java
01 _________ class Animal{
02 _________ void show( );
03 }
04 class Dog extends Animal{
05 void show(){
06 System.out.print("dog");
07 }
08 }
09 public class Soojebi {
10 public static void main(String[ ] args) {
11 Animal d=new Dog( );
12 d.show( );
13 }
14 }
```
* **출력 결과**: `dog`

### 정답 및 풀이 결과
* **작성한 답**: `abstract` (추상 클래스와 추상 메서드의 문법적 구조에 대한 왜 그런지에 대한 상세 이론 설명 요청 포함)
* **모범 답안**: **`abstract`**
* **결과**: **정답 (이론 완벽 복습 완료)**

### 핵심 해설
* **빈칸에 `abstract`가 들어가는 결정적 이유**:
  1. **몸체(Body)가 없는 메서드 선언**:
     * 2라인을 보면 `void show();` 처럼 뒤에 중괄호 `{ }` 구현부 없이 바로 세미콜론 `;`으로 마감되어 있습니다.
     * 이처럼 시그니처만 존재하고 실제 구현이 없는 메서드를 **추상 메서드(Abstract Method)**라고 하며, 선언할 때는 명시적으로 수식어 **`abstract`**를 붙여야 합니다.
  2. **추상 메서드를 포함한 클래스의 정의**:
     * 클래스 내부에 단 하나의 추상 메서드라도 존재한다면, 그 클래스는 온전한 인스턴스를 직접 생성(new)할 수 없는 미완성 설계도 상태가 됩니다.
     * 따라서 이 클래스 선언문인 1라인 앞에도 반드시 **`abstract`**를 붙여 **추상 클래스(Abstract Class)**로 선언해 주어야 자바 컴파일러가 에러를 내지 않습니다.
* **추상 클래스와 상속의 관계**:
  * 추상 클래스 `Animal`을 상속(extends)받은 일반 자식 클래스 `Dog`는 부모의 미완성 추상 메서드인 `show()`를 반드시 **오버라이딩(Overriding / 재정의)**하여 완성형 메서드(5~7라인)로 구현해야만 객체화가 가능해집니다.
  * 메인 메서드에서 다형성에 의해 부모 타입 변수 `d`에 자식 객체 `Dog`를 할당하고 `d.show()`를 실행하면, 동적 바인딩에 의해 자식의 완성된 `show()` 메서드가 호출되어 화면에 `dog`가 출력됩니다.

## 30. 소프트웨어 공학 - 요구사항 수집(도출) 기법 (롤플레잉 vs 시나리오)

### 문제
요구사항 수집 기법 중 현실에 일어나는 장면을 설정하고 여러 사람이 각자가 맡은 역할을 연기함으로써 요구사항을 분석하고 수집하는 방법은 무엇인지 쓰시오.

### 정답 및 풀이 결과
* **작성한 답**: `시나리오` $\rightarrow$ **오답** (동작 흐름의 본질적 차이에 대한 설명 요청)
* **모범 답안**: **`롤플레잉`** (또는 `역할 연기`)
* **결과**: **오답 정정 및 완벽 분석 완료**

### 핵심 해설
* **롤플레잉 (Role Playing / 역할 연기)**:
  * **본질**: **사람이 직접 몸을 움직여 연기(Acting)하는 것**입니다.
  * **동작 방식**: 요구사항 분석가와 사용자가 실제 현장의 특정 상황을 극화(장면 설정)한 뒤, 각자 배역을 맡아 연극을 하듯이 연기하며 가상의 업무 프로세스를 연출해 봅니다. 이 과정에서 직접 체험하며 미처 생각하지 못했던 예외 상황이나 시스템 요구사항을 몸소 파악하게 됩니다.
  * **지문 단서**: "여러 사람이 각자가 맡은 **역할을 연기함으로써**" $\rightarrow$ '연기(Playing)'와 '역할(Role)'이 핵심 힌트입니다.
* **시나리오 (Scenario)와의 차이점**:
  * **본질**: **가상의 상황을 텍스트나 그림 문서(스토리보드)로 시뮬레이션하는 것**입니다.
  * **동작 방식**: "만약 사용자가 이런 조작을 하면 시스템이 어떻게 반응할 것인가?"를 명문화된 시나리오 문서나 그림으로 흐름을 작성하고 순차적으로 검토하는 비언어적/비체험적 분석 기법입니다. (연기하는 과정이 없습니다.)

### 💡 ★강사 특별 조언: 요구사항 도출(수집) 기법 핵심 키워드 총정리★

## 31. SQL - 다중 컬럼 GROUP BY 및 집계 함수를 이용한 통계 쿼리

### 문제
다음 [직원] 테이블에 대해 부서, 직책별 급여의 합계를 구하는 SQL 쿼리를 작성하시오. (결과 테이블의 급여합계 컬럼명 맵핑을 고려한다.)

[직원] 테이블
| 이름 | 부서 | 직책 | 급여 |
| :--- | :--- | :--- | :--- |
| 워싱턴 | 영업 | 차장 | 600 |
| 링컨 | 영업 | 차장 | 500 |
| 루스벨트 | 영업 | 과장 | 400 |
| 트루먼 | 전산 | 과장 | 200 |
| 아이젠하워 | 전산 | 사원 | 500 |
| 케네디 | 전산 | 사원 | 100 |

[결과] 테이블
| 부서 | 직책 | 급여합계 |
| :--- | :--- | :--- |
| 영업 | 차장 | 1100 |
| 영업 | 과장 | 400 |
| 전산 | 과장 | 200 |
| 전산 | 사원 | 600 |

### 정답 및 풀이 결과
* **작성한 답**: `SELECT 부서, 직책, SUM(급여) AS 급여합계 FROM 직원 GROUP BY 부서, 직책;` (대소문자 무관 및 괄호 마감 무시 처리 검토)
* **모범 답안**:
  * **`SELECT 부서, 직책, SUM(급여) AS 급여합계 FROM 직원 GROUP BY 부서, 직책;`** (또는 `AS` 생략 및 소문자 기입 가능)
* **결과**: **정답 (100% 매칭 및 SQL 문법 완벽 작성)**

### 핵심 해설
* **다중 컬럼 GROUP BY 연산**:
  * "부서, 직책별" 급여합계를 구해야 하므로, 그룹화 기준을 하나가 아닌 둘(`부서`, `직책`)로 설정해야 합니다.
  * 따라서 `GROUP BY 부서, 직책` 구문을 통해 두 필드가 모두 일치하는 행들끼리 그룹으로 묶이게 됩니다.
* **집계 함수 및 별칭(Alias) 지정**:
  * 부서별/직책별 급여의 합계를 구하기 위해 `SUM(급여)` 함수를 사용합니다.
  * 결과 테이블의 세 번째 컬럼명이 **`급여합계`**로 출력되도록 `AS 급여합계` 별칭을 정확하게 붙여주었습니다. (`AS` 키워드는 생략해도 무방합니다.)
* **SQL 쿼리 작성 시 팁**:
  * 실기 시험 채점 시 SQL 명령어의 대소문자는 구분하지 않습니다.
  * 쿼리 끝에 붙는 세미콜론 `;`은 문장의 마감을 의미하며, 채점 기준상 생략해도 정답 인정되는 경우가 대부분이나 표준 문법 준수를 위해 습관적으로 적어주는 것이 가장 좋습니다.

## 32. Java - 배열 내 최대값(Max) 및 최소값(Min) 탐색 알고리즘

### 문제
아래 자바 코드에서 빈칸 ①과 ②에 들어갈 알맞은 코드를 쓰시오. (단, 6라인의 중복 선언 `int i, max, min;`은 무시하고, 10라인의 `三`은 `=`으로 간주하여 풀이한다.)
```java
public class Soojebi {
    public static void main(String[ ] args){
        int[ ] arr={10, 30, 50, 70, 90};
        int i, max, min;
        max=min=arr[0];
        for(i=0;i<5;i++){
            if( ① > max)
                max=arr[i];
            if( ② < min)
                min=arr[i];
        }
        System.out.printf("%d %d\n", max, min);
    }
}
```
* **출력 결과**: `90 10`

### 정답 및 풀이 결과
* **작성한 답**: `①: arr[i]`, `②: arr[i]` (배열 순회 기반 최소/최대값 도출 알고리즘 개념 정확히 대입)
* **모범 답안**:
  * **①: arr[i]** (또는 `arr[i]`의 공백 허용)
  * **②: arr[i]**
* **결과**: **정답 (100% 매칭 및 알고리즘 분석 성공)**

### 핵심 해설
* **배열의 최대값 / 최소값 찾기 기본 동작 메커니즘**:
  1. **초기화**: 5라인 `max = min = arr[0]`에 의해 배열의 첫 번째 원소(`10`)를 임시 최대값과 최소값으로 각각 설정해 둡니다.
  2. **루프 순회**: `for` 루프를 사용해 배열의 모든 원소(`i = 0`부터 `4`까지)를 차례대로 순회합니다.
  3. **최대값 갱신**:
     * 순회 중 만난 현재 원소인 **`arr[i]`**가 현재까지 기록된 최대값 `max`보다 크다면(`arr[i] > max`), 그 원소를 새로운 최대값으로 등록합니다 (`max = arr[i]`).
     * 따라서 ①에는 현재 배열의 요소인 **`arr[i]`**가 들어갑니다.
  4. **최소값 갱신**:
     * 순회 중 만난 현재 원소인 **`arr[i]`**가 현재까지 기록된 최소값 `min`보다 작다면(`arr[i] < min`), 그 원소를 새로운 최소값으로 등록합니다 (`min = arr[i]`).
     * 따라서 ②에도 동일하게 현재 배열의 요소인 **`arr[i]`**가 들어갑니다.
* **디버깅 결과 검증**:
  * 루프가 완료되면 `max`에는 가장 큰 값인 **`90`**이 남고, `min`에는 가장 작은 값인 **`10`**이 유지되어 출력 포맷 `%d %d`에 맞추어 `90 10`이 출력됩니다.

## 33. 통합 구현 - EAI(Enterprise Application Integration)의 4대 구축 유형

### 문제
다음이 설명하는 EAI 구축 유형은 무엇인지 쓰시오.
* 단일한 접점의 허브 시스템을 통하여 데이터를 전송하는 중앙 집중식 방식
* 허브 장애 시 전체 장애 발생

### 정답 및 풀이 결과
* **작성한 답**: 미작성 (강사 풀이용 질문 및 상세 개념 정리 요청)
* **모범 답안**: **`허브 앤 스포크 (Hub & Spoke)`**
* **결과**: **이론 완벽 정리 완료**

### 핵심 해설
* **EAI (기업 애플리케이션 통합)**:
  * 기업 내부에서 운영되는 서로 다른 기종의 애플리케이션 및 플랫폼 간의 데이터를 상호 연동하고 통합하는 기술/솔루션입니다.
* **EAI 4대 구축 유형 특징 분석**:
  1. **Point-to-Point (점대점 / P2P)**:
     * 애플리케이션을 **1대1로 직접 연결**하는 방식입니다.
     * 가장 기초적인 방식으로 중간 미들웨어 없이 구축되지만, 시스템이 늘어날수록 연결선이 폭발적으로 증가하여 유지보수가 불가능해집니다. (연결선 개수: $N(N-1)/2$)
  2. **Hub & Spoke (허브 앤 스포크)**:
     * 중앙에 유일한 **허브(Hub) 시스템**을 두고, 주변의 어플리케이션들(**Spoke**)은 이 허브를 통해서만 통신하는 중앙 집중형 방식입니다.
     * **장점**: 모든 데이터 흐름이 통제되므로 연결 관리가 대단히 용이합니다.
     * **단점**: 중앙 허브에 병목 현상이 발생할 수 있으며, 허브가 고장 나면 전체 시스템이 마비되는 **단일 장애점(Single Point of Failure)** 문제가 심각합니다. (본 문제의 지문 내용입니다.)
  3. **Message Bus (메시지 버스 / ESB)**:
     * 애플리케이션들 사이에 공통 통로인 **메시지 버스(Message Bus)**를 배치하고, 어플리케이션은 어댑터를 통해 버스에 데이터를 던져 미들웨어 방식으로 중계 및 전송하는 방식입니다.
     * 확장성이 매우 뛰어나고 느슨한 결합(Decoupling)을 지원하여 대용량 처리에 유리합니다.
  4. **Hybrid (하이브리드)**:
     * **Hub & Spoke** 방식과 **Message Bus** 방식을 결합한 혼합형 방식입니다.
     * 예컨대 한 그룹(부서) 내에서는 통제가 편한 Hub & Spoke를 사용하고, 각 그룹 간의 통신은 유연한 Message Bus 방식을 취해 데이터 병목을 최소화합니다.

## 34. 데이터베이스 - 데이터 중복으로 인한 이상 현상(Anomaly)의 정의 및 종류

### 문제
데이터를 데이터베이스에 저장할 때 불필요하게 중복되어 릴레이션 조작 시 예기치 못한 곤란한 현상이 발생하는 문제점은 무엇인지 쓰시오.

### 정답 및 풀이 결과
* **작성한 답**: `이상현상` (이상 현상의 3가지 종류 및 구체적인 서술형 키워드 매칭 분석)
* **모범 답안**: **`이상 현상 (Anomaly)`** (또는 `이상`)
* **결과**: **정답 (100% 매칭 및 개념 복습 성공)**

### 핵심 해설
* **이상 현상 (Anomaly)**:
  * 릴레이션(테이블) 설계 시 속성들 간의 함수 종속성을 제대로 고려하지 않고 정규화를 생략하여, 불필요하게 데이터가 중복 보관될 때 데이터의 삽입, 삭제, 갱신 조작 중에 발생하는 논리적 오류 상태입니다.
* **이상 현상의 3가지 종류 (암기 필수: "삽 - 삭 - 갱")**:
  1. **삽입 이상 (Insertion Anomaly)**:
     * **개념**: 데이터를 테이블에 삽입할 때 원하지 않는 불필요한 데이터까지 억지로 함께 삽입해야 하거나, 특정 값이 없어 삽입 자체가 불가능해지는 현상입니다.
     * **예시**: 수강 테이블에 신입생 학번을 넣으려는데 수강하는 과목코드(기본키 일부)가 없어서 학생 등록이 안 되는 경우.
  2. **삭제 이상 (Deletion Anomaly)**:
     * **개념**: 테이블에서 한 튜플(행)을 삭제할 때, 그 튜플에 묶여 있던 무관하고 유용한 정보까지 연쇄적으로 함께 영구 유실되는 현상입니다.
     * **예시**: 수강을 포기한 학생의 학번을 지웠더니 그 학생의 이름, 주소, 연락처 등 고유 정보까지 통째로 날아가는 경우.
  3. **갱신 이상 (Update Anomaly)**:
     * **개념**: 테이블의 특정 튜플 값을 갱신(수정)할 때, 데이터의 중복성 때문에 일부 튜플만 갱신되고 다른 중복된 튜플은 누락되어 데이터의 불일치(모순)가 발생하는 현상입니다.
     * **예시**: 한 학생의 주소가 이사로 바뀌었는데 중복 저장된 여러 수강 내역 중 일부 행만 주소가 갱신되어, 한 학생에 대해 주소가 두 개로 상충되어 조회되는 경우.

## 35. C언어 제어문 - 반복문을 이용한 거듭제곱(Power) 계산

### 문제
아래 C 코드를 실행했을 때 출력되는 결과를 쓰시오. (단, 1라인의 `include`는 `#include`로, 10라인의 `Soojebi(2,10》`은 `Soojebi(2, 10)`으로 간주하여 풀이한다.)
```c
#include <stdio.h>
int Soojebi(int base, int exp){
    int i, result=1;
    for(i=0; i<exp; i++)
        result *= base;
    return result;
}

int main() {
    printf("%d", Soojebi(2, 10));
    return 0;
}
```

### 정답 및 풀이 결과
* **작성한 답**: `1024` (2의 10승 거듭제곱 연산 원리 정확히 연산)
* **모범 답안**: **`1024`**
* **결과**: **정답 (100% 매칭 및 계산 성공)**

### 핵심 해설
* **거듭제곱 사용자 정의 함수 분석**:
  * `Soojebi(2, 10)`이 호출되면, 밑(`base`)은 `2`가 되고 지수(`exp`)는 `10`이 됩니다.
  * `result` 변수가 `1`로 초기화된 상태에서, `for` 루프가 `0`부터 `9`까지 총 `10`회 동작하며 `result = result * base` 연산을 수행합니다.
  * 즉, 1에다가 2를 10번 연속해서 곱하므로 수학적으로 $2^{10}$ 연산을 수행하게 됩니다.
* **디버깅 계산**:
  * $2^1 = 2$
  * $2^2 = 4$
  * $2^3 = 8$
  * ...
  * $2^9 = 512$
  * $2^{10} = 1024$
  * 함수는 최종 누적 곱값인 **`1024`**를 반환하여 출력하므로 정답은 `1024`입니다.

## 36. 네트워크 - FLSM(고정 길이 서브넷 마스크) 서브네팅 및 IP Subnet-Zero 연산

### 문제
`200.1.1.0/24` 네트워크를 FLSM 방식을 이용하여 10개의 Subnet으로 나누고 IP Subnet-Zero를 적용했다. 이때 서브네팅된 네트워크 중 10번째 네트워크의 Broadcast IP 주소는 무엇인지 쓰시오.

### 정답 및 풀이 결과
* **작성한 답**: 미작성 (IP 옥텟 이진수 변환까지 시도했으나 10번째 범위 도출 실패로 질의 포함)
* **모범 답안**: **`200.1.1.159`**
* **결과**: **이론 완벽 정리 완료**

### 핵심 해설
* **서브넷팅 단계별 정밀 계산**:
  1. **빌려올 서브넷 비트($S$) 계산**:
     * 10개의 서브넷 대역을 만들기 위해 필요한 서브넷 비트 수를 구합니다.
     * $2^S \ge 10$을 만족하는 최소 정수 $S = 4$ 입니다. ($2^3 = 8 < 10$ 이므로 3비트는 부족하고, $2^4 = 16 \ge 10$ 이므로 4비트를 빌려야 합니다.)
  2. **서브넷 마스크 및 블록 크기 계산**:
     * 호스트 영역(마지막 8비트) 중 상위 4비트를 서브넷으로 할당하므로 서브넷 마스크는 `/28` (`255.255.255.240`)이 됩니다.
     * 남은 호스트 영역 비트는 $8 - 4 = 4$비트이므로, 각 서브넷 대역의 크기(간격)는 $2^4 = \mathbf{16}$이 됩니다.
  3. **IP Subnet-Zero 규칙 적용**:
     * `IP Subnet-Zero`란 서브넷 비트가 모두 0인 첫 번째 서브넷 대역(0번째 서브넷)을 버리지 않고 **사용 가능 대역에 포함**시키는 표준 규칙입니다.
     * 따라서 각 서브넷 대역의 인덱스는 `0`부터 시작합니다.
       * 1번째 서브넷 시작 주소: $0 \times 16 = \mathbf{0}$ (`200.1.1.0`)
       * 2번째 서브넷 시작 주소: $1 \times 16 = \mathbf{16}$ (`200.1.1.16`)
       * 3번째 서브넷 시작 주소: $2 \times 16 = \mathbf{32}$ (`200.1.1.32`)
       * ...
       * $N$번째 서브넷 시작 주소 공식: $(N - 1) \times 16$
  4. **10번째 서브넷의 주소 범위 도출**:
     * $N = 10$이므로 시작 주소는 $(10 - 1) \times 16 = \mathbf{144}$ 가 되어 네트워크 주소는 `200.1.1.144`가 됩니다.
     * 블록 크기가 16이므로 10번째 서브넷의 IP 대역 범위는 **`200.1.1.144 ~ 200.1.1.159`** 가 됩니다. (다음 11번째 대역은 160부터 시작)
  5. **브로드캐스트(Broadcast) 주소 결정**:
     * 브로드캐스트 주소는 해당 서브넷 범위의 **가장 마지막 주소**입니다.
     * 따라서 10번째 네트워크의 브로드캐스트 주소는 **`200.1.1.159`**가 됩니다.

     * **키워드**: **"HTTP/HTTPS(웹 애플리케이션 프로토콜) 전용"**, "SQL 인젝션, XSS 웹 공격 차단" (7레이어 통제)
  5. **가상 사설망 (VPN / Virtual Private Network)**:
     * **키워드**: **"공용망을 전용망(사설망)처럼 사용"**, "터널링(Tunneling)", "암호화 통신 채널 제공"

## 39. 운영체제 - 가상 기억장치 관리의 페이지 교체 알고리즘 (LRU, LFU, FIFO, OPT)

### 문제
다음은 페이지 교체 알고리즘에 대한 설명이다. 괄호 ( ) 안에 들어갈 용어를 쓰시오.
* ( ① ) 기법은 사용된 시간을 확인하여 가장 오랫동안 사용되지 않은 페이지를 선택하여 교체하는 기법으로 최근에 참조된 페이지는 앞으로도 참조될 가능성이 크고, 최근에 참조되지 않은 페이지는 앞으로도 참조되지 않을 가능성이 크다는 전제로 구현된 기법이다.
* ( ② ) 기법은 사용된 횟수를 확인하여 참조 횟수가 가장 적은 페이지를 선택하여 교체하는 기법으로 기억장치에 저장된 페이지 중에서 사용한 횟수가 가장 적은 페이지를 교체하는 기법이다.

### 정답 및 풀이 결과
* **작성한 답**: `①: LRU`, `②: LFU` (가상메모리 교체 정책의 시간 기준과 횟수 기준의 차이 분석 포함)
* **모범 답안**:
  * **①: LRU (Least Recently Used)**
  * **②: LFU (Least Frequently Used)**
* **결과**: **정답 (100% 매칭 및 개념 정리 성공)**

### 핵심 해설
* **페이지 교체 알고리즘**:
  * 가상 기억장치(Virtual Memory)를 사용할 때, 주기억장치의 모든 페이지 프레임이 가득 찬 상태에서 새로운 페이지를 올리려면 기존 페이지 중 하나를 선택해 보조기억장치로 내려보내야(Swap-out) 합니다. 이때 어떤 페이지를 선택할지 결정하는 알고리즘입니다.
* **주요 5대 알고리즘 핵심 비교**:
  1. **LRU (Least Recently Used / 최근 최소 사용)**:
     * **핵심 기준**: **시간(Time / Recency)**
     * **동작**: 최근 가장 오랫동안 참조(사용)되지 않은 페이지를 교체 대상으로 선정합니다. (지문의 ①번 설명)
     * **사상**: 국부성(Locality)의 법칙에 기반하여, 최근에 안 쓰인 것은 앞으로도 안 쓰일 확률이 높다는 가정을 취합니다.
  2. **LFU (Least Frequently Used / 최소 빈도 사용)**:
     * **핵심 기준**: **참조 횟수(Frequency / Count)**
     * **동작**: 과거에 누적된 **사용 횟수**가 가장 적은 페이지를 교체 대상으로 선정합니다. (지문의 ②번 설명)
  3. **FIFO (First-In, First-Out / 선입선출)**:
     * **핵심 기준**: **주기억장치 적재 시간(First-In)**
     * **동작**: 주기억장치에 가장 먼저 들어와서 가장 오랫동안 머물러 있던 페이지를 무조건 먼저 교체합니다.
  4. **OPT (Optimal / 최적 교체)**:
     * **핵심 기준**: **미래 예측**
     * **동작**: 앞으로 **가장 오랫동안 사용되지 않을** 페이지를 교체합니다.
     * **특징**: 미래를 알아야 하므로 이론적 최적성만 제공할 뿐 실제 하드웨어로 구현하는 것은 불가능합니다.
  5. **NUR (Not Used Recently / 최근 미사용)**:
     * **핵심 기준**: **참조 비트(Reference Bit) & 변형 비트(Modified Bit)**
     * **동작**: 최근 사용 여부 비트를 2개 사용하여 오버헤드를 줄이며 LRU를 간소하게 구현한 실무적 기법입니다.

## 40. C언어 제어문 - 중첩 for 루프와 switch-case 조건절의 복합 연산

### 문제
아래 C 코드를 실행했을 때 출력되는 결과를 쓰시오. (단, 5라인의 `j느`는 `j=1`로 간주하여 풀이한다.)
```c
#include <stdio.h>
int main(){
    int i, j;
    for(i=1; i<3; i++){
        for(j=1; j<=i; j++){
            switch((i+j)%2){
                case 0:
                    printf("%d", i+j);
                    break;
                case 1:
                    printf("%d", i-j);
            }
        }
    }
    return 0;
}
```

### 정답 및 풀이 결과
* **작성한 답**: `214` (루프 변수 i, j의 조합별 분기 조건 및 출력문 연산 과정을 정밀히 추적)
* **모범 답안**: **`214`**
* **결과**: **정답 (100% 매칭 및 계산 성공)**

### 핵심 해설
* **중첩 for 루프의 범위 분석**:
  * 바깥 루프 `i`: `1`부터 `2`까지 증가하며 총 2회전 수행합니다 (`i < 3`).
  * 안쪽 루프 `j`: `1`부터 현재 `i`값 이하까지 증가합니다 (`j <= i`).
* **디버깅 실행 순서 상세 추적**:
  1. **`i = 1` 단계**:
     * `j`의 범위는 `1`부터 `1`까지이므로 `j = 1` 1회만 동작합니다.
     * **`i = 1, j = 1`**:
       * switch 조건식: `(i+j)%2` $\rightarrow$ `(1+1)%2` = **`0`**
       * 분기: `case 0` 실행 $\rightarrow$ `printf("%d", i+j)` $\rightarrow$ `1+1` = **`2`** 출력 후 `break;`로 switch문 탈출.
  2. **`i = 2` 단계**:
     * `j`의 범위는 `1`부터 `2`까지이므로 `j = 1, 2` 총 2회 동작합니다.
     * **`i = 2, j = 1`**:
       * switch 조건식: `(i+j)%2` $\rightarrow$ `(2+1)%2` = **`1`**
       * 분기: `case 1` 실행 $\rightarrow$ `printf("%d", i-j)` $\rightarrow$ `2-1` = **`1`** 출력.
     * **`i = 2, j = 2`**:
       * switch 조건식: `(i+j)%2` $\rightarrow$ `(2+2)%2` = **`0`**
       * 분기: `case 0` 실행 $\rightarrow$ `printf("%d", i+j)` $\rightarrow$ `2+2` = **`4`** 출력 후 `break;`로 switch문 탈출.
* **최종 출력**:
  * 개행(줄바꿈)문자 없이 숫자가 순서대로 출력되므로 최종 정답은 **`214`**입니다.

## 41. 데이터베이스 - 정규화(Normalization) 단계별 특징 및 다치 종속(MVD) 제거

### 문제
다음 설명 중 괄호 ( ) 안에 들어갈 알맞은 용어를 쓰시오.

[테이블 예시]
* 직원 자격증 언어 테이블: `(개발자, 자격증, 언어)`
* 테이블 분리 결과: `(개발자, 자격증)` 및 `(개발자, 언어)`

* 〈개발자〉마다〈자격증〉값들이 여러 개 존재하고, 특정〈개발자〉마다〈언어〉값들이 여러 개 존재하는 경우 다치 종속 관계라고 한다.
* 〈개발자〉별로 여러〈자격증〉값을 가지고 있고,〈개발자〉별로 여러〈언어〉값을 가지고 있으므로〈개발자, 자격증〉,〈개발자, 언어〉 테이블로 분리하여 관리하면 다치 종속 관계를 제거하기 때문에 ( ) 을/를 만족한다.

### 정답 및 풀이 결과
* **작성한 답**: `제 5 정규형` $\rightarrow$ **오답** (4정규형과 5정규형 매칭 기법 혼동으로 정정 가이드 요청 포함)
* **모범 답안**: **`제4정규형 (4NF)`**
* **결과**: **오답 정정 및 완벽 분석 완료**

### 핵심 해설
* **오답 분석 (4NF vs 5NF)**:
  * 다치 종속(MVD)을 제거하여 만족하는 정규형은 **제4정규형 (4NF)**입니다.
  * 조인 종속(JD)을 제거하여 만족하는 정규형은 **제5정규형 (5NF)**입니다.
  * 작성자님께서 정규화 기법 암기 단어 순서와 정규형 번호 매칭을 헷갈리셔서 5정규형으로 기입하는 오답을 냈습니다.
* **정규화 단계의 정확한 도식 (치트키: "도 - 부 - 이 - 결 - 다 - 조")**:
  * **도**: 도메인 원자값 $\rightarrow$ **제1정규형 (1NF)**
  * **부**: 부분 함수 종속 제거 $\rightarrow$ **제2정규형 (2NF)**
  * **이**: 이행적 함수 종속 제거 $\rightarrow$ **제3정규형 (3NF)**
  * **결**: 결정자 후보키 아닌 것 제거 $\rightarrow$ **BCNF (보이스-코드 정규형)**
  * **다**: **다치 종속 (Multi-valued Dependency) 제거** $\rightarrow$ **제4정규형 (4NF)** (본 문제의 정답)
  * **조**: **조인 종속 (Join Dependency) 제거** $\rightarrow$ **제5정규형 (5NF)**

## 42. 데이터베이스 - 정규화 기초: 제2정규형(2NF)과 완전 함수 종속(Full Functional Dependency)

### 문제
다음 설명의 괄호 ( ① ) 안에 들어갈 가장 알맞은 데이터베이스 핵심 용어를 쓰시오.
* 제2정규형 (2NF)은 제1정규형(1NF)을 만족하면서, 기본키가 아닌 모든 속성이 기본키에 대해 ( ① ) 종속을 만족하도록 하는 정규화 단계이다.
* 즉, 복합 기본키의 일부분에 종속되는 '부분 함수 종속성'을 제거하여, 기본키 전체에 의해서만 결정되도록 릴레이션을 분리하는 과정이다.

### 정답 및 풀이 결과
* **작성한 답**: `제 2 정규형` $\rightarrow$ **오답** (문제 지문 오독 및 빈칸 요구 속성 불일치 실수로 정정 가이드 완료)
* **모범 답안**: **`완전 함수`** (또는 `완전 함수 종속`)
* **결과**: **오답 정정 및 지문 해독 역량 보완 완료**

### 핵심 해설
* **오답 분석 (지문 오독 해방)**:
  * 지문 첫머리에 대놓고 **"제2정규형(2NF)은..."**이라고 전제되어 있었으므로 빈칸에 들어갈 답은 정규형의 이름이 아닌, 제2정규형의 충족 조건인 **`완전 함수 (종속)`**이 들어갔어야 합니다.
  * 실제 시험장에서도 이처럼 질문의 요구 대상을 끝까지 꼼꼼히 확인하지 않아 허망하게 점수를 잃는 경우가 대단히 많습니다. 문제를 정확하게 정독하는 습관이 중요합니다.
* **부분 함수 종속 vs 완전 함수 종속 완벽 비교**:
  * **부분 함수 종속 (Partial Functional Dependency)**:
    * 기본키가 두 개 이상으로 구성된 복합키 일 때, 기본키의 **일부 속성**에 의해서만 다른 일반 속성이 결정되는 상태입니다.
  * **완전 함수 종속 (Full Functional Dependency)**:
    * 기본키가 여러 개일 때, 기본키 **전체 조합**에 의해서만 일반 속성이 결정되는 깨끗한 상태입니다.
  * **예시 상황**:
    * 수강 테이블 `(학번, 과목코드, 성적, 지도교수)`에서 기본키는 `(학번, 과목코드)` 복합키입니다.
    * 이 중에서 `성적`은 학생(`학번`)과 들은 과목(`과목코드`) 두 가지가 다 합쳐져야 결정되므로 **완전 함수 종속** 상태입니다.
    * 하지만 `지도교수`는 과목코드와 상관없이 학생(`학번`) 한 명에 의해 바로 결정되므로 복합키의 일부인 `학번`에만 엮여 있는 **부분 함수 종속** 상태입니다.
    * 이 `지도교수` 속성을 별도 테이블로 떼어내어 **모든 필드가 기본키 전체에만 완전히 묶이도록** 만드는 단계가 바로 **제2정규형**입니다.
