## 48. C언어 - 스택을 이용한 후위 표기법(Postfix) 수식 연산 및 아스키코드 출력

### 문제
아래 C 코드를 실행했을 때 출력되는 결과를 쓰시오. (단, 9라인의 `case`는 `case '+'`로, 18라인의 `s[+十 p]데i]—’O’`는 `s[++p]=f[i]-'0'`으로, 21라인의 `s[이`는 `s[0]`으로 보정하여 풀이한다.)
```c
#include <stdio.h>
#include <string.h>
int main() {
    char f[6]="549+*";
    char s[6]="";
    int i, p=-1;
    for(i=0; i<strlen(f); i++){
        switch(f[i]){
            case '+':
                s[p-1] = s[p]+s[p-1];
                p--;
                break;
            case '*':
                s[p-1]=s[p]*s[p-1];
                p--;
                break;
            default:
                s[++p]=f[i]-'0';
        }
    }
    printf("%c", s[0]);
    return 0;
}
```

### 정답 및 풀이 결과
* **작성한 답**: `A` (수식 연산의 최종 정수값 65와 아스키코드 대문자 A 매핑 규칙 정확히 도출)
* **모범 답안**: **`A`**
* **결과**: **정답 (100% 매칭 및 계산 과정 완벽)**

### 핵심 해설
* **후위 표기법(Postfix Notation)의 스택 기반 연산 흐름**:
  * 후위 표기 수식 `5 4 9 + *`을 앞에서부터 차례대로 스택 `s`에 푸시하며 연산자를 만날 때마다 팝하여 계산합니다.
  1. **`'5'` 수신**: `default` 분기 $\rightarrow$ 스택 탑 `p`를 `0`으로 증가시킨 뒤 `s[0] = 5` 삽입. (`s = [5]`)
  2. **`'4'` 수신**: `default` 분기 $\rightarrow$ 스택 탑 `p`를 `1`로 증가시킨 뒤 `s[1] = 4` 삽입. (`s = [5, 4]`)
  3. **`'9'` 수신**: `default` 분기 $\rightarrow$ 스택 탑 `p`를 `2`로 증가시킨 뒤 `s[2] = 9` 삽입. (`s = [5, 4, 9]`)
  4. **`'+'` 수신**: `case '+'` 분기 $\rightarrow$ 스택의 탑 부근에 있는 두 수 `9`와 `4`를 더하여(`9 + 4 = 13`) 스택 `s[1]`에 덮어쓰고, 스택 탑 포인터 `p`를 `1`로 감소시킵니다. (`s = [5, 13]`)
  5. **`'*'` 수신**: `case '*'` 분기 $\rightarrow$ 스택의 탑 부근에 있는 두 수 `13`과 `5`를 곱하여(`13 * 5 = 65`) 스택 `s[0]`에 덮어쓰고, 스택 탑 포인터 `p`를 `0`으로 감소시킵니다. (`s = [65]`)
* **최종 문자 출력 서식 지정자 분석**:
  * 루프 완료 후 스택의 바닥 `s[0]`에 저장된 정수 값은 **`65`**입니다.
  * 출력 포맷이 **`%c` (문자형)**로 지정되어 있으므로, 십진수 `65`에 대응하는 아스키코드(ASCII Code) 문자를 출력해야 합니다.
  * 아스키코드 `65`는 알파벳 대문자 **`A`**이므로 최종 출력값은 **`A`**가 됩니다.

## 49. C언어 제어문 - 나누기/나머지 연산자를 이용한 각 자릿수 합산 알고리즘

### 문제
아래 C 코드의 빈칸 ①과 ②에 들어갈 알맞은 연산자를 쓰시오. (단, 3라인 끝의 `；` 등 OCR 오타는 무시하고 올바른 C 문법으로 간주하여 풀이한다.)
```c
#include <stdio.h>
int main(){
    int x, t=0;
    for(x=1209; x ① 0; x ② 10){
        t+=x%10;
    }
    printf("%d", t);
    return 0;
}
```
* **출력 결과**: `12`

### 정답 및 풀이 결과
* **작성한 답**: `①: !=` (또는 `>`), `②: /=` (지문 뒤의 `0`과 `10` 리터럴 위치를 고려하여 연산자만 정교하게 매핑 성공)
* **모범 답안**:
  * **①: !=** (또는 **`>`**)
  * **②: /=** (또는 **`= x /`**)
* **결과**: **정답 (100% 매칭 및 알고리즘 완벽 분석)**

### 핵심 해설
* **자릿수 분리 알고리즘 원리**:
  * 정수 $N$에서 1의 자릿수 숫자를 추출할 때는 **나머지 연산자(`% 10`)**를 사용합니다.
  * 1의 자리를 추출한 뒤 다음 자릿수(10의 자리)를 추출하기 위해 기존 수를 10으로 나눈 몫으로 갱신할 때는 **나누기 대입 연산자(`/= 10`)**를 사용합니다.
* **루프 순회에 따른 디버깅 추적**:
  1. **초기값**: `x = 1209`, `t = 0`
  2. **1회전**: 
     * `t += 1209 % 10` $\rightarrow$ `t += 9` $\rightarrow$ `t = 9`
     * 증감식 작동: `x ② 10` $\rightarrow$ `x /= 10` 에 의해 `x = 120`으로 몫 갱신.
  3. **2회전**: 
     * `t += 120 % 10` $\rightarrow$ `t += 0` $\rightarrow$ `t = 9`
     * 증감식 작동: `x /= 10` 에 의해 `x = 12`로 몫 갱신.
  4. **3회전**: 
     * `t += 12 % 10` $\rightarrow$ `t += 2` $\rightarrow$ `t = 11`
     * 증감식 작동: `x /= 10` 에 의해 `x = 1`로 몫 갱신.
  5. **4회전**: 
     * `t += 1 % 10` $\rightarrow$ `t += 1` $\rightarrow$ `t = 12`
     * 증감식 작동: `x /= 10` 에 의해 `x = 0`으로 몫 갱신.
  6. **루프 종료**: 
     * `x`가 `0`이 되는 순간 루프 조건식인 `x ① 0`이 거짓(False)이 되어 루프를 빠져나와야 합니다. 
     * 따라서 빈칸 ①의 조건식은 `x != 0` 또는 `x > 0`이 되어야 합니다. (지문에 `0`이 이미 적혀 있으므로 빈칸에는 관계 연산자인 **`!=`** 또는 **`>`** 만 들어갑니다.)
