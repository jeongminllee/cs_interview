## 146. 운영체제 - 응용 소프트웨어 기초 기술 (프로세스 선점형 스케줄링 라운드 로빈 Round Robin 특징)

### 문제
모든 프로세스에 대해 같은 크기의 CPU 시간을 할당(시간 할당량)하고, 프로세스가 할당된 시간 내에 처리 완료를 못하면 준비 큐 리스트의 가장 뒤로 보내지고, CPU는 대기 중인 다음 프로세스로 넘어가는 스케줄링 기법은 무엇인지 쓰시오.

### 정답 및 풀이 결과
* **작성한 답**: `라운드 로빈` (동일한 Time Quantum 할당 및 시분할 기반 선점 처리 스케줄링의 고유 정의를 정확하게 식별하여 정답 도출)
* **모범 답안**: **`라운드 로빈 (Round Robin)`** (또는 `Round Robin`, `RR`, `라운드 로빈`)
* **결과**: **정답 (100% 매칭)**

### 핵심 해설
이 문제는 운영체제(OS)의 프로세스 관리 중 CPU를 효율적으로 분배하기 위한 **프로세스 스케줄링 알고리즘 중 시분할 시스템의 대표적인 선점(Preemptive) 스케줄링인 라운드 로빈(Round Robin)의 고유 개념**을 식별하는 문제입니다.

* **라운드 로빈(Round Robin)의 핵심 기출 특징 정밀 분석**:
  1. **동일한 시간 할당량 (Time Slice / Time Quantum)**: 각 프로세스는 규정된 일정 시간 동안만 CPU를 점유할 수 있습니다.
  2. **선점(Preemptive) 방식**: 할당된 시간이 경과하면 CPU 소유권을 강제로 빼앗기고 준비 큐(Ready Queue)의 맨 뒤로 이동합니다.
  3. **시간 할당량 크기에 따른 양면성 (시험 단골 족보)**:
     * *시간 할당량이 너무 크면*: 먼저 들어온 프로세스가 끝날 때까지 대기하게 되므로 **FCFS (First-Come, First-Served) 알고리즘과 동일**해집니다.
     * *시간 할당량이 너무 작으면*: 프로세스 간 빈번한 잦은 전환으로 인해 **문맥 교환(Context Switching) 오버헤드**가 극도로 증가하여 시스템 성능이 저하됩니다.

## 147. 운영체제 - 응용 소프트웨어 기초 기술 (운영체제 OS 2대 핵심 구성 요소 명령어 해석기 쉘 Shell vs 시스템 핵심 커널 Kernel)

### 문제
다음 운영체제 주요 구성 요소에 대한 설명의 괄호 ( ) 안에 들어갈 알맞은 용어를 쓰시오.
* ( ① )은/는 사용자가 입력시킨 명령어 라인을 읽어 필요한 시스템 기능을 실행시키는 명령어 해석기로, 시스템과 사용자 간의 인터페이스를 제공한다.
* ( ② )은/는 운영체제의 핵심이 되는 기능들이 모여 있는 컴퓨터 프로그램이다.

### 정답 및 풀이 결과
* **작성한 답**:
  * **①: 쉘 (Shell)**
  * **②: 커널 (Kernel)**
* **모범 답안**:
  * **①: 쉘 (Shell)** (또는 `Shell`, `쉘`)
  * **②: 커널 (Kernel)** (또는 `Kernel`, `커널`)
* **결과**: **정답 (100% 매칭)**

### 핵심 해설
이 문제는 컴퓨터 시스템 하드웨어와 사용자 사이를 매개하는 **운영체제(OS)의 가장 기본적인 2대 핵심 구성 요소인 쉘(Shell)과 커널(Kernel)의 고유 기능 정의**를 상호 구별할 수 있는지 검증하는 단답형 기출 문제입니다.

* **운영체제 2대 핵심 요소 정밀 분석**:
  1. **쉘 (Shell) (지문 ①번)**:
     * *역할*: 사용자와 시스템(커널) 사이에서 교량 역할을 수행하는 **명령어 해석기(Command Interpreter)**입니다.
     * *동작*: 사용자가 쉘 프롬프트에서 입력한 CLI 명령어나 스크립트를 해석하여 하위 커널이 이해할 수 있는 시스템 콜(System Call) 형태로 번역해 전달합니다.
     * *대표 종류*: `bash` (리눅스 표준), `sh` (Bourne 쉘), `csh` (C 쉘), `zsh` 등.
  2. **커널 (Kernel) (지문 ②번)**:
     * *역할*: 운영체제의 최심부 심장이 되는 **핵심 제어 프로그램**입니다.
     * *동작*: 부팅 직후 주기억장치(RAM)에 상주하면서 하드웨어 자원(CPU, 메모리, 디스크, I/O 장치 등)을 직접 제어하고 관리합니다.
     * *5대 주요 관리 기능*: 프로세스 관리, 기억장치(메모리) 관리, 파일 시스템 관리, 입출력(I/O) 장치 관리, 프로세스 간 통신(IPC) 및 보안.

## 148. SQL 응용 - 데이터베이스 구축 (교수 테이블 조회 SQL 작성 ORDER BY 절의 다중 컬럼 내림차순 DESC 및 오름차순 ASC 적용)

### 문제
[교수] 테이블에서 다음 조건에 부합하도록 데이터를 조회하는 SQL 구문을 작성하시오. (단, 마크다운 HTML 코드 블록에서 문제 조건과 테이블 구조를 정밀 추출하여 풀이한다.)
* **[조건]**: 강의실은 내림차순(DESC)으로 정렬한 후, 같은 값일 때 이름을 오름차순(ASC)으로 정렬하시오.
* **[교수 테이블]**: (컬럼: `이름`, `과목`, `강의실`)

### 정답 및 풀이 결과
* **작성한 답**: `SELECT * FROM 교수 ORDER BY 강의실 DESC, 이름 ASC;` (오름차순 키워드 ASC의 명시적 작성 여부에 대한 채점 안전성을 고려하여 무결한 1·2차 다중 정렬 구문 작성 성공)
* **모범 답안**: **`SELECT * FROM 교수 ORDER BY 강의실 DESC, 이름 ASC;`** (또는 `SELECT * FROM 교수 ORDER BY 강의실 DESC, 이름;`)
* **결과**: **정답 (100% 매칭)**

### 핵심 해설
이 문제는 관계형 데이터베이스(RDBMS)의 표준 SQL 구문 중 **`SELECT` 문의 `ORDER BY` 절을 활용하여 다중 컬럼(Multi-Column)을 기준으로 1차, 2차 정렬 순서와 정렬 방향(오름차순 `ASC` vs 내림차순 `DESC`)을 지정**할 수 있는지 검증하는 빈출 DML 작성 문제입니다.

* **SQL ORDER BY 구문 정밀 해체**:
  1. **기본 조회 절**: `SELECT * FROM 교수` $\rightarrow$ 교수 테이블의 모든 속성(`이름`, `과목`, `강의실`)을 가져옵니다.
  2. **1차 정렬 조건 (`강의실 DESC`)**:
     * `강의실` 컬럼을 기준으로 **내림차순(DESC, Descending)** 정렬합니다.
     * 알파벳 순서상 `'B'`가 `'A'`보다 크므로, `'B'` 강의실인 수제비쌤 행이 최상단(1행)에 먼저 배치됩니다.
  3. **2차 정렬 조건 (`이름 ASC`)**:
     * 1차 정렬 결과에서 `강의실` 값이 동일하게 `'A'`로 일치하는 행들(두음쌤, 보안쌤, 지기쌤)에 한해 **2차 정렬**을 적용합니다.
     * `이름` 컬럼을 기준으로 **오름차순(ASC, Ascending)** 정렬합니다.
     * 한글 가나다순으로 `'두음쌤'` ➔ `'보안쌤'` ➔ `'지기쌤'` 순서로 행이 정렬됩니다.
  4. **`ASC` 키워드 작성 팁**:
     * SQL에서 `ORDER BY` 시 정렬 키워드를 생략하면 기본값(Default)으로 `ASC`(오름차순)가 적용됩니다.
     * 하지만 정보처리기사 실기 시험지 조건문에 **"오름차순으로 정렬"**이 명시되어 있을 때에는, 채점위원의 명시성 감점 요인을 사전에 차단하기 위해 **`ASC` 키워드를 명시적으로 꼭 표기**해 주는 것이 가장 이상적인 정답 작성 방식입니다!

## 149. Python - 프로그래밍 언어 활용 (딕셔너리 Dictionary get 메서드의 존재하지 않는 키 참조 및 기본값 Default Value 반환)

### 문제
아래 파이썬 코드를 실행했을 때 출력되는 결과를 쓰시오. (단, 코드 블록 HTML 내에서 바르게 파이썬 코드로 디해싱하여 풀이한다.)
```python
a = {'name': 'soojebi', 'phone': '010-1234-5678', 'birth': '0909'}
print(a.get('age', 'none'))
```

### 정답 및 풀이 결과
* **작성한 답**: `none` (HTML 토큰에서 파이썬 딕셔너리 구문을 정확히 복원하고, get 메서드가 부재 키 참조 시 2번째 인자인 기본값을 반환하는 메커니즘을 적용하여 정답 도출 성공)
* **모범 답안**: **`none`**
* **결과**: **정답 (100% 매칭)**

### 핵심 해설
이 문제는 파이썬의 핵심 자료구조인 **딕셔너리(Dictionary) 객체의 `get()` 메서드 동작 원리**와, 존재하지 않는 키(Key)를 참조할 때 **오류(KeyError) 방지를 위한 기본값(Default Value) 설정 기능**을 바르게 이해하고 있는지 검증하는 단답형 계산 문제입니다.

* **파이썬 딕셔너리 `get(key, default)` 동작 정밀 해체**:
  1. 딕셔너리 `a`에는 `'name'`, `'phone'`, `'birth'` 세 개의 키(Key)가 존재합니다.
  2. `a.get('age', 'none')` 구문이 실행되면, 먼저 첫 번째 매개변수인 `'age'` 키가 딕셔너리 `a`에 존재하는지 조회합니다.
  3. **키 참조 및 예외 방어 기작**:
     * `'age'` 키는 딕셔너리 `a`에 존재하지 않습니다.
     * 일반 대괄호 참조 방식인 `a['age']`로 접근하면 `KeyError` 예외가 발생하며 프로그램이 즉시 중단되지만, **`get()` 메서드를 사용하면 예외가 터지지 않습니다.**
     * 두 번째 인자로 지정된 기본값 문자열인 **`'none'`** 이 최종 반환됩니다.
  4. `print()` 함수는 이 문자열의 따옴표를 떼어내고 화면에 **`none`** 을 출력합니다.
  5. **★ [수험생 실전 통찰 팁] 딕셔너리 카운팅(Counter) 구현 패턴**:
     * 실전 프로그래밍에서 빈도수(Frequency)를 집계할 때 `dict[key] = dict.get(key, 0) + 1` 공식을 사용하는 이유가 바로 이 지점에 있습니다.
     * 키가 최초 등장할 때는 `get(key, 0)`에 의해 기본값 `0`이 반환된 뒤 `+ 1`이 되어 `1`로 초기 적재되고, 이미 존재하는 키일 때는 기존 카운트 값에 `+ 1`이 중첩되어 `KeyError` 없이 안전하게 카운터(Counter) 로직을 완성하게 됩니다.

## 150. Java - 프로그래밍 언어 활용 (클래스 로딩 정적 static 블록 인스턴스 초기화 블록 부모 생성자 super 및 오버로딩 순서 정밀 트레이싱)

### 문제
아래 자바 코드를 실행했을 때 출력되는 결과를 쓰시오. (단, 코드 블록 HTML 내에서 바르게 자바 코드로 디해싱하여 풀이한다.)
```java
class Super {
    static int a = 10;
    int b = 20;
    static { a += 30; }
    { a += 5; b += 5; }

    Super() { b += 5; }

    void display() { System.out.print(a + b + " "); }
}
class Sub extends Super {
    static { a += 10; }
    { a += 10; b += 10; }
    Sub() { b += 5; }
    Sub(int n) { b += n; }
    @Override
    void display() { System.out.print(a + b + " "); }
}
public class Soojebi {
    public static void main(String[] args) {
        Super obj1 = new Super();
        obj1.display();
        Super obj2 = new Sub();
        obj2.display();
        Sub obj3 = new Sub(30);
        obj3.display();
    }
}
```

### 정답 및 풀이 결과
* **작성한 답**: `75 115 155 ` (정적 변수 a의 공유 누적과 인스턴스 변수 b의 독립 초기화, 부모-자식 블록 가동 순서를 정밀 추적하여 킬러 문항 정답 도출 성공)
* **모범 답안**: **`75 115 155 `** (또는 `75 115 155`)
* **결과**: **정답 (100% 매칭)**

### 핵심 해설
이 문제는 자바 프로그래밍 시험에서 최고 난이도로 출제되는 **JVM의 클래스 로딩 시점(`static { }`), 인스턴스 초기화 블록(`{ }`), 부모 생성자(`super()`) 자동 호출 순서, 그리고 정적 변수(`static`)와 인스턴스 변수 간의 생명주기 차이**를 정밀하게 트레이싱할 수 있는지 검증하는 최상급 융합 문제입니다.

* **JVM 4단계 객체 생성 및 초기화 우선순위 족보**:
  1. **`1순위: 클래스 로딩 (static)`** $\rightarrow$ JVM에 해당 클래스가 처음 참조될 때 딱 1번만 실행 (`static` 변수 및 `static { }` 블록 가동).
  2. **`2순위: 부모 클래스의 인스턴스 초기화`** $\rightarrow$ 자식 객체 생성 시 부모의 `int b = 20;` ➔ 부모 인스턴스 블록 `{ }` ➔ 부모 생성자 `Super()` 순서 가동.
  3. **`3순위: 자식 클래스의 인스턴스 초기화`** $\rightarrow$ 자식 인스턴스 블록 `{ }` 가동.
  4. **`4순위: 자식 생성자 본문 실행`** $\rightarrow$ 호출된 자식 생성자 `Sub()` 또는 `Sub(int n)` 실행.

* **메인 메서드 3개 객체 생성 트레이싱 분석**:

  1. **`Super obj1 = new Super();`**:
     * *`Super` 클래스 로딩*: `static int a = 10;` ➔ `static { a += 30; }` $\rightarrow$ 정적 변수 **$a = 40$**
     * *`obj1` 인스턴스 생성*: `b = 20` ➔ `Super` 인스턴스 블록 `{ a += 5; b += 5; }` ($a=45, b=25$) ➔ `Super()` 생성자 (`b += 5` $\rightarrow$ $b=30$)
     * *`obj1.display()`*: $a + b = 45 + 30 =$ **`75 `** 인쇄 ($a$는 공유 변수이므로 45 유지)

  2. **`Super obj2 = new Sub();`**:
     * *`Sub` 클래스 최초 로딩*: `Sub` 정적 블록 `static { a += 10; }` 가동 $\rightarrow$ **$a = 45 + 10 = 55$**
     * *`obj2` 인스턴스 생성*:
       * ① **부모 `Super` 가동**: `b = 20` ➔ `Super` 인스턴스 블록 ($a=60, b=25$) ➔ `Super()` 생성자 ($b=30$)
       * ② **자식 `Sub` 가동**: `Sub` 인스턴스 블록 `{ a += 10; b += 10; }` ($a=70, b=40$) ➔ `Sub()` 생성자 (`b += 5` $\rightarrow$ $b=45$)
     * *`obj2.display()`*: $a + b = 70 + 45 =$ **`115 `** 인쇄 ($a$는 70으로 갱신)

  3. **`Sub obj3 = new Sub(30);` (★ 수험생의 $b$ 변수 미스터리 완벽 해소)**:
     * *클래스 로딩*: `Super`, `Sub` 모두 이미 완료되었으므로 `static` 블록은 실행되지 않음! (출발점 $a = 70$)
     * *`obj3` 인스턴스 생성*:
       * ① **부모 `Super` 가동**: `b = 20` ➔ `Super` 인스턴스 블록 ($a=75, b=25$) ➔ `Super()` 생성자 ($b=30$)
       * ② **자식 `Sub` 가동**:
         * 자식 인스턴스 블록 `{ a += 10; b += 10; }` 이 생성자 본문보다 **먼저** 가동됩니다!
         * 이로 인해 $a = 75 + 10 = 85$, **$b = 30 + 10 = 40$** 가 됩니다!
         * 마지막으로 호출된 오버로딩 생성자 `Sub(30)` 본문 `b += 30` 이 실행되어 **$b = 40 + 30 = 70$** 이 완성됩니다!
     * *`obj3.display()`*: $a + b = 85 + 70 =$ **`155 `** 인쇄!

  4. **★ [수험생과의 질답 Q&A 심화 필기 노트] Java 상속 생성자 super() 매칭 메커니즘**:
     * **Q1. 자식 생성자 `Sub(30)`을 호출했을 때, 명시하지 않은 부모 생성자 `Super()`는 왜 가동하는가?**
       * **A1**: 자바 문법 규칙상 자식 클래스의 모든 생성자 맨 첫 줄에는 컴파일러가 생략된 **`super();`** 문장을 자동으로 삽입합니다. 이는 부모의 뼈대(인스턴스 변수 및 초기화 블록)가 먼저 완전하게 메모리에 구축되어야 자식이 안전하게 상속받아 사용할 수 있다는 자바 객체지향의 철칙 때문입니다. (아파트 뼈대 공사 후 내부 인테리어 착수 비유)
     * **Q2. `super()` 와 `super(args)` 의 매칭 원리는?**
       * **A2**:
         1. 개발자가 아무것도 안 적거나 `super()` 를 명시한 경우 $\rightarrow$ 부모 클래스의 **인자 없는 기본 생성자 `Super()`** 매칭 및 호출.
         2. 개발자가 `super(object)` 처럼 인자를 명시한 경우 $\rightarrow$ 부모 클래스의 **해당 인자 타입을 매개변수로 받는 오버로딩 생성자 `Super(object)`** 가 명시적 매칭 및 호출.

## 151. Java - 프로그래밍 언어 활용 (예외 처리 try-catch-finally 블록 내 return 문의 finally 우선 가동 및 static 변수 연산 출력)

### 문제
아래 자바 코드를 실행했을 때 출력되는 결과를 쓰시오. (단, 코드 블록 HTML 내에서 바르게 자바 코드로 디해싱하여 풀이한다.)
```java
public class Soojebi {
  static int count = 0;
  public static void main(String[] args) {
    int res = test(10);
    System.out.println(res + " " + count);
  }
  public static int test(int n) {
    try {
      count++;
      if (n > 5) {
          n += count;
          return n;
      }
      return n;
    }
    catch (Exception e) {
        count -= 1;
        return 0;
    }
    finally {
      count += 10;
      n += 100;
      System.out.print("F ");
    }
  }
}
```

### 정답 및 풀이 결과
* **작성한 답**: `11 1` (try 문 내의 return 구문 만남 시 메서드가 조기 종료된다고 오해하여, finally 블록의 무조건 실행 원칙과 내부 System.out.print("F ") 출력 누락으로 정답 도출 실패)
* **모범 답안**: **`F 11 11`**
* **결과**: **오답 (finally 블록의 return 선행 가동 및 내부 print 출력 누락)**

### 핵심 해설
이 문제는 자바 프로그래밍의 예외 처리(Exception Handling) 구조인 **`try-catch-finally` 문에서 `try` 블록 내부의 `return` 문을 만나더라도, 메서드가 즉시 탈출하지 않고 `finally` 블록을 100% 선행 가동시킨 후 최종 반환하는 자바의 필수 보장 매커니즘**을 이해하고 있는지 검증하는 킬러 함정 문제입니다.

* **단계별 트레이싱 및 2대 초빈출 킬러 함정 해체**:
  1. **`test(10)` 메서드 호출 및 `try` 블록 진입**:
     * `count++;` $\rightarrow$ 정적 변수 `count` 값이 0에서 **1**로 증가합니다.
     * `if (n > 5)` $\rightarrow$ $10 > 5$ 는 **True (참)** 입니다.
     * `n += count;` $\rightarrow$ $n = 10 + 1 =$ **`11`** 이 됩니다.
     * `return n;` $\rightarrow$ 반환값 `11`을 임시 보관(예약)하지만, **메서드를 바로 탈출하지 못하고 `finally` 블록으로 강제 이동합니다.**
  2. **`finally` 블록 무조건 가동 (★ 킬러 함정 구간)**:
     * `count += 10;` $\rightarrow$ `count` 값이 $1 + 10 =$ **`11`** 로 갱신됩니다!
     * `n += 100;` $\rightarrow$ $n$이 111이 되지만, 이미 `return`할 반환값(11)은 `try`문에서 11로 임시 확정되었으므로 리턴값에는 영향을 주지 않습니다.
     * **`System.out.print("F ");`** 가 가동되어 화면에 **`F `** 가 즉시 1차 인쇄됩니다.
  3. **메서드 탈출 및 `main` 복귀**:
     * `test(10)` 메서드는 임시 보관해둔 반환값 **`11`** 을 리턴하며 최종 종료됩니다.
     * `res` = 11, `count` = 11
  4. **`main` 메서드 출력문 실행**:
     * `System.out.println(res + " " + count);` 가 가동되어 **`11 11`** 이 2차 인쇄됩니다.
  5. **최종 종합 출력물**:
     * 1차 인쇄된 `F ` 뒤에 2차 인쇄된 `11 11` 이 이어지므로 최종 결과는 **`F 11 11`** 이 됩니다.
  6. **★ [수험생의 핵심 통찰 노하우] return 선행과 static 전역 변수의 관계**:
     * `try` 문 내부에서 `return n;` 을 만나는 순간 기본 타입 리턴값(`11`)은 이미 확정되어 보관되며, 그 즉시 `finally` 블록으로 넘어가 선행 실행됩니다.
     * `finally` 내부에서 정적 변수(`static count`)는 파이썬의 `global` 전역 변수처럼 프로그램 전체에 연산이 그대로 즉시 반영되어 `count = 11`이 되고, internal `print("F ")` 가 1차 인쇄된 후 탈출하여 최종 `F 11 11` 구조를 완성하게 됩니다.

## 152. Java - 프로그래밍 언어 활용 (this 키워드 누락에 따른 매개변수 섀도잉 Variable Shadowing 무효화 및 static 객체 공유 변수 연산)

### 문제
아래 자바 코드를 실행했을 때 출력되는 결과를 쓰시오. (단, 수험생이 기술한 복원 코드에 기초하여 정밀 풀이한다.)
```java
class Person {
    String name;
    static int cnt = 0;

    Person(String name) {
        this.name = name;
        cnt++;
    }
    Person() {
        this.name = null;
        cnt++;
    }
    void setName(String name) {
        name = name; // ★ this 키워드 누락 킬러 함정 구간!
    }
    String getName() {
        return "Person";
    }
}

public class Soojebi {
    public static void main(String[] args) {
        Person p1 = new Person("soojebi");
        Person p2 = new Person();
        p1.setName("java");
        System.out.println(p1.name);
        System.out.println(Person.cnt);
        System.out.println(p1.getName());
    }
}
```

### 정답 및 풀이 결과
* **작성한 답**:
  `soojebi`
  `2`
  `Person` (this 키워드 누락 시 인스턴스 멤버 변수가 갱신되지 않는 변수 섀도잉 현상을 정밀 간파하고 static cnt의 객체 간 공유 특징을 적용하여 정답 도출 성공)
* **모범 답안**:
  **`soojebi`**
  **`2`**
  **`Person`**
* **결과**: **정답 (100% 매칭)**

### 핵심 해설
이 문제는 자바의 클래스 설계 시 멤버 변수(Field)와 메서드 매개변수(Parameter)의 이름이 동일할 때, **`this` 참조 키워드의 유무에 따른 변수 섀도잉(Variable Shadowing) 현상**과, 모든 인스턴스가 단 하나의 메모리 공간을 공유하는 **정적 변수(`static cnt`)의 누적 동작**을 정밀하게 트레이싱할 수 있는지 검증하는 킬러 문제입니다.

* **단계별 트레이싱 및 킬러 함정 해체**:
  1. **`Person p1 = new Person("soojebi");`**:
     * `p1.name = "soojebi"` 로 설정됩니다.
     * `cnt++` $\rightarrow$ 정적 변수 `cnt`가 0에서 **1**로 증가합니다.
  2. **`Person p2 = new Person();`**:
     * `p2.name = null` 로 설정됩니다.
     * `cnt++` $\rightarrow$ 정적 변수 `cnt`는 모든 객체가 공유하므로 $1 + 1 =$ **`2`** 가 됩니다.
  3. **`p1.setName("java");` (★ 초빈출 킬러 함정 구간)**:
     * `setName(String name)` 메서드 내부 코드가 `this.name = name;` 이 아니라 **`name = name;`** 으로 작성되어 있습니다.
     * 이 구문은 인스턴스 필드인 `this.name`을 수정하는 것이 아니라, **자기 자신인 지역 매개변수 `name`에 자기 자신("java")을 다시 대입하고 끝나는 자기할당(Self-Assignment) 무효 연산**입니다.
     * 따라서 인스턴스 변수 `p1.name` 은 전혀 변경되지 않고 그대로 **`"soojebi"`** 로 유지됩니다!
  4. **출력문 3줄 가동**:
     * `System.out.println(p1.name);` ➔ **`soojebi`**
     * `System.out.println(Person.cnt);` ➔ **`2`**
     * `System.out.println(p1.getName());` ➔ 메서드가 리턴하는 문자열 **`Person`**

## 153. C언어 - 프로그래밍 언어 활용 (문자열 포인터 증감 연산자 p++ 및 ++p와 서식 지정자 %s %c 동작 트레이싱)

### 문제
아래 C언어 코드를 실행했을 때 출력되는 결과를 쓰시오. (단, 수험생이 기술한 복원 코드 및 풀이 조건에 기초하여 정밀 해설한다.)
```c
#include <stdio.h>
int main() {
    char *p = "soojebi world";
    printf("%s\n", p++);
    printf("%s\n", ++p);
    printf("%c\n", *p);
    printf("%c\n", *p++);
    return 0;
}
```

### 정답 및 풀이 결과
* **작성한 답**:
  `soojebi world`
  `ojebi world`
  `o`
  `o` (포인터 전위/후위 증감연산의 실행 시점과 서식자 %s 및 %c의 데이터 읽기 단위를 정확하게 적용하여 정답 도출 성공)
* **모범 답안**:
  **`soojebi world`**
  **`ojebi world`**
  **`o`**
  **`o`**
* **결과**: **정답 (100% 매칭)**

### 핵심 해설
이 문제는 C언어의 핵심인 **문자열 포인터(Pointer)의 증감 연산자(`p++` vs `++p`)에 따른 주소 이동 시점**과, **서식 지정자 `%s`(문자열) 및 `%c`(단일 문자)의 데이터 참조 방식**을 정확하게 트레이싱할 수 있는지 검증하는 빈출 문제입니다.

* **단계별 라인 트레이싱 분석**:
  1. **`char *p = "soojebi world";`**:
     * 포인터 변수 `p`가 문자열 상수 `"soojebi world"`의 시작 위치인 인덱스 0번 문자 `'s'`의 메모리 주소를 가리킵니다.
  2. **`printf("%s\n", p++);` (1번째 출력)**:
     * **후위 증감(`p++`)**이므로, 현재 `p`가 가리키는 위치(`'s'`)부터 `%s`로 전체 문자열을 인쇄한 후 `p`를 1 증가시킵니다.
     * 인쇄: **`soojebi world`**
     * 주소 이동: `p`는 인덱스 1번인 문자 `'o'`의 주소를 가리키게 됩니다.
  3. **`printf("%s\n", ++p);` (2번째 출력)**:
     * **전위 증감(`++p`)**이므로, 먼저 `p`를 1 증가시킨 후 출력합니다.
     * 주소 이동: `p`는 인덱스 2번인 문자 `'o'`의 주소를 가리킵니다.
     * 인쇄: `%s`는 인덱스 2번 `'o'`부터 문자열 끝까지 인쇄하므로 **`ojebi world`** 가 출력됩니다.
  4. **`printf("%c\n", *p);` (3번째 출력)**:
     * 현재 `p`는 인덱스 2번 문자 `'o'`를 가리키고 있습니다.
     * `%c`는 단일 문자를 인쇄하므로 `*p`에 해당하는 문자 **`o`** 가 1자 인쇄됩니다. (주소 이동 없음)
  5. **`printf("%c\n", *p++);` (4번째 출력)**:
     * `*p++` 연산자 우선순위는 포인터 간접 참조 `*` 와 후위 증감 `++` 의 조합입니다.
     * 현재 `p`가 가리키는 인덱스 2번 문자 **`o`** 를 먼저 `%c`로 1자 인쇄합니다.
     * 인쇄 직후 후위 증감 `p++`가 실행되어 `p`는 인덱스 3번 문자 `'j'`의 주소로 이동합니다.

## 154. C언어 - 프로그래밍 언어 활용 (문자 배열 아스키코드 연산 x[i]-'A' 및 for 조건식 대입 연산 0 거짓 종결 트레이싱)

### 문제
아래 C언어 코드를 실행했을 때 출력되는 결과를 쓰시오. (단, 코드 블록 HTML 내에서 바르게 C언어 코드로 디해싱하여 풀이한다.)
```c
#include <stdio.h>

int main(){
  char x[] = "EATS";
  char s[100] = "";
  int i=0;

  for(i=0; i<100; i++)
    s[i] = i%5;

  for(i=0; s[x[i]] += s[x[i]]; i++)
    printf("%d", x[x[i]-'A']);

  return 0;
}
```

### 정답 및 풀이 결과
* **작성한 답**: `0` (i=0 일 때 NULL 문자 '\0'의 정수값 0이 출력되고, i=1 일 때 s['A']의 값 0이 조건식이 되어 for 문이 조기 거짓 종결되는 메커니즘을 정확히 추적 성공)
* **모범 답안**: **`0`**
* **결과**: **정답 (100% 매칭)**

### 핵심 해설
이 문제는 C언어의 **문자 배열 아스키코드 연산(`'A'` = 65, `'E'` = 69)**과, **2차 중첩 배열 인덱스 참조(`x[x[i]-'A']`)**, 그리고 **`for` 문 조건식에 대입 연산자(`+=`)가 사용되었을 때 결과가 `0`이 되면 거짓(False)으로 즉시 종결되는 제어 흐름**을 정밀하게 분석할 수 있는지 평가하는 고난도 킬러 문제입니다.

* **단계별 트레이싱 및 정밀 해체**:
  1. **첫 번째 `for` 루프 (`s` 배열 초기화)**:
     * `s[i] = i % 5` 식이 0부터 99까지 실행됩니다.
     * 인덱스가 5의 배수(`0, 5, 10, ..., 65, 70, 75, ...`)인 위치의 `s[i]` 값은 모두 **`0`** 입니다!
     * 특히, `'A'`(65) 위치의 값: **`s[65] = 65 % 5 = 0`**
     * `'E'`(69) 위치의 값: **`s[69] = 69 % 5 = 4`**
  2. **두 번째 `for` 루프 1회차 (`i = 0`)**:
     * `x[0]` = `'E'` (아스키코드 69)
     * *조건식 평가*: `s[x[0]] += s[x[0]]` $\rightarrow$ `s[69] += s[69]` $\rightarrow$ $4 + 4 =$ **`8`** (0이 아니므로 **True**)
     * *출력문 실행*: `printf("%d", x[x[0]-'A']);`
       * `x[0] - 'A'` = $69 - 65 =$ **`4`**
       * `x[4]` 참조: `"EATS"` 문자열의 4번 인덱스에는 NULL 문자 **`'\0'`** (아스키 0)가 위치함!
       * `printf("%d", '\0')` 에 의해 숫자 **`0`** 가 1차 출력됩니다.
  3. **두 번째 `for` 루프 2회차 (`i = 1`)**:
     * `x[1]` = `'A'` (아스키코드 65)
     * *조건식 평가*: `s[x[1]] += s[x[1]]` $\rightarrow$ `s[65] += s[65]`
     * 1단계에서 `s[65]` 는 **`0`** 이었습니다.
     * 따라서 `s[65] += s[65]` 은 $0 + 0 =$ **`0`** 이 됩니다!
     * **C언어 조건식에서 `0` 은 거짓(False)이므로 `for` 반복문이 즉시 종결(Exit)됩니다!**
  4. **최종 출력**: `i=0` 일 때 출력된 **`0`** 이 최종 정답이 됩니다.
  5. **★ [수험생의 본질 통찰 노트] 필수 아스키 3대 기준값('A'=65)의 위력**:
     * 본 문제는 결국 대문자 시작 기준값인 **`'A' = 65`** 를 정확히 숙지하고 있어야 $65 \% 5 = 0$ 이라는 거짓(False) 종결 스위치를 당길 수 있는 구조였습니다.
     * C언어 실기 시험 대비 필수 3대 아스키 기준값인 **`'A'=65`**, **`'a'=97`**, **`'0'=48`** 은 단답형 및 트레이싱 문제의 생사줄임을 재확인합니다.

## 155. C언어 - 프로그래밍 언어 활용 (이중 논리 부정 연산자 !(!a)의 정수 1 환산 및 for 증감식 i *= a+b 연산자 우선순위 트레이싱)

### 문제
아래 C언어 코드를 실행했을 때 출력되는 결과를 쓰시오. (단, 코드 블록 HTML 내에서 바르게 C언어 코드로 디해싱하여 풀이한다.)
```c
#include <stdio.h>

int main(){
  int a = 3;
  int b = !(!a);
  int i = 0;
  
  for(i=1; i<20; i *= a+b)
    printf("%d\n", i);

  return 0;
}
```

### 정답 및 풀이 결과
* **작성한 답**: `1\n6` (이중 논리 부정 !(!a)의 연산 결과가 원본 값 3이 된다고 착각하여 b=3 및 a+b=6 으로 판단해 정답 도출 실패)
* **모범 답안**:
  **`1`**
  **`4`**
  **`16`** (또는 `1\n4\n16`)
* **결과**: **오답 (이중 논리 부정의 C언어 정수 1 환산 특징 및 복합 대입 연산자 곱셈 누적 오류)**

### 핵심 해설
이 문제는 C언어의 **논리 부정 연산자(`!`)가 0이 아닌 참(True) 값에 적용될 때 정수 `1` 또는 `0`으로 변환되는 특징**과, **`for` 문 증감식의 복합 대입 연산자(`*=`)와 산술 연산자(`+`) 간의 우선순위 연산**을 정확하게 계산할 수 있는지 검증하는 빈출 함정 문제입니다.

* **단계별 트레이싱 및 킬러 함정 해체**:
  1. **`int b = !(!a);` (★ 초빈출 킬러 함정 구간)**:
     * `a = 3` (C언어에서 0이 아닌 정수 3은 **참 True** 입니다.)
     * 1차 부정 `!a` $\rightarrow$ `!3` $\rightarrow$ 참을 부정하므로 정수 **`0` (거짓 False)** 이 됩니다.
     * 2차 부정 `!(!a)` $\rightarrow$ `!0` $\rightarrow$ 거짓을 부정하므로 정수 **`1` (참 True)** 이 됩니다!
     * **자바나 파이썬처럼 원본 값 3으로 돌아가는 것이 아니라, C언어의 논리 연산 결과는 무조건 정수 `1`로 치환되어 `b = 1` 이 됩니다!**
  2. **`for(i=1; i<20; i *= a+b)` 증감식 구조 해석**:
     * `a + b` = $3 + 1 =$ **`4`**
     * `i *= a + b` 식은 대입 연산자 우선순위에 의해 **`i = i * (a + b)`** 로 묶입니다.
     * 즉, `i` 값에 매 루프마다 **`4` 를 곱하는 연산**(`i = i * 4`)입니다.
  3. **`for` 루프 반복 수행**:
     * **1회차 (`i = 1`)**: $1 < 20$ (참) $\rightarrow$ `printf`에 의해 **`1`** 인쇄.
       * 증감식: `i` = $1 \times 4 =$ **`4`**
     * **2회차 (`i = 4`)**: $4 < 20$ (참) $\rightarrow$ `printf`에 의해 **`4`** 인쇄.
       * 증감식: `i` = $4 \times 4 =$ **`16`**
     * **3회차 (`i = 16`)**: $16 < 20$ (참) $\rightarrow$ `printf`에 의해 **`16`** 인쇄.
       * 증감식: `i` = $16 \times 4 =$ **`64`**
     * **4회차 (`i = 64`)**: $64 < 20$ (**거짓 False**) $\rightarrow$ `for` 문 종료!
  4. **★ [수험생과의 질답 Q&A 필기 노트] 주요 언어 3종 (C vs Java vs Python) 논리 부정 !(!a) 반환 타입 비교 족보**:
     * **C 언어**: 논리 연산의 결과가 정수(int) 타입 **`1`** 또는 **`0`** 으로 변환되어 저장됨! (원본 숫자 3으로 돌아가지 않고 `1` 이 됨)
     * **Java 언어**: 정수형 변수 `int a = 3;` 에 논리 연산자 `!a` 를 취하면 **컴파일 에러 (Compile Error)** 발생! (`boolean` 변수 `boolean a = true;` 에 `!(!a)` 를 취해야 **`true`** 로 원상 복귀됨)
     * **Python 언어**: 숫자 `a = 3` 에 `not(not a)` 를 취하면 원본 숫자 3이 아니라 불리언 타입인 **`True`** 값으로 환산됨!

## 156. 서비스 운영 및 인프라 구축 - 응용 소프트웨어 기초 기술 (참조의 국소성 Locality 3대 유형 공간 Spatial 시간 Temporal 순차 Sequential 국부성)

### 문제
다음 국부성(Locality) 유형 설명의 괄호 ( ) 안에 들어갈 알맞은 용어를 쓰시오.
* ( ① ) locality는 프로세스 실행 시 일정 위치의 페이지를 집중적으로 액세스하는 현상으로 사례로는 배열 순회 등이 있다.
* ( ② ) locality는 최근 사용되었던 기억장소들이 집중적으로 액세스하는 현상으로 사례로는 Loop, Stack, Sub Routine 등이 있다.
* ( ③ ) locality는 데이터가 순차적으로 액세스 되는 현상으로 사례로는 순차적 코드 실행이 있다.

### 정답 및 풀이 결과
* **작성한 답**:
  * **①: 공간 (Spatial)**
  * **②: 시간 (Temporal)**
  * **③: 순차 (Sequential)**
* **모범 답안**:
  * **①: 공간** (또는 `공간 국부성`, `Spatial`)
  * **②: 시간** (또는 `시간 국부성`, `Temporal`)
  * **③: 순차** (또는 `순차 국부성`, `Sequential`)
* **결과**: **정답 (100% 매칭)**

### 핵심 해설
이 문제는 컴퓨터 시스템의 주기억장치 및 캐시 메모리(Cache Memory)의 효율적 운용과 가상 메모리 관리의 기초가 되는 **참조의 국소성(Locality of Reference) 3대 개념인 공간 국부성, 시간 국부성, 순차 국부성의 대표적 사례와 특징**을 상호 구분할 수 있는지 검증하는 단답형 기출 문제입니다.

* **국부성(Locality) 3대 유형 정밀 분석 및 대표 사례 족보**:
  1. **공간 국부성 (Spatial Locality) (지문 ①번)**:
     * *정의*: 프로세스 실행 시 기억장치의 특정 위치 근처(주변 장소)에 인접한 데이터들이 연속적으로 액세스되는 특성.
     * *대표 사례*: **배열 순회(Array Traversal)**, 인접 변수 선언/참조, 테이블 요소 접근.
  2. **시간 국부성 (Temporal Locality) (지문 ②번)**:
     * *정의*: 최근에 한번 참조된 기억장소의 데이터가 가까운 미래에 또다시 집중적으로 재참조되는 특성.
     * *대표 사례*: **반복문 루프(Loop: for/while)**, **스택(Stack)**, **부프로그램(Subroutine)**, 카운팅(Counting) 변수.
  3. **순차 국부성 (Sequential Locality) (지문 ③번)**:
     * *정의*: 프로그램의 데이터나 명령어가 메모리에 저정된 순서대로 차례차례 순차적으로 실행되는 특성.
     * *대표 사례*: **순차적 코드 실행(Sequential Code Execution)**, 명령 프로그램 카운터(PC)의 순차 증가.

## 157. 서비스 운영 및 인프라 구축 - 소프트웨어 개발 보안 구축 (재해 복구 시스템 DRS 대기 센터 4대 유형 Hot Site vs Warm Site 복구 시간 RTO 기준 구별)

### 문제
다음 재해 복구 센터(DRS) 유형 설명의 괄호 ( ) 안에 들어갈 알맞은 용어를 쓰시오.
* ( ① )은/는 주 센터와 동일한 수준의 자원을 대기 상태로 원격지에 보유하면서 동기, 비동기 방식의 미러링을 통하여 데이터의 최신 상태를 유지하고 있는 재해복구센터로 RTO는 4시간 이내이다.
* ( ② )은/는 ( ① )와/과 유사하나 재해복구센터에 주 센터와 동일한 수준의 자원을 보유하는 대신 중요성이 높은 자원만 부분적으로 재해복구센터에 보유하고 있는 센터로 RTO는 수일~수주이다.

### 정답 및 풀이 결과
* **작성한 답**:
  * **①: Hot Site**
  * **②: Cold Site** (부분적 중요 자원 보유 및 수일~수주 RTO 스펙을 최하위 인프라 부지 확보 단계인 Cold Site로 착각하여 ②번 오답 처리)
* **모범 답안**:
  * **①: Hot Site (핫 사이트)**
  * **②: Warm Site (웜 사이트)**
* **결과**: **부분 정답 (①번 정답, ②번 웜 사이트 Warm Site 오답)**

### 핵심 해설
이 문제는 정보시스템 재해 복구 관리(Disaster Recovery System) 영역에서 비즈니스 연속성(BCP) 확보를 위해 구축하는 **DRS 4대 대기 복구 센터(Mirror Site, Hot Site, Warm Site, Cold Site)의 자원 보유 수준 및 목표 복구 시간(RTO: Recovery Time Objective)**을 구별할 수 있는지 평가하는 기출 단답형 문제입니다.

* **DRS 4대 대기 복구 센터 정밀 스펙 비교 족보**:
  1. **`Mirror Site (미러 사이트)`**:
     * *자원 수준*: 주 센터와 동일한 시스템 및 데이터를 실시간 복제(Active-Active).
     * *RTO*: **`0 (즉시 복구)`** / 데이터 손실 0.
  2. **`Hot Site (핫 사이트) (지문 ①번)`**:
     * *자원 수준*: 주 센터와 **동일한 수준의 자원을 대기 상태(Active-Standby)**로 보유하며 동기/비동기 실시간 미러링 수행.
     * *RTO*: **`수시간 이내 (보통 4시간 이내)`**.
  3. **`Warm Site (웜 사이트) (지문 ②번)`**:
     * *자원 수준*: 주 센터 자원 전체 대신 **중요성이 높은 자원만 '부분적으로' 보유**하고 백업 데이터를 수송/복구함.
     * *RTO*: **`수일 ~ 수주 이내`**.
  4. **`Cold Site (콜드 사이트)`**:
     * *자원 수준*: 데이터 복구 시설의 **기본 부지(장소), 전력/통신 기반 시설만 확보**해 두고 재해 시 신규 장비를 새로 구매 및 배치함.
     * *RTO*: **`수주 ~ 수개월 이내`**.

## 158. 서비스 운영 및 인프라 구축 - 소프트웨어 개발 보안 구축 (경유지 반사 서버 Reflector 및 IP 스푸핑 기반 DRDoS 분산 반사 서비스 거부 공격)

### 문제
다음 3단계 보안 공격 기법 설명에 해당하는 보안 공격 명칭을 쓰시오.
* **1단계**: 공격자는 많은 수의 경유지 서버에 특정 질의 명령어(요청)를 일괄적으로 보냄. 자신의 IP를 공격 대상 IP로 변조(IP Spoofing)하여 공격 대상 IP가 응답받도록 하고, 효과적인 공격을 위하여 경유지 서버가 응답하는 양이 큰 질의 명령어를 사용함.
* **2단계**: 경유지 서버는 정상 요청으로 인식하여 공격 대상 서버에 명령어 결과(응답)를 보냄.
* **3단계**: 많은 수의 경유지 서버로부터 응답을 받은 공격 대상 네트워크는 트래픽이 급증하여 회선 대역폭이 가득 차게 됨.

### 정답 및 풀이 결과
* **작성한 답**: `DDoS (분산 서비스 거부 공격)` (분산된 트래픽 공격의 대분류인 DDoS 개념에 90% 이상 정밀 접근했으나, 경유지 반사체 Reflector 및 IP 스푸핑 증폭 메커니즘을 구별하는 파생 기술명 DRDoS 기재 누락으로 오답 처리)
* **모범 답안**: **`DRDoS`** (또는 `분산 반사 서비스 거부 공격`, `DRDoS (분산 반사 서비스 거부 공격)`)
* **결과**: **오답 (일반 DDoS와 반사체 Reflector 기반 DRDoS 세부 구별 실패)**

### 핵심 해설
이 문제는 정보보안 및 소프트웨어 개발 보안 영역의 대표적인 서비스 거부 공격(DoS/DDoS) 파생 유형 중 **경유지 서버(Reflector)를 반사체로 이용하고 출발지 IP를 변조(IP Spoofing)하여 응답 트래픽을 증폭시켜 공격하는 DRDoS(Distributed Reflective Denial of Service, 분산 반사 서비스 거부 공격)** 의 동작 메커니즘을 식별할 수 있는지 평가하는 기출 단답형 문제입니다.

* **DDoS vs DRDoS 2대 킬러 차이점 족보**:
  1. **`일반 DDoS (Distributed Denial of Service)`**:
     * *방식*: 공격자가 악성코드에 감염된 **좀비 PC(Botnet)** 들을 직접 조종하여 공격 대상 서버로 **직접(Direct)** 대량의 트래픽을 쏘아 보냄.
  2. **`DRDoS (Distributed Reflective Denial of Service) (본 문제)`**:
     * *방식*: 공격자가 자신의 IP를 공격 대상의 IP로 속이는 **`IP 스푸핑(IP Spoofing)`** 을 적용한 후, 정상 작동하는 무수히 많은 **`경유지 서버(Reflector, 반사체)`** 로 소량의 질의 요청을 보냄.
     * *결과*: 경유지 서버들이 반사적으로 **엄청난 크기의 응답(증폭 트래픽)** 을 공격 대상 서버로 한꺼번에 쏟아부어 공격 대상의 네트워크 대역폭을 마비시킴.

## 159. 서비스 운영 및 인프라 구축 - 응용 소프트웨어 기초 기술 (국가망 및 광역 공중통신망 상호 접속 커버리지 WAN Wide Area Network 광역 통신망)

### 문제
다음 설명에 해당하는 네트워크 종류(용어)를 쓰시오.
* ( ) 은/는 국가망 또는 각 국가의 공중통신망을 상호 접속시키는 국제정보통신망으로 설계 및 구축, 운용되는 네트워크이다.
* 공중통신망 사업자가 구축하고, 일반 대중 가입자들에게 보편적인 정보통신 서비스를 제공한다.
* 거리에 제약이 없으나 다양한 경로를 지나 정보가 전달되므로 LAN보다 속도가 느리고 에러율도 높다.

### 정답 및 풀이 결과
* **작성한 답**: `VPN` (공중망을 전용선처럼 암호화 터널링하여 사설망으로 활용하는 보안 네트워크인 VPN 개념으로 잘못 오인하여 오답 처리)
* **모범 답안**: **`WAN`** (또는 `광역 통신망`, `Wide Area Network`, `WAN (광역 통신망)`)
* **결과**: **오답 (네트워크 커버리지 개념인 WAN과 보안 터널링 기술인 VPN 상호 혼동)**

### 핵심 해설
이 문제는 컴퓨터 네트워크 관리 및 인프라 영역에서 **물리적 커버리지(전송 거리 및 광역 범위)에 따른 네트워크 분류인 WAN(Wide Area Network, 광역 통신망)** 의 고유 특성과 명세를 구별할 수 있는지 검증하는 빈출 단답형 문제입니다.

* **네트워크 물리 커버리지 4대 분류 족보**:
  1. **`PAN (Personal Area Network, 개인 영역 네트워크)`**:
     * *범위*: 10m 이내 개인 최단 거리. (Bluetooth, UWB, Zigbee)
  2. **`LAN (Local Area Network, 근거리 통신망)`**:
     * *범위*: 사무실, 학교, 집 등 수 km 이내의 한정된 지역. 높은 전송 속도 및 낮은 에러율. (Ethernet, Wi-Fi)
  3. **`MAN (Metropolitan Area Network, 도시권 통신망)`**:
     * *범위*: 대도시 영역(수십 km 이내). LAN과 WAN의 중간 연결체. (FDDI, DQDB)
  4. **`WAN (Wide Area Network, 광역 통신망) (본 문제)`**:
     * *범위*: **국가, 대륙, 전 세계를 상호 접속시키는 광범위 통신망.**
     * *특징*: 공중통신망 사업자(KT, SKT 등)가 구축하며, **거리 제약이 없으나 LAN에 비해 속도가 느리고 에러율이 상대적으로 높음.**

* **오답 파해: VPN (Virtual Private Network, 가상 사설망) 과의 차이**:
  * **VPN**은 커버리지 종류가 아니라, 인터넷과 같은 공중망(Public Network)을 마치 암호화된 전용선(Private Line)처럼 안전하게 사용할 수 있도록 **터널링(Tunneling) 및 암호화(Encryption)**를 적용하는 **보안 기술**입니다.

## 160. 서비스 운영 및 인프라 구축 - 소프트웨어 개발 보안 구축 (소프트웨어 개발사 배포 업데이트 서버 침투 및 파일 변조 기반 공급망 공격 Supply Chain Attack)

### 문제
다음 설명에 해당하는 보안 공격 기법 명칭을 쓰시오.
* 소프트웨어 개발사의 네트워크에 침투하여 소스 코드의 수정 등을 통해 악의적인 코드를 삽입하거나 배포 서버에 접근하여 악의적인 파일로 변경하는 방식을 통해 사용자 PC에 소프트웨어를 설치 또는 업데이트 시에 자동적으로 감염되도록 하는 공격 기법

### 정답 및 풀이 결과
* **작성한 답**: `웜 (Worm)` (네트워크를 통해 스스로 복제 전파되는 악성 바이러스 프로그램인 웜 개념으로 잘못 오인하여 오답 처리)
* **모범 답안**: **`공급망 공격`** (또는 `Supply Chain Attack`, `공급망 공격 (Supply Chain Attack)`)
* **결과**: **오답 (소프트웨어 개발사/업데이트 서버 경로 오염 기반 공급망 공격과 악성코드 웜 상호 구분 실패)**

### 핵심 해설
이 문제는 소프트웨어 개발 보안 및 인프라 영역에서 최근 핫이슈로 빈출되는 **신뢰된 소프트웨어 개발사나 배포/업데이트 서버(Supply Chain)를 먼저 오염시켜 해당 소프트웨어를 다운로드받는 대규모 정식 사용자 PC를 연쇄 감염시키는 공급망 공격(Supply Chain Attack)** 의 동작 메커니즘을 구별할 수 있는지 평가하는 단답형 기출 문제입니다.

* **공급망 공격(Supply Chain Attack)의 핵심 메커니즘 정밀 분석**:
  1. **침투 타겟**: 사용자 PC가 아닌, 사용자들이 100% 신뢰하는 **`소프트웨어 개발사`**, **`업데이트 서버`**, **`하드웨어 제조업체`** 를 1차 타겟으로 침투합니다.
  2. **오염 방식**: 정식 업데이트 파일이나 소스 코드에 악성코드를 숨겨 놓습니다(백도어/악성 모듈 변조).
  3. **파급력**: 사용자가 정식 업데이트 버튼을 누르는 순간 정상 파일로 위장하여 자동 감염되므로 기존 백신(Vaccine) 필터링을 완벽히 무력화시키는 위협을 가집니다.
  4. *대표 사례*: SolarWinds 사태, 3CX 전화 소프트웨어 업데이트 감염 사태.

* **오답 파해: 웜 (Worm) 과의 확실한 구분점**:
  * **`웜 (Worm)`**: 다른 정상 파일이나 서버에 의존하지 않고, 독자적으로 작동하여 네트워크를 통해 **스스로를 자가 복제(Self-Replication)**하면서 확산되는 악성 프로그램입니다.
  * 지문처럼 **"소프트웨어 개발사의 배포/업데이트 서버 경로를 오염시켜 감염"**시키는 공격 형태는 악성코드 종류가 아닌 **`공급망 공격 (Supply Chain Attack)`** 이 정답입니다.

## 161. Python - 프로그래밍 언어 활용 (문자열 슬라이싱 word[::-1] 기반 팰린드롬 Palindrome 회문 단어 판별 및 최장 길이 len 갱신 출력)

### 문제
아래 파이썬 코드를 실행했을 때 출력되는 결과를 쓰시오. (단, 코드 블록 HTML 내에서 바르게 파이썬 코드로 디해싱하여 풀이한다.)
```python
def soojebi(text):
  str = ""
  for word in text.split():
    if word == word[::-1]:
      if len(word) > len(str):
        str = word
  if str:
    print(f"{str}")
  else:
    print("No found")

text = "banana racecar apple madam noon civic rotor"
soojebi(text)
```

### 정답 및 풀이 결과
* **작성한 답**: `madam` (슬라이싱 word[::-1] 이 팰린드롬(회문)을 판별한다는 원리는 파악했으나, 문자열 앞부분의 최장 단어인 racecar(길이 7)가 madam(길이 5)보다 길어 len 갱신 조건을 충족하지 못함을 놓쳐 오답 처리)
* **모범 답안**: **`racecar`**
* **결과**: **오답 (팰린드롬 판별 성공 후 len(word) > len(str) 최장 단어 길이 비교 갱신 누락)**

### 핵심 해설
이 문제는 파이썬의 핵심 문자열 조작 문법인 **슬라이싱 스텝 `word[::-1]` 을 활용한 팰린드롬(Palindrome, 회문) 검사 로직**과, **`len(word) > len(str)` 조건식을 통한 최장 단어 길이 선택 알고리즘**을 정확하게 트레이싱할 수 있는지 평가하는 문제입니다.

* **단계별 단어 트레이싱 분석**:
  1. `text.split()` ➔ `['banana', 'racecar', 'apple', 'madam', 'noon', 'civic', 'rotor']` 리스트로 분할됩니다.
  2. **`word = 'banana'`**: `'banana' == 'ananab'` $\rightarrow$ **False (거짓)**
  3. **`word = 'racecar'`**:
     * `'racecar' == 'racecar'` $\rightarrow$ **True (참: 팰린드롬 단어!)**
     * `len('racecar')` (7) $>$ `len(str)` (0) $\rightarrow$ **True**
     * **`str = 'racecar'`** 로 갱신됩니다! (현재 최장 길이 = 7)
  4. **`word = 'apple'`**: `'apple' == 'elppa'` $\rightarrow$ **False (거짓)**
  5. **`word = 'madam'`**:
     * `'madam' == 'madam'` $\rightarrow$ **True (참: 팰린드롬 단어!)**
     * `len('madam')` (5) $>$ `len(str)` (7) $\rightarrow$ **False (거짓!)**
     * 길이 5는 기존 최장인 7보다 작으므로 `str` 변수가 갱신되지 않고 그대로 **`'racecar'`** 가 유지됩니다!
  6. **`word = 'noon'`, `'civic'`, `'rotor'`**: 모두 팰린드롬이지만 길이가 각각 4, 5, 5 로 기존 7보다 작아 갱신되지 않습니다.
  7. **최종 출력**: `str` 인 **`racecar`** 가 화면에 인쇄됩니다.

## 162. 소프트웨어 개발 - 소프트웨어 공학 (객체지향 설계 5대 원칙 SOLID 중 단일 책임 원칙 SRP 및 개방 폐쇄 원칙 OCP)

### 문제
다음 객체지향 설계 원칙 설명의 괄호 ( ) 안에 들어갈 알맞은 용어를 쓰시오.
* ( ① )은/는 하나의 클래스는 하나의 목적을 위해서 생성되며, 클래스가 제공하는 모든 서비스는 하나의 책임을 수행하는 데 집중되어 있어야 한다는 원칙이다.
* ( ② )은/는 소프트웨어의 구성요소(컴포넌트, 클래스, 모듈, 함수)는 확장에는 열려있고, 변경에는 닫혀있어야 한다는 원칙이다.

### 정답 및 풀이 결과
* **작성한 답**:
  * **①: 싱글톤** (SOLID 원칙의 S인 단일 책임 원칙 SRP를 GoF 디자인 패턴인 싱글톤 Singleton으로 착각하여 오답)
  * **②: 열림 닫힘 원칙** (OCP 개방 폐쇄 원칙의 개념적 핵심을 바르게 떠올려 100% 정답 인정)
* **모범 답안**:
  * **①: 단일 책임 원칙** (또는 `SRP`, `Single Responsibility Principle`)
  * **②: 개방 폐쇄 원칙** (또는 `OCP`, `Open-Closed Principle`, `개방-폐쇄 원칙`)
* **결과**: **부분 정답 (①번 오답, ②번 개방 폐쇄 원칙 정답)**

### 핵심 해설
이 문제는 객체지향 설계 및 소프트웨어 공학 영역의 최빈출 뼈대 이론인 **객체지향 5대 설계 원칙 (SOLID 원칙)** 중 **단일 책임 원칙(SRP)** 과 **개방 폐쇄 원칙(OCP)** 의 정의와 명세를 식별할 수 있는지 평가하는 기출 단답형 문제입니다.

* **객체지향 5대 설계 원칙 (SOLID) 완전 해체 족보**:
  1. **`S: 단일 책임 원칙 (SRP - Single Responsibility Principle) (지문 ①번)`**:
     * *개념*: 하나의 클래스는 **오직 단 하나의 책임(하나의 목적)**만을 가져야 함. 클래스를 변경해야 하는 이유는 오직 하나뿐이어야 함.
  2. **`O: 개방 폐쇄 원칙 (OCP - Open-Closed Principle) (지문 ②번)`**:
     * *개념*: **확장에는 열려(Open) 있고, 변경(수정)에는 닫혀(Closed) 있어야 한다**는 원칙. 기존 코드를 수정하지 않고 새로운 기능을 추가할 수 있도록 인터페이스를 추상화함.
  3. **`L: 리스코프 치환 원칙 (LSP - Liskov Substitution Principle)`**:
     * *개념*: 자식 클래스(하위 타입)는 언제나 부모 클래스(상위 타입)를 대체할 수 있어야 함. (부모 타입 객체 자리에 자식 객체를 넣어 놓아도 정상 가동됨)
  4. **`I: 인터페이스 분리 원칙 (ISP - Interface Segregation Principle)`**:
     * *개념*: 하나의 범용 인터페이스보다 클라이언트가 실제로 사용하는 여러 개의 구체적인 인터페이스로 분리해야 함. (자신이 사용하지 않는 메서드에 의존하지 않음)
  5. **`D: 의존 역전 원칙 (DIP - Dependency Inversion Principle)`**:
     * *개념*: 구체적인 구현 클래스에 의존하지 말고, 변하지 않는 **추상화(인터페이스)**에 의존해야 함.

* **오답 파해: 싱글톤 (Singleton) 과의 명확한 차이**:
  * **`싱글톤 (Singleton)`**: 객체지향 5대 설계 원칙(SOLID)이 아니라, GoF 디자인 패턴(Design Pattern) 중 생성 패턴의 하나로서 **"전체 시스템에서 인스턴스(객체)를 오직 단 1개만 생성하여 공유"**하는 클래스 생성 패턴입니다.

## 163. 데이터베이스 구축 - SQL 응용 및 데이터베이스 기초 (식별자 및 키 Key 2대 핵심 제약조건 유일성 Uniqueness 및 최소성 Minimality)

### 문제
다음 식별자(Key)의 제약 특성 설명의 괄호 ( ) 안에 들어갈 알맞은 용어를 쓰시오.
* ( ① )은/는 식별자에 의해 엔터티 내에 모든 튜플들을 유일하게 구분하는 특성이다.
* ( ② )은/는 최소한의 속성으로 식별자를 구성하는 특성이다.

### 정답 및 풀이 Result
* **작성한 답**:
  * **①: 원자성** (트랜잭션 ACID 4대 특성 중 하나인 원자성 Atomicity 개념으로 잘못 오인하여 오답)
  * **②: 최소성** (식별자를 구성하는 최소한의 속성 특성을 정밀 파악하여 100% 정답)
* **모범 답안**:
  * **①: 유일성** (또는 `Uniqueness`, `유일성 (Uniqueness)`)
  * **②: 최소성** (또는 `Minimality`, `최소성 (Minimality)`)
* **결과**: **부분 정답 (①번 오답, ②번 최소성 정답)**

### 핵심 해설
이 문제는 데이터베이스(RDB) 설계 및 모델링 영역에서 후보키, 기본키, 슈퍼키를 정의하는 핵심 제약조건인 **유일성(Uniqueness)** 과 **최소성(Minimality)** 의 정의를 구별할 수 있는지 평가하는 기출 단답형 문제입니다.

* **DB 키(Key)의 2대 제약 특성 정밀 비교 족보**:
  1. **`유일성 (Uniqueness) (지문 ①번)`**:
     * *개념*: 하나의 키 값으로 릴레이션(테이블) 내에 존재하는 **모든 튜플(행)을 서로 구별하여 오직 단 하나만 유일하게 식별할 수 있는 성질.**
     * *만족하는 키*: 슈퍼키(Super Key), 후보키(Candidate Key), 기본키(Primary Key), 대체키(Alternate Key).
  2. **`최소성 (Minimality) (지문 ②번)`**:
     * *개념*: 튜플을 유일하게 구별하기 위해 필요한 속성(컬럼)들의 집합 중 **더 이상 제거할 수 없는 꼭 필요한 최소한의 속성으로만 구성되는 성질.**
     * *만족하는 키*: 후보키(Candidate Key), 기본키(Primary Key), 대체키(Alternate Key). (※ 슈퍼키는 유일성은 만족하지만 최소성은 만족하지 못함!)

* **오답 파해: 원자성 (Atomicity) 과의 명확한 차이**:
  * **`원자성 (Atomicity)`**: 데이터베이스 **트랜잭션(Transaction)의 4대 특성(ACID)** 중 하나로서 "트랜잭션의 연산은 DB에 모두 반영되거나(All) 전혀 반영되지 않아야 함(Nothing)"을 뜻하는 성질입니다. 튜플 구별 제약조건인 **`유일성`** 과 완전히 다른 영역의 단어입니다.

* **★ [수험생 질답 심화 필기 노트] 식별자(Entity Identifier)의 4대 특성 완전 확장 족보**:
  1. **`유일성 (Uniqueness)`**: 엔티티 내의 모든 튜플을 유일하게 구별함. (본 문제 ①번)
  2. **`최소성 (Minimality)`**: 튜플을 유일하게 구별하는 최소한의 속성으로 구성됨. (본 문제 ②번)
  3. **`불변성 (Immutability)`**: 한번 식별자로 지정되면 식별자의 값은 변하지 않아야 함. (예: 학번, 주민번호 등)
  4. **`존재성 (Existence / Non-null)`**: 주 식별자는 반드시 데이터 값이 존재해야 함. (Null 값 절대 불가)

## 164. 소프트웨어 개발 - 요구사항 확인 및 객체지향 분석 (UML 클래스 다이어그램 간 관계 중 합성 Composition 및 집합 Aggregation)

### 문제
다음 UML 클래스 다이어그램 간 관계 설명의 괄호 ( ) 안에 들어갈 알맞은 용어를 쓰시오.
* ( ① ) 관계는 포함하는 사물의 변화가 포함되는 사물에게 영향을 미치는 관계를 표현하고 복합 관계라고도 한다.
* ( ① ) 관계는 포함되는 쪽(부분)에서 포함하는 쪽(전체)으로 속이 채워진 마름모를 연결하여 표현한다.
* ( ② ) 관계는 하나의 객체에 여러 개의 독립적인 객체들이 구성되는 관계로 하나의 사물이 다른 사물에 포함되어 있는 관계를 표현한다.
* ( ② ) 관계는 포함되는 쪽(부분)에서 포함하는 쪽(Whole)으로 속이 빈 마름모를 연결하여 표현한다.

### 정답 및 풀이 결과
* **작성한 답**:
  * **①: 포함** (Composition에 해당하는 번역 용어로 '포함', '합성', '복합' 모두 정답 인정)
  * **②: 일반화** (속이 빈 마름모 도형 표기를 is-a 상속 관계인 일반화 Generalization 속이 빈 화살표와 착각하여 오답)
* **모범 답안**:
  * **①: 합성** (또는 `복합`, `포함`, `Composition`, `합성 관계`)
  * **②: 집합** (또는 `Aggregation`, `집합 관계`)
* **결과**: **부분 정답 (①번 포함 정답 인정, ②번 일반화 오답)**

### 핵심 해설
이 문제는 소프트웨어 요구사항 분석 및 설계 영역에서 UML(Unified Modeling Language) 클래스 다이어그램의 **클래스 간 관계(Relationships) 중 전체(Whole)와 부분(Part)의 포함 관여성을 나타내는 합성 관계(Composition)와 집합 관계(Aggregation)의 개념 및 그래픽 표기 도형(마름모)**을 구별할 수 있는지 검증하는 기출 문제입니다.

* **합성(Composition) vs 집합(Aggregation) 킬러 비교 족보**:
  1. **`합성 / 복합 관계 (Composition) (지문 ①번)`**:
     * *개념*: 전체(Whole)와 부분(Part)의 **생명주기(Lifecycle)가 완전히 일치**하는 관계. 전체 객체가 소멸되면 부분 객체도 함께 소멸됨. (강한 결합)
     * *표기 도형*: 부분에서 전체 쪽으로 **`속이 채워진 검은 마름모 (◆)`** 로 연결함.
  2. **`집합 관계 (Aggregation) (지문 ②번)`**:
     * *개념*: 전체(Whole)와 부분(Part)이 **독립적인 생명주기**를 가짐. 전체 객체가 소멸되어도 부분 객체는 독립적으로 존재함. (약한 결합)
     * *표기 도형*: 부분에서 전체 쪽으로 **`속이 빈 흰색 마름모 (◇)`** 로 연결함.

* **UML 클래스 간 6대 관계 도형 표기법 완전 정리**:
  * **연관 (Association)**: 실선 (`──`) 또는 실선 화살표 (`──>`)
  * **의존 (Dependency)**: 점선 화살표 (`- - - >`)
  * **집합 (Aggregation)**: **속이 빈 마름모 (`◇──`)** (지문 ②번)
  * **합성 (Composition)**: **속이 채워진 마름모 (`◆──`)** (지문 ①번)
  * **일반화 (Generalization)**: **속이 빈 삼각 화살표 (`──▷`)** (상속 is-a 관계)
  * **실체화 (Realization)**: **점선 속이 빈 삼각 화살표 (`- - -▷`)** (인터페이스 구현)

## 165. 소프트웨어 개발 - 요구사항 확인 및 아키텍처 설계 (소프트웨어 아키텍처 패턴 중 파이프-필터 패턴 Pipe-Filter Pattern 및 브로커 패턴 Broker Pattern)

### 문제
다음 소프트웨어 아키텍처 패턴 설명의 괄호 ( ) 안에 들어갈 알맞은 용어를 쓰시오.
* ( ① )은/는 데이터 스트림을 생성하고 처리하는 시스템에서 사용 가능한 단방향 패턴으로 서브 시스템이 입력 데이터를 받아 처리하고, 결과를 다음 서브 시스템으로 넘겨주는 과정을 반복하는 방식이다.
* ( ② )은/는 분리된 컴포넌트들로 이루어진 분산 시스템에서 사용되고, 이 컴포넌트들은 원격 서비스 실행을 통해 상호 작용이 가능한 패턴으로 컴포넌트 간의 통신을 조정하는 역할을 수행하는 방식이다.

### 정답 및 풀이 결과
* **작성한 답**:
  * **①: 파이프 필터 패턴** (데이터 스트림 처리 아키텍처 패턴의 표준 명칭을 100% 바르게 도출해 정답)
  * **②: 제어 패턴** (분산 컴포넌트 간의 원격 통신 조정 및 중재 매개체인 브로커 패턴 Broker Pattern 명칭을 잘못 오인하여 오답)
* **모범 답안**:
  * **①: 파이프-필터 패턴** (또는 `파이프 필터`, `Pipe-Filter Pattern`, `파이프-필터`)
  * **②: 브로커 패턴** (또는 `Broker Pattern`, `브로커`)
* **결과**: **부분 정답 (①번 파이프 필터 패턴 정답, ②번 제어 패턴 오답)**

### 핵심 해설
이 문제는 소프트웨어 구조 설계(Architecture Design) 영역에서 시스템 전체의 뼈대를 형성하는 **주요 소프트웨어 아키텍처 패턴(Software Architecture Patterns) 4대 요소인 파이프-필터 패턴과 브로커 패턴의 핵심 명세 및 역할을 구별**할 수 있는지 검증하는 빈출 기출 문제입니다.

* **소프트웨어 아키텍처 패턴 4대 대표 유형 완전 해체 족보**:
  1. **`파이프-필터 패턴 (Pipe-Filter Pattern) (지문 ①번)`**:
     * *개념*: **데이터 스트림(Data Stream)** 을 처리하는 서브시스템 구축에 활용. 데이터 가공/변환을 담당하는 **필터(Filter)** 들을 데이터 전송 통로인 **파이프(Pipe)** 로 단방향 연결함.
     * *대표 사례*: UNIX / Linux CLI 쉘의 파이프 명령어(`cat file.txt | grep "error" | sort`).
  2. **`브로커 패턴 (Broker Pattern) (지문 ②번)`**:
     * *개념*: **분산 시스템(Distributed System)** 환경에서 클라이언트와 서버 사이의 통신 및 원격 서비스 실행을 **중재(Broker)** 해 주는 패턴. 서버는 자신의 서비스를 브로커에게 등록하고, 클라이언트는 브로커를 통해 서비스를 요청함.
     * *대표 사례*: CORBA, Java RMI.
  3. **`MVC 패턴 (Model-View-Controller Pattern)`**:
     * *개념*: 시스템을 비즈니스 로직(**Model**), 사용자 화면 UI(**View**), 흐름 제어/중재자(**Controller**)의 3가지 서브시스템으로 분리함.
  4. **`피어 투 피어 패턴 (Peer-to-Peer Pattern)`**:
     * *개념*: 각 컴포넌트(Peer)가 클라이언트이면서 동시에 서버 역할을 모두 수행하는 대등한 분산 아키텍처 패턴.

## 166. 서비스 운영 및 인프라 구축 - 응용 소프트웨어 기초 기술 (OSI 7계층 중 2계층 데이터 링크 계층 Data Link Layer 프로토콜 ㉢ PPP 및 ㉦ HDLC 식별)

### 문제
다음 보기 중에서 **데이터 링크 계층(Data Link Layer)**에 해당하는 프로토콜만을 모두 고르시오.
```text
㉠ UDP                 ㉡ IP                   ㉢ PPP 
㉣ HTTP               ㉤ ARP               ㉥ FTP
㉦ HDLC              ㉧ TCP
```

### 정답 및 풀이 결과
* **작성한 답**: `㉠, ㉤, ㉦` (4계층 전송 계층인 UDP와 3계층 네트워크 계층인 ARP를 2계층으로 잘못 포함시키고, 직렬 점대점 2계층 프로토콜인 ㉢ PPP를 선택에서 누락하여 오답)
* **모범 답안**: **`㉢, ㉦`** (또는 `㉢ PPP`, `㉦ HDLC`, `ㄷ, ㅅ`)
* **결과**: **오답 (계층별 프로토콜 식별 미흡: ㉢ PPP 누락, ㉠ UDP 및 ㉤ ARP 착각)**

### 핵심 해설
이 문제는 네트워크 아키텍처(OSI 7계층 및 TCP/IP 4계층) 영역에서 각 계층별로 작동하는 **대표적인 통신 프로토콜(Protocols)의 계층적 위치**를 정확하게 식별할 수 있는지 검증하는 기출 단답형 보기 선택 문제입니다.

* **보기 8개 프로토콜의 OSI 7계층 위치 완전 해체 족보**:
  1. **`㉢ PPP (Point-to-Point Protocol)`**: **`2계층 (데이터 링크 계층)`** ➔ **[정답]**
     * *역할*: 1:1 직렬 통신 회선에서 데이터 전송, 에러 검출, 에선 프레임을 형성하는 표준 데이터 링크 프로토콜.
  2. **`㉦ HDLC (High-level Data Link Control)`**: **`2계층 (데이터 링크 계층)`** ➔ **[정답]**
     * *역할*: 비트(Bit) 위주의 고속 표준 데이터 링크 제어 프로토콜. (명칭 자체에 Data Link 가 들어있음!)
  3. **`㉠ UDP (User Datagram Protocol)`**: `4계층 (전송 계층)`
     * *역할*: 비연결형, 신뢰성은 없으나 빠른 전송을 담당하는 4계층 프로토콜.
  4. **`㉡ IP (Internet Protocol)`**: `3계층 (네트워크 계층)`
     * *역할*: 패킷 주소 지정 및 라우팅(경로 설정)을 담당하는 3계층 메인 프로토콜.
  5. **`㉣ HTTP (HyperText Transfer Protocol)`**: `7계층 (응용 계층)`
     * *역할*: 웹 브라우저와 서버 간 문서 전송 7계층 프로토콜.
  6. **`㉤ ARP (Address Resolution Protocol)`**: `3계층 (네트워크 계층)`
     * *역할*: 논리적 IP 주소를 물리적 MAC 주소로 변환하는 3계층 연동 프로토콜.
  7. **`㉥ FTP (File Transfer Protocol)`**: `7계층 (응용 계층)`
     * *역할*: 파일 전송 담당 7계층 프로토콜.
  8. **`㉧ TCP (Transmission Control Protocol)`**: `4계층 (전송 계층)`
     * *역할*: 신뢰성 있는 연결형 전송 담당 4계층 프로토콜.

## 167. 서비스 운영 및 인프라 구축 - 응용 소프트웨어 기초 기술 (IPv4 vs IPv6 4대 규격 스펙 주소 비트 수 및 헤더 바이트 크기 비교)

### 문제
다음 IPv4 및 IPv6 프로토콜 설명의 괄호 ( ) 안에 들어갈 알맞은 숫자를 쓰시오.
* IPv4는 인터넷에서 사용되는 패킷 교환 네트워크상에서 데이터를 교환하기 위한 ( ① )비트 주소 체계를 갖는 네트워크 계층의 프로토콜이다.
* IPv4 헤더(Header)는 IP 패킷의 앞부분에서 주소 등 각종 제어정보를 담고 있으며 IPv4 헤더 사이즈는 옵션 미 지정시에 최소 ( ② )바이트 이상이다.
* IPv6는 ( ③ )비트의 주소 공간을 제공한다.
* IPv6 주소의 경우 일반적으로 16비트 단위로 나누어지며 각 16비트 블록은 다시 4자리 16진수로 변환되고 콜론으로 구분된다.
* IPv6 헤더 사이즈는 ( ④ )바이트의 고정된 길이로 되어 있다.

### 정답 및 풀이 결과
* **작성한 답**:
  * **①: 128** (IPv4 32비트와 IPv6 128비트 위치 착각 오답)
  * **②: 4** (IPv4 최소 헤더 20바이트 오답)

* **모범 답안**:
  * **①: 32**
  * **②: 20**
  * **③: 128**

  4. **IPv6 고정 헤더 크기 (지문 ④번)**: **`40 바이트 (Bytes)`** (확장 헤더를 제외한 기본 헤더가 40 Bytes 고정적)

## 168. 데이터베이스 구축 - SQL 응용 (다중 테이블 INNER JOIN 및 GROUP BY 부서별 AVG 평균 급여 산출 후 ORDER BY DESC 내림차순 정렬)

### 문제
다음 `[EMP]` 및 `[DEPT]` 테이블을 참고하여 아래 SQL 쿼리문의 실행 결과를 쓰시오.

**[EMP]**
| emp_no | emp_nm | dept_id | salary |
| :---: | :---: | :---: | :---: |
| 101 | 홍길동 | 10001 | 3700 |
| 101 | 김철수 | 10002 | 4200 |

```sql
SELECT d.dept_name, AVG(e.salary) AS avg_salary
```

     * 1위 (5000): **회계팀**

## 169. Python - 프로그래밍 언어 활용 (기본 매개변수 가변 객체 Mutable Default Argument 객체 메모리 공유 함정 및 len 연산)

### 문제
아래 파이썬 코드를 실행했을 때 출력되는 결과를 쓰시오. (단, 코드 블록 HTML 내에서 바르게 파이썬 코드로 디해싱하여 풀이한다.)
```python
def add_item(item, lst=[]):
    lst.append(item)
    return lst

add_item(1)
add_item(2)
result = add_item(3, [])

print(len(result) + len(add_item(4)))
```

### 정답 및 풀이 결과
* **작성한 답**: `2` (함수가 호출될 때마다 기본 인자 lst=[] 가 새로 초기화된다고 착각하여 len(result)=1 과 len(add_item(4))=1 의 합인 2로 잘못 계산하여 오답)
* **모범 답안**: **`4`**
* **결과**: **오답 (파이썬 기본 매개변수 가변 객체 Mutable Default Argument 지속 공유 스펙 트레이싱 실패)**

### 핵심 해설
이 문제는 파이썬 언어의 대표적인 초빈출 킬러 함정인 **"함수 정의 시 생성되는 가변 기본 매개변수(Mutable Default Argument, `lst=[]`)의 메모리 객체 지속 공유 특징"**을 정밀하게 트레이싱할 수 있는지 검증하는 고난도 문제입니다.

* **단계별 라인 트레이싱 및 메모리 공유 분석**:
  1. **`def add_item(item, lst=[])` 정의 시점 (★ 파이썬 핵심 스펙)**:
     * 파이썬은 함수가 `def`로 정의될 때 기본 매개변수인 `lst=[]` 객체를 **딱 1번만 생성**하고, 이후 인자 없이 호출될 때 이 동일한 리스트 메모리 객체를 계속 재사용(공유)합니다!
  2. **1번째 호출 `add_item(1)`**:
     * 기본 리스트에 `1` 추가 $\rightarrow$ 기본 리스트: `[1]`
  3. **2번째 호출 `add_item(2)`**:
     * 1회차의 기본 리스트 `[1]` 에 `2` 추가 $\rightarrow$ 기본 리스트: **`[1, 2]`**
  4. **3번째 호출 `result = add_item(3, [])`**:
     * 명시적으로 새 리스트 인자 **`[]`** 가 전달됨! (기본 리스트를 쓰지 않음)
     * 새 리스트에 `3` 추가 $\rightarrow$ `result` 는 **`[3]`** 이 됨. (`len(result)` = **`1`**)
     * (※ 기존 기본 리스트는 여전히 `[1, 2]` 상태로 유지됨!)
  5. **4번째 호출 `add_item(4)` (`print` 문 내부)**:
     * 다시 인자가 없어 기존 기본 리스트 `[1, 2]` 를 재사용함!
     * `4` 추가 $\rightarrow$ 반환값: **`[1, 2, 4]`** (`len(add_item(4))` = **`3`**)
  6. **`print(len(result) + len(add_item(4)))` 연산**:
     * `len(result)` [1] + `len(add_item(4))` [3] = $1 + 3 =$ **`4`**

## 170. Python - 프로그래밍 언어 활용 (집합 Set 자료형 4대 연산자 | 합집합 & 교집합 - 차집합 ^ 대칭차집합 출력 트레이싱)

```python
x = {1, 2}
```

## 171. Python - 프로그래밍 언어 활용 (튜플 언패킹 루프 for a,b,c in terms 및 거듭제곱 연산자 ** 지수법칙 0제곱 1 연산 트레이싱)

### 문제
아래 파이썬 코드를 실행했을 때 출력되는 결과를 쓰시오. (단, 코드 블록 HTML 내에서 바르게 파이썬 코드로 디해싱하여 풀이한다.)
```python
data = [
    (3, 2, 1),    
    (2, 1, 2),    
    (-5, 1, 1),   
    (7, 0, 0)     
]
x = 2
y = 3

def soojebi(terms, x, y):
  result = 0
  for a, b, c in terms:
    term_value = a * (x ** b) * (y ** c)
    result += term_value
  return result

result = soojebi(data, x, y)
print(f"{result}")
```

### 정답 및 풀이 결과
* **작성한 답**: `49` (튜플 요소 4개의 a * (x**b) * (y**c) 연산과 2**0=1, 3**0=1 지수법칙을 100% 바르게 대입 트레이싱하여 정답 도출 성공)
* **모범 답안**: **`49`**
* **결과**: **정답 (100% 매칭)**

### 핵심 해설
이 문제는 파이썬의 **튜플 다중 언패킹(Unpacking) 루프(`for a, b, c in terms`)**와, **거듭제곱 연산자(`**`)의 연산 우선순위**, 그리고 **모든 수의 0제곱이 1이 되는 지수 법칙**을 정확하게 계산할 수 있는지 평가하는 문제입니다.

* **단계별 4개 튜플 트레이싱 연산 과정**:
  * 공통 조건: `x = 2`, `y = 3`
  * 계산식: `term_value = a * (2 ** b) * (3 ** c)`
  1. **`(a, b, c) = (3, 2, 1)`**:
     * $3 \times (2^2) \times (3^1) = 3 \times 4 \times 3 =$ **`36`**
     * `result` = $0 + 36 =$ **`36`**
  2. **`(a, b, c) = (2, 1, 2)`**:
     * $2 \times (2^1) \times (3^2) = 2 \times 2 \times 9 =$ **`36`**
     * `result` = $36 + 36 =$ **`72`**
  3. **`(a, b, c) = (-5, 1, 1)`**:
     * $-5 \times (2^1) \times (3^1) = -5 \times 2 \times 3 =$ **`-30`**
     * `result` = $72 + (-30) =$ **`42`**
  4. **`(a, b, c) = (7, 0, 0)` (★ 지수법칙 킬러 튜플)**:
     * $2^0 = 1$, $3^0 = 1$ 이므로 $\rightarrow 7 \times 1 \times 1 =$ **`7`**
     * `result` = $42 + 7 =$ **`49`**
  5. **최종 출력**: `print(f"{result}")` ➔ **`49`**

## 172. Java - 프로그래밍 언어 활용 (다형성 오버라이딩 B.speak 동적 바인딩 및 throw new ArithmeticException 예외 캐치 try-catch-finally 트레이싱)

### 문제
아래 자바 코드를 실행했을 때 출력되는 결과를 쓰시오. (단, 수험생이 기술한 복원 코드에 기초하여 정밀 풀이한다.)
```java
class A {
  void speak() throws RuntimeException {
    System.out.print("A");
    throw new NullPointerException();
  }
}

class B extends A {
  @Override
  void speak() throws ArithmeticException {
    System.out.print("D");
    throw new ArithmeticException();
  }
}

public class Soojebi {
  public static void main(String[] args) {
    A a = new B();
    try {
      a.speak();
    }
    catch (ArithmeticException e) {
      System.out.print("E1");
    }
    catch (NullPointerException e) {
      System.out.print("E2");
    }
    catch (Exception e) {
      System.out.print("E3");
    }
    finally {
      System.out.print("F");
    }
  }
}
```

### 정답 및 풀이 결과
* **작성한 답**: `DE1F` (A a = new B() 의 오버라이딩 동적 바인딩으로 B.speak()가 실행됨을 간파하고, throw new ArithmeticException() 발생에 따른 catch(ArithmeticException) E1 출력 및 finally F 선행 출력을 정확히 추적하여 정답 도출 성공)
* **모범 답안**: **`DE1F`**
* **결과**: **정답 (100% 매칭)**

### 핵심 해설
이 문제는 자바의 핵심인 **오버라이딩(Overriding) 다형성에 의한 동적 바인딩 메커니즘**과, **`throw new` 키워드로 발생시킨 예외의 `catch` 조건 매칭**, 그리고 **`finally` 블록의 무조건 실행 보장성**을 정밀하게 조합하여 트레이싱할 수 있는지 검증하는 빈출 킬러 종합 문제입니다.

* **단계별 라인 트레이싱 분석**:
  1. **`A a = new B();`**:
     * 부모 타입 참조 변수 `a`에 실제 인스턴스 `new B()` 가 생성됩니다.
  2. **`a.speak();` (동적 바인딩 가동!)**:
     * 부모 `A` 의 메서드가 아닌, 실제 인스턴스인 **자식 `B` 클래스의 재정의된 `speak()` 메서드가 가동**됩니다!
     * `B`의 `speak()` 본문 실행:
       * `System.out.print("D");` ➔ **`"D"`** 1차 인쇄!
       * `throw new ArithmeticException();` ➔ **`ArithmeticException` 예외 객체를 강제로 발생시키며 탈출!**
  3. **`catch` 블록 매칭**:
     * 발생된 예외: **`ArithmeticException`**
     * 첫 번째 `catch (ArithmeticException e)` 조건과 100% 일치합니다.
     * 실행: `System.out.print("E1");` ➔ **`"E1"`** 2차 인쇄!
     * (이후의 `NullPointerException`, `Exception` 캐치 블록은 실행되지 않고 넘어갑니다.)
  4. **`finally` 블록 실행 (무조건 100% 가동)**:
     * `System.out.print("F");` ➔ **`"F"`** 3차 인쇄!
  5. **최종 출력 결과**: `"D"` + `"E1"` + `"F"` = **`DE1F`**

## 173. Java - 프로그래밍 언어 활용 (JVM static 변수 초기화 fn() 및 static 초기화 블록 순차 가동과 main x++ 증가 트레이싱)

### 문제
아래 자바 코드를 실행했을 때 출력되는 결과를 쓰시오. (단, 수험생이 기술한 복원 코드 및 풀이 과정에 기초하여 정밀 해설한다.)
```java
public class Soojebi {
  static int x = fn();
  static int y = fn();
  static int fn() {
    System.out.print("A");
    return 1;
  }
  static {
	System.out.print("B");
	x++;
  }
  static {
	System.out.print("C");
	x++;
  }
  public static void main(String[] args) {
    System.out.print(x);
  }
}
```

### 정답 및 풀이 결과
* **작성한 답**: `AABC3` (프로그램의 엔트리 포인트인 main()을 먼저 확인하여 출력이 x 임을 파악하고, 클래스 로딩 시 static 변수 선언 fn() 두 번(AA) 및 static { } 블록 B, C가 선행 실행된 후 x의 최종값 3이 인쇄되는 타임라인을 100% 바르게 분석하여 정답 도출 성공)
* **모범 답안**: **`AABC3`**
* **결과**: **정답 (100% 매칭)**

### 핵심 해설
이 문제는 자바의 클래스 로딩 메커니즘인 **`static` 정적 필드 초기화 방식과 `static { }` 초기화 블록의 위에서부터 아래로의 순차 실행 흐름**, 그리고 **`main` 메서드 가동 시점 이전의 로딩 타이밍**을 바르게 이해하고 있는지 평가하는 고난도 기출 문제입니다.

* **JVM 클래스 로딩 및 `static` 순차 실행 트레이싱**:
  1. **`Soojebi` 클래스 로딩 (`main` 실행 전 선행 가동)**:
     * 클래스가 JVM 메모리에 탑재될 때, 모든 `static` 변수 및 `static { }` 블록이 선언된 순서(위 $\rightarrow$ 아래)대로 실행됩니다.
  2. **`static int x = fn();` 실행**:
     * `fn()` 메서드가 호출되어 **`"A"`** 1차 인쇄.
     * `x` 의 초기값은 `1` 이 됨.
  3. **`static int y = fn();` 실행**:
     * `fn()` 메서드가 또 호출되어 **`"A"`** 2차 인쇄. (누적 출력: `"AA"`)
     * `y` 의 초기값은 `1` 이 됨.
  4. **첫 번째 `static { }` 블록 가동**:
     * **`"B"`** 3차 인쇄. (누적 출력: `"AAB"`)
     * `x++` 실행 $\rightarrow$ `x` 는 1에서 **`2`** 로 증가!
  5. **두 번째 `static { }` 블록 가동**:
     * **`"C"`** 4차 인쇄. (누적 출력: `"AABC"`)
     * `x++` 실행 $\rightarrow$ `x` 는 2에서 **`3`** 으로 증가!
  6. **`main(String[] args)` 메서드 드디어 실행**:
     * `System.out.print(x);` ➔ 현재 `x`의 최종 값인 **`3`** 이 5차 인쇄.
  7. **최종 출력 문자열**: **`AABC3`**

## 174. Java - 프로그래밍 언어 활용 (배열 매개변수 참조 전달 Call by Reference 및 메서드 내 new 배열 재할당 참조 변수 Scope 독립성 트레이싱)

### 문제
아래 자바 코드를 실행했을 때 출력되는 결과를 쓰시오. (단, 수험생이 기재한 복원 코드 및 상세 트레이싱 풀이에 기초하여 정밀 해설한다.)
```java
public class Soojebi {
  static void func(int[] a) {
    a[0]++;
    a = new int[]{5, 6, 7};
    a[0]++;
  }
  public static void main(String[] args) {
    int[] arr = {1, 2, 3};
    func(arr);
    System.out.println(arr[0]);
  }
}
```

### 정답 및 풀이 결과
* **작성한 답**: `2` (func(arr) 호출 시 a[0]++ 에 의해 원본 배열 주소의 arr[0]이 2가 된 후, func 내부에서 a = new int[]{5,6,7} 로 참조 주소가 바뀌어도 main의 원본 arr에는 아무 영향을 주지 않음을 정밀 파악하여 정답 도출 성공)
* **모범 답안**: **`2`**
* **결과**: **정답 (100% 매칭)**

### 핵심 해설
이 문제는 자바의 **배열 참조 전달(Call by Value of Reference) 메커니즘**과, **메서드 내부에서 `new` 키워드로 배열을 재할당(`a = new int[]`)했을 때 참조 주소가 끊어지고 독립적인 유효범위(Scope)를 가지는 기작**을 트레이싱할 수 있는지 평가하는 초빈출 킬러 문제입니다.

* **단계별 라인 트레이싱 및 메모리 주소 분석**:
  1. **`int[] arr = {1, 2, 3};` (`main` 진입)**:
     * 메인 메모리에 배열 `{1, 2, 3}` 이 생성됩니다. (주소: `@100`)
  2. **`func(arr);` 메서드 호출**:
     * 매개변수 `a`에 원본 배열 주소 `@100` 이 전달됩니다.
  3. **`a[0]++;` (원본 배열 변경)**:
     * 주소 `@100` 의 0번 인덱스 원소가 `1`에서 **`2`** 로 변경됩니다! (현재 원본 배열: `{2, 2, 3}`)
  4. **`a = new int[]{5, 6, 7};` (★ 킬러 함정 구간: 참조 주소 교체)**:
     * 지역 변수 `a`에 새 배열 `{5, 6, 7}` (주소: `@200`) 의 주소가 대입됩니다.
     * **이제 매개변수 `a`는 원본 배열 `@100` 과 인연이 끊어지고 새 주소 `@200` 을 가리킵니다!**
  5. **`a[0]++;` (새 배열 연산)**:
     * 새 주소 `@200` 의 0번 인덱스 원소가 `5`에서 `6`으로 증가합니다. (※ 원본 배열 `@100` 에는 1도 영향을 주지 않음!)
  6. **`func` 메서드 종료 후 `main` 으로 복귀**:
     * `System.out.println(arr[0]);` ➔ 원본 배열 `@100` 의 0번 인덱스 값인 **`2`** 가 출력됩니다!

## 175. C언어 - 프로그래밍 언어 활용 (자릿수 분리 연산 num%10 및 num/=10 함수 soojebi 의 0,1,2 포함 여부 1~20 배열 트레이싱 xx0000000xxxxxxxxxxx)

### 문제
아래 C언어 코드를 실행했을 때 출력되는 결과를 쓰시오. (단, 코드 블록 HTML 내에서 바르게 C언어 코드로 디해싱하여 풀이한다.)
```c
#include <stdio.h>

int soojebi(int num) {
  while (num > 0) {
    int digit = num % 10;
    if (digit == 0 || digit == 1 || digit == 2)
      return 1;
    num /= 10;
  }
  return 0;
}

int main() {
  int i;
  int arr[20];
  int size = sizeof(arr)/sizeof(arr[0]);    
  for (i = 0; i < size; i++) 
    arr[i] = i + 1;    
  for (i = 0; i < size; i++) {
    if (soojebi(arr[i])) printf("x");
    else printf("0");      
  }
  return 0;
}
```

### 정답 및 풀이 결과
* **작성한 답**: `xx0000000xxxxxxxxxxx` (soojebi 함수가 10진수 자릿수 중 0, 1, 2가 하나라도 포함되면 1(참)을 반환한다는 기작을 간파하고, 1~2에서 xx, 3~9에서 0 7개, 10~20에서 십의 자리가 1과 2로 포함되는 x 11개를 정확히 카운트하여 100% 정답 도출 성공)
* **모범 답안**: **`xx0000000xxxxxxxxxxx`**
* **결과**: **정답 (100% 매칭)**

### 핵심 해설
이 문제는 C언어의 **자릿수 추출 공식(`num % 10` 및 `num /= 10`)**과, **함수 반환값에 따른 조건문 실행(`if (soojebi(arr[i]))`)**, 그리고 **10~20 구간의 십의 자리(1과 2) 포함 여부를 2차 검증하는 트레이싱 능력**을 종합 평가하는 킬러 문제입니다.

* **단계별 1~20 숫자 20개 트레이싱 및 출력 카운트**:
  1. `arr` 배열 요소: 1부터 20까지 정수가 순서대로 탑재됨 (`arr[i] = i + 1`)
  2. **1 ~ 2 (2개)**: 자릿수가 1과 2이므로 `soojebi()` 는 1(참) 반환 ➔ **`xx`**
  3. **3 ~ 9 (7개)**: 자릿수 3, 4, 5, 6, 7, 8, 9 에는 0, 1, 2가 단 하나도 없으므로 0(거짓) 반환 ➔ **`0000000`**
  4. **10 ~ 20 (11개) (★ 킬러 구간)**:
     * 10 ~ 19: 십의 자리가 모두 **`1`** 이므로 자릿수 분리 루프에 의해 모두 1(참) 반환!
     * 20: 십의 자리가 **`2`**, 일의 자리가 **`0`** 이므로 1(참) 반환!
     * 따라서 10부터 20까지 총 11개 숫자 모두 `soojebi()` 가 1(참)을 리턴함 ➔ **`xxxxxxxxxxx`**
  5. **최종 출력 문자열 합체**: **`xx0000000xxxxxxxxxxx`**
  6. **★ [수험생의 본질 통찰 노트] 함수 soojebi(num)의 정체**:
     * 본 문제는 결국 주어진 10진수 숫자 내부의 일의 자리뿐만 아니라 십의 자리까지 자릿수들을 훑어서 **'0', '1', '2' 가 단 하나라도 포함되어 있는가**를 판별하는 문제였습니다.
     * 시험장에서 순간적인 착각이 들더라도 self-correcting(자정 능력)을 발휘하여 자릿수 포함 여부의 본질로 돌아와 정답을 도출해 내는 검증 프로세스의 중요성을 확인합니다.

## 176. C언어 - 프로그래밍 언어 활용 (문자열 단어 수 word 및 공백 포함 전체 글자 수 character 카운팅 알고리즘 트레이싱)

### 문제
아래 C언어 코드를 실행했을 때 출력되는 결과를 쓰시오. (단, 수험생이 기재한 복원 코드 및 풀이 과정에 기초하여 정밀 해설한다.)
```c
#include <stdio.h>

int soojebi(char *buffer, int *pos) {
  return buffer[(*pos)++];
}

int fn(char c) {
  if(c == ' ' || c == '\t' || c == '\n'){
    return 1;
  }
  else{
    return 0;
  }
}

int main() {
  int c;
  int in_word = 0;
  int word = 0; 
  int character = 0;
  char buffer[] = "hello soojebi world";
  int pos = 0;  
  
  while ((c = soojebi(buffer, &pos)) != '\0') {
    character++;
    if (fn(c)) {
      in_word = 0;
    } else if (!in_word) {
      in_word = 1;
      word++;
    }
  }

  printf("word: %d, character: %d\n", word, character);
 
  return 0;
}
```

### 정답 및 풀이 결과
* **작성한 답**: `word: 3, character: 18` (in_word 상태 플래그 기반 단어 개수 word=3 은 100% 바르게 계산하였으나, 공백 2개가 포함된 전체 문자열 "hello soojebi world" 의 총 글자 수가 19자임을 카운팅하는 과정에서 1자 오차가 발생하여 오답)
* **모범 답안**: **`word: 3, character: 19`**
* **결과**: **부분 정답 (word: 3 정답, character: 19 1자 오차 오답)**

### 핵심 해설
이 문제는 C언어의 **포인터 간접 참조 및 후위 증감 연산자(`buffer[(*pos)++]`)를 활용한 문자열 순회**와, **공백 문자 기준 단어 경계 판단 알고리즘(`in_word` 플래그 관리)**, 그리고 **공백 포함 전체 글자 수 카운팅 연산**을 정확히 계산할 수 있는지 검증하는 기출 문제입니다.

* **단계별 2대 변수 카운팅 정밀 해체**:
  1. **`word` (단어 개수 카운팅)**:
     * `fn(c)` 가 공백(`' '`)을 만나면 `in_word = 0` 으로 초기화.
     * 공백이 아닌 알파벳을 만나는 순간 `!in_word` 가 참이 되어 `word++` 1 증가.
     * 단어 구분: `hello` (1번째) ➔ `soojebi` (2번째) ➔ `world` (3번째)
     * **`word = 3`**
  2. **`character` (총 글자 수 카운팅)**:
     * `while` 루프가 `'\0'` (NULL 문자)를 만나기 전까지 매 문자를 읽을 때마다 무조건 `character++` 실행.
     * 대상 문자열: `"hello soojebi world"`
     * 글자 수 구성:
       * `hello` (5자)
       * `' '` (공백 1자)
       * `soojebi` (7자)
       * `' '` (공백 1자)
       * `world` (5자)
     * **총 글자 수 (`character`)** = $5 + 1 + 7 + 1 + 5 =$ **`19`**
  3. **★ [수험생 깨달음 노트] 후위 증감 연산자 `buffer[(*pos)++]` 의 내부 3단계 대입/증가 분해 원리**:
     * 수험생 질답 통찰: `buffer[(*pos)++]` 연산은 C언어 내부적으로 아래와 같이 **'현재 위치의 문자를 먼저 대입/할당받은 후 pos 가 1 증가'**하는 3단계 순서로 동작합니다.
     ```c
     // buffer[(*pos)++] 내부 어셈블리 3단계 분해
     char temp = buffer[*pos];  // 1. 현재 pos 위치의 문자(예: pos=0 이면 'h')를 먼저 할당받음!
     (*pos)++;                  // 2. 할당받은 그 0.0001초 직후 pos 값이 0에서 1로 증가!
     return temp;               // 3. 1번에서 할당받아둔 문자를 리턴하여 c = temp 로 넘김!
     ```
     * 따라서 `while` 루프가 돌 때마다 `c = buffer[pos]` 로 0번 문자인 `'h'` 부터 18번 문자인 `'d'` 까지 순서대로 안전하게 읽히고, 매 루프 진입 직후 `character++` 가 실행되어 총 **`19`**자 카운팅이 정확하게 도출됨을 완전히 이해함.

## 177. C언어 - 자료구조 & 프로그래밍 언어 활용 (선형 큐 Queue 구현 enq 삽입 및 deq 추출 후위 증감 연산자 data[front++] 선입선출 FIFO 트레이싱 sum=18)

### 문제
아래 C언어 코드를 실행했을 때 출력되는 결과를 쓰시오. (단, 수험생이 기재한 복원 코드 및 상세 트레이싱 풀이에 기초하여 정밀 해설한다.)
```c
#include <stdio.h>

int data[10];
int front, rear;

void enq(int val) {
  data[rear] = val;
  rear++;
}

int deq() {
  return data[front++];
}

int main(){
  int sum = 0;
  enq(10);
  enq(8);
  sum += deq();
  enq(2);
  sum += deq();
  printf("%d", sum);
  return 0;
}
```

### 정답 및 풀이 결과
* **작성한 답**: `18` (전역변수 front, rear의 0 초기화 및 큐(Queue)의 선입선출(FIFO) 특성을 정확히 파악하고, deq() 호출 시 data[front++] 후위 연산자 반환에 따라 10과 8이 차례대로 더해져 sum=18 이 되는 타임라인을 100% 바르게 추적하여 정답 도출 성공)
* **모범 답안**: **`18`**
* **결과**: **정답 (100% 매칭)**

### 핵심 해설
이 문제는 C언어로 구현된 **선형 큐(Queue) 자료구조의 삽입(`enq`)과 삭제(`deq`) 메커니즘**, 그리고 **후위 증감 연산자(`data[front++]`)의 '선 반환 후 증가' 특성**을 정확히 이해하고 트레이싱할 수 있는지 검증하는 기출 문제입니다.

* **단계별 큐(Queue) 메모리 및 `sum` 트레이싱 과정**:
  1. **전역 변수 초기화**: `front = 0`, `rear = 0` (C언어 전역 변수는 0으로 자동 초기화됨)
  2. **`enq(10)`**: `data[0] = 10`, `rear` 가 0에서 **`1`** 로 증가.
  3. **`enq(8)`**: `data[1] = 8`, `rear` 가 1에서 **`2`** 로 증가.
  4. **`sum += deq()` (1차 추출)**:
     * `deq()` 호출: `data[front]` 즉 `data[0]` 의 값인 **`10`** 이 먼저 반환되고, `front` 가 0에서 **`1`** 로 증가.
     * `sum` = $0 + 10 =$ **`10`**
  5. **`enq(2)`**: `data[2] = 2`, `rear` 가 2에서 **`3`** 으로 증가.
  6. **`sum += deq()` (2차 추출)**:
     * `deq()` 호출: `data[front]` 즉 `data[1]` 의 값인 **`8`** 이 먼저 반환되고, `front` 가 1에서 **`2`** 로 증가.
     * `sum` = $10 + 8 =$ **`18`**
  7. **최종 출력**: `printf("%d", sum)` ➔ **`18`**

## 178. 소프트웨어 공학 - 애플리케이션 테스트 관리 (블랙박스 테스트 기법 - 경계값 분석 Boundary Value Analysis -1, 0, 50, 51 선정)

## 180. 정보보안 - 소프트웨어 개발 보안 구축 (웹 브라우저 탭 피싱 공격 - target=_blank window.opener 백그라운드 탭 변조 탭내빙 Tabnabbing)

### 문제
* HTML 문서 내에서 링크(target이 _blank인 태그)를 클릭했을 때 새롭게 열린 탭(페이지)에서 기존의 문서인 location을 피싱 사이트로 변경해 정보를 탈취하는 공격 기술을 말한다.
* 이 공격은 메일이나 SNS와 같은 오픈 커뮤니티에서 사용될 수 있다.
* 사용자의 클릭을 유도하여 웹 브라우저의 탭을 피싱 사이트로 이동시키는 기존의 피싱 기법과 달리 사용자가 페이지에서 아무런 행위를 하지 않아도 사용자의 눈을 피해 열린 탭 중 하나를 피싱 페이지로 로드한다.

보기:
㉠ Tabphishing    ㉡ Tabnabbing        ㉢ Tabsmishing  
㉣ TabSnooping  ㉤ Tabjacking          ㉥ Tabshadowing 
㉦ Tabhijacking   ㉧ Tabspoofing

### 정답 및 풀이 결과
* **작성한 답**: `㉥ Tabshadowing` (그림자처럼 몰래 숨어서 탭을 바꾼다는 뉘앙스에서 Tabshadowing 을 짐작하여 선택했으나 정식 기술 용어 미인지로 오답)
* **모범 답안**: **`㉡ Tabnabbing`** *(또는 `탭내빙`)*
* **결과**: **오답 (정식 기술 명칭 ㉡ Tabnabbing 탭내빙 미선택)**

### 핵심 해설
이 문제는 웹 브라우저의 `target="_blank"` 속성 링크를 악용하여 백그라운드에 열려있는 이전 탭의 URL을 몰래 피싱 사이트로 바꿔치기하는 **탭내빙(Tabnabbing)** 공격 기법을 식별할 수 있는지 평가하는 신유형 기출 문제입니다.

* **탭내빙 (Tabnabbing) 공격 원리와 작동 메커니즘**:
  1. **Nabbing (낚아채다, 훔치다)**: 브라우저 탭(Tab)을 사용자 모르게 낚아채어 피싱 사이트로 전환시키는 기술입니다.
  2. **`target="_blank"` 의 보안 허점**:
     * HTML에서 `<a href="..." target="_blank">` 링크를 클릭하면 새 탭이 열리며, 이 새 탭의 JavaScript는 `window.opener` 객체를 통해 자신을 연 부모 탭(이전 탭)의 주소(`window.opener.location`)를 제어할 수 있는 권한을 가집니다!
  3. **공격 시나리오**:
     * 사용자가 새 탭에 몰두하는 동안, 해커는 `window.opener.location = "가짜_네이버_로그인_페이지"` 코드를 실행하여 부모 탭을 피싱 사이트로 둔갑시킵니다.
     * 사용자가 기존 탭으로 돌아왔을 때 세션이 끊긴 것으로 착각하고 다시 로그인 비밀번호를 입력하게 만듭니다.
* **★ 핵심 방어 대책 (시큐어 코딩 족보)**:
  * HTML 링크 태그에 **`rel="noopener noreferrer"`** 속성을 부여하여 새 탭이 부모 탭(`window.opener`)의 참조 주소에 접근하지 못하도록 차단해야 합니다!

## 182. 데이터베이스 구축 - SQL 응용 (SQL 그룹 집계 함수 AVG 의 NULL 행 제외 무시 연산 및 0 값 카운트 산술 덧셈 쿼리 연산)

### 문제
다음 `[TB]` 테이블을 참고하여 아래 SQL 쿼리문의 실행 결과를 쓰시오.

**[TB]**
| COL1 | COL2 |
| :---: | :---: |
| 1 | 0 |
| 3 | NULL |
| NULL | 4 |

**[SQL]**
```sql
SELECT AVG(COL1) + AVG(COL2) FROM TB;
```

### 정답 및 풀이 결과
* **작성한 답**: `4` (AVG 집계 함수가 NULL 행을 무시하고 유효한 데이터 개수로만 평균을 산출한다는 스펙을 파악하고, AVG(COL1)=(1+3)/2=2, AVG(COL2)=(0+4)/2=2 의 합 4를 정확히 계산하여 100% 정답 도출 성공)
* **모범 답안**: **`4`**
* **결과**: **정답 (100% 매칭)**

### 핵심 해설
이 문제는 RDB의 **SQL 그룹 집계 함수(`AVG`, `SUM`, `COUNT`, `MAX`, `MIN` 등)가 `NULL` 값을 연산 대상에서 무시(제외)하는 핵심 메커니즘**과, **`0` 은 엄연한 숫자로 카운트에 포함되는 특성**을 결합하여 트레이싱할 수 있는지 검증하는 빈출 킬러 쿼리 문제입니다.

* **단계별 `AVG()` 그룹 함수 계산 트레이싱**:
  1. **`AVG(COL1)` 연산**:
     * `COL1` 데이터: `1`, `3`, `NULL`
     * **`NULL` 무시**: 유효한 행은 `1`과 `3` (총 2개 행!)
     * 평균: $(1 + 3) / 2 = 4 / 2 =$ **`2`**
  2. **`AVG(COL2)` 연산**:
     * `COL2` 데이터: `0`, `NULL`, `4`
     * **`NULL` 무시**: 유효한 행은 `0`과 `4` (총 2개 행! ※ 0은 숫자 값이므로 평균 개수에 포함됨!)
     * 평균: $(0 + 4) / 2 = 4 / 2 =$ **`2`**
  3. **`SELECT AVG(COL1) + AVG(COL2)` 최종 연산**:
     * $2 + 2 =$ **`4`**

## 183. 인터페이스 구현 - 인터페이스 설계 (웹 자원 상태 전송 아키텍처 - 로이 필딩 Roy Fielding 창안 REST Representational State Transfer)

### 문제
* ( ① )은/는 웹과 같은 분산 하이퍼미디어 환경에서 자원의 존재 상태 정보를 표준화된 메서드로 주고받는 웹 아키텍처이다. 
* ( ① )은/는 웹의 창시자중의 한 사람인 Roy Fielding의 논문에 의해서 소개되었다.

### 정답 및 풀이 결과
* **작성한 답**: `반응형 웹` (웹 분산 하이퍼미디어 환경이라는 지문에 착안하여 화면 레이아웃 기술인 반응형 웹을 연상하여 적었으나, 로이 필딩이 발표한 웹 자원 전송 아키텍처 명칭 미인식으로 오답)
* **모범 답안**: **`REST`** *(또는 `Representational State Transfer`)*
* **결과**: **오답 (정식 아키텍처 명칭 REST 미기재)**

### 핵심 해설
이 문제는 웹 서비스 인터페이스 설계의 핵심 표준 아키텍처인 **REST(Representational State Transfer)**의 정의와 창시자인 **로이 필딩(Roy Fielding)** 박사의 학술적 기원을 바르게 이해하고 있는지 평가하는 빈출 용어 기입형 문제입니다.

* **REST (Representational State Transfer) 핵심 3대 개념**:
  1. **자원 (Resource - URI)**: 클라이언트가 요청하는 모든 웹 요소(데이터, 이미지, 텍스트 등)를 고유한 **URI**로 표현합니다.
  2. **행위 (Verb - HTTP Method)**: 자원에 대한 조작 행위를 **HTTP 4대 메서드**로 명확히 표현합니다:
     * **`GET`**: 자원 조회 / 읽기 (Read)
     * **`POST`**: 자원 생성 (Create)
     * **`PUT`**: 자원 수정 / 갱신 (Update)
     * **`DELETE`**: 자원 삭제 (Delete)
  3. **표현 (Representation - Payload)**: JSON, XML 등의 형태 데이터 객체 형태로 자원의 상태를 전달합니다.
* **★ 킬러 힌트 공식 (시험 암기 족보)**:
  * 지문에 **`로이 필딩 (Roy Fielding)`** 또는 **`분산 하이퍼미디어 자원 상태 정보 전송`** 키워드가 나오면 100% **`REST`** 가 정답입니다!
* **★ [수험생의 본질 통찰 노트] RESTful 개발의 정체**:
  * 수험생 질답 통찰: 개발 현장 및 실무에서 "RESTful하게 개발한다"는 말은 결국 데이터베이스의 **`CRUD (Create, Read, Update, Delete)` 연산을 HTTP 4대 메서드(`POST`, `GET`, `PUT`, `DELETE`)로 규격화하여 명확하게 처리할 수 있는가**를 의미함을 완벽히 이해함.

두 릴레이션 R과 S에 동시에 속하는 모든 튜플을 포함하는 결과 릴레이션을 생성하는 일반 집합 연산자는 무엇인지 쓰시오.

### 정답 및 풀이 결과
* **작성한 답**: `and 연산자 묻는건가? &&` (두 릴레이션에 동시에 속해야 한다는 논리곱 AND 교집합 조건의 본질은 정확히 간파하였으나, DB 관계 대수 정식 집합 연산자 명칭인 교집합(∩) 미기재로 오답)
* **모범 답안**: **`교집합`** *(또는 `∩`, `Intersection`)*
* **결과**: **오답 (정식 관계 대수 집합 연산자 명칭 교집합 ∩ 미기재)**

### 핵심 해설
이 문제는 데이터베이스 관계 데이터 모델(Relational Data Model)의 **관계 대수(Relational Algebra) 4대 일반 집합 연산자**의 정의와 개념을 정확히 알고 있는지 평가하는 빈출 용어 기입형 문제입니다.

* **관계 대수 4대 일반 집합 연산자 완전 해체 족보**:
  1. **`합집합 (Union, ∪)`**: 릴레이션 R이나 S 중 **어느 한쪽에라도 속하는** 모든 튜플의 집합.
  2. **`교집합 (Intersection, ∩)` (★ 본 문제)**: 릴레이션 R과 S에 **동시에(공통으로) 속하는** 튜플의 집합.
  3. **`차집합 (Difference, －)`**: 릴레이션 R에는 속하고 **S에는 속하지 않는** 튜플의 집합.
  4. **`카티션 프로덕트 (Cartesian Product, ×)`**: R의 각 튜플과 S의 각 튜플을 **모두 결합한 직교 곱** 튜플의 집합 (결과 튜플 수 = $R \times S$, 속성 수 = $R + S$).

## 185. 데이터베이스 구축 - 응용 소프트웨어 기초 기술 (개념적 데이터 모델링 - 엔터티 및 관계 도식화 ERD 개체-관계 다이어그램 Entity-Relationship Diagram)

### 문제
각 업무 분석에서 도출된 엔터티 간의 관계를 이해하기 쉽게 도식화된 다이어그램은 무엇인지 쓰시오.

### 정답 및 풀이 결과
* **작성한 답**: `er다이어그램` (엔터티 Entity와 관계 Relationship를 이해하기 쉽게 나타낸 다이어그램의 정체를 정확히 파악하여 100% 정답 도출 성공)
* **모범 답안**: **`ERD`** *(또는 `ER 다이어그램`, `개체-관계 다이어그램`, `Entity-Relationship Diagram`)*
* **결과**: **정답 (100% 매칭)**

### 핵심 해설
이 문제는 데이터베이스 설계 단계 중 피터 체인(Peter Chen)이 제안한 **개념적 데이터 모델링(Conceptual Data Modeling)**의 핵심 산출물인 **ERD(Entity-Relationship Diagram, 개체-관계 다이어그램)**의 정의를 알고 있는지 평가하는 초빈출 기초 용어 문제입니다.

* **ERD 피터 체인 표기법(Chen's Notation) 4대 도형 족보 (무조건 출제!)**:
  1. **사각형 (Rectangle)**: **개체 (Entity)** ➔ DB 테이블 대상이 되는 데이터 집합.
  2. **마름모 (Diamond)**: **관계 (Relationship)** ➔ 엔터티 간의 연관성/매핑 관계.
  3. **타원 (Ellipse/Oval)**: **속성 (Attribute)** ➔ 엔터티가 가진 구체적인 세부 필드 항목 (밑줄 타원: 기본키 PK 속성).
  4. **선 (Line)**: 개체, 관계, 속성을 서로 **연결 (Link)**.

## 186. 소프트웨어 공학 - 요구사항 확인 (GoF 디자인 패턴 - 메모리 공유 경량화 Flyweight 플라이웨이트 & 동적 기능 확장 Decorator 데코레이터)

### 문제
* ( ① )은/는 다수의 객체로 생성될 경우 모두가 갖는 본질적인 요소를 클래스 화하여 공유함으로써 메모리를 절약하고, ‘클래스의 경량화’를 목적으로 하는 디자인 패턴으로 여러 개의 ‘가상 인스턴스’를 제공하여 메모리를 절감하는 패턴이다.
* ( ② )은/는 기존에 구현되어 있는 클래스에 필요한 기능을 추가해 나가는 설계 패턴으로 기능 확장이 필요할 때 객체 간의 결합을 통해 기능을 동적으로 유연하게 확장할 수 있게 해주어 상속의 대안으로 사용하는 디자인 패턴이다.

### 정답 및 풀이 결과
* **작성한 답**: `① Flyweight, ② Decorator` (객체 공유 메모리 절감 기작 Flyweight 와 덧붙이기 동적 기능 확장 Decorator 의 본질을 정확히 간파하여 100% 정답 도출 성공)
* **모범 답안**: **`① 플라이웨이트 (Flyweight)`, `② 데코레이터 (Decorator)`**
* **결과**: **정답 (100% 매칭)**

### 핵심 해설
이 문제는 소프트웨어 아키텍처 재사용성을 높이는 **GoF(Gang of Four) 23대 디자인 패턴** 중 **구조 패턴(Structural Pattern)**에 속하는 **플라이웨이트(Flyweight)**와 **데코레이터(Decorator)** 패턴의 정밀 정의를 알고 있는지 평가하는 빈출 기출 문제입니다.

* **GoF 7대 구조 패턴(Structural Patterns) 주요 족보**:
  1. **`플라이웨이트 (Flyweight)` (지문 ①번)**:
     * **키워드**: `공유 (Sharing)`, `메모리 절감`, `경량화`, `가상 인스턴스`
     * **개념**: 다수의 객체가 동일한 본질적 요소를 공유하게 하여 **메모리 사용량을 획기적으로 줄이는** 패턴.
  2. **`데코레이터 (Decorator)` (지문 ②번)**:
     * **키워드**: `동적 기능 확장`, `장식/덧붙임`, `상속의 대안`, `객체 결합`
     * **개념**: 기존 클래스를 수정하지 않고 **동적으로 덧붙여 기능을 확장**해 나가는 패턴 (상속(Inheritance)의 단점을 보완).
  3. **`어댑터 (Adapter)`**:
     * **키워드**: `인터페이스 호환`, `개조/변환기`
     * **개념**: 호환성이 맞지 않는 두 인터페이스를 연결해 주는 래퍼(Wrapper) 패턴.
  4. **`파사드 (Facade)`**:
     * **키워드**: `복잡한 서브시스템의 단순한 창구/인터페이스 제공`
  5. **`프록시 (Proxy)`**:
     * **키워드**: `대리인`, `접근 제어`, `지연 로딩`

## 187. 소프트웨어 공학 - 요구사항 확인 (애자일 개발 방법론 - XP 12대 실천 원칙 중 공동 코드 소유권 Collective Code Ownership & 리팩토링 Refactoring)

### 문제
* ( ① )은/는 팀의 모든 구성원이 전체 코드에 대해 수정, 개선, 확장할 수 있는 권한과 책임을 공유하는 XP의 원칙으로 코드 소유권을 특정 개발자나 모듈 담당자에게 국한시키지 않고, 팀 전체가 함께 관리한다.
* ( ② )은/는 소프트웨어의 외부 기능을 변경하지 않고 내부 구조를 개선하여, 코드의 가독성, 유지보수성, 확장성을 높이는 기법이자 원리이다.

### 정답 및 풀이 결과
* **작성한 답**: `① Whole Team (홀 팀), ② 리팩토링` (②번 리팩토링 Refactoring 의 100% 정답을 맞추었으나, ①번 코드 소유권 팀 공유 원칙의 정식 명칭인 '공동 코드 소유권 Collective Code Ownership' 대신 'Whole Team (전체 팀 역할 수용)' 을 기술하여 오답)
* **모범 답안**: **`① 공동 코드 소유권 (Collective Code Ownership)`**, **`② 리팩토링 (Refactoring)`**
* **결과**: **부분 정답 (①번 오답, ②번 정답)**

### 핵심 해설
이 문제는 애자일(Agile) 모형의 대표적인 개발 방법론인 **XP(eXtreme Programming)의 12대 실천 원칙(Practices)** 중 **공동 코드 소유권(Collective Code Ownership)**과 **리팩토링(Refactoring)**의 정의를 명확히 구분할 수 있는지 평가하는 빈출 기출 문제입니다.

* **XP 12대 실천 원칙(Practices) 핵심 족보**:
  1. **`공동 코드 소유권 (Collective Code Ownership)` (지문 ①번)**:
     * **키워드**: `전체 코드에 대한 수정/개선 권한 공유`, `팀 전체가 소유 및 함께 관리`
     * **개념**: 시스템의 모든 코드는 팀원 누구나 언제든지 수정할 수 있다는 원칙 (특정 담당자 독점 금지).
  2. **`전체 팀 (Whole Team)` (★ 수험생 작성 답과의 개념적 구별)**:
     * **키워드**: `개발자, 고객, 테스트 담당자 등 모든 역할이 하나의 팀 구성`
     * **개념**: 프로젝트와 관련된 모든 인가된 사람들(고객 포함)이 한 팀을 이루어 일한다는 원칙.
  3. **`리팩토링 (Refactoring)` (지문 ②번)**:
     * **키워드**: `외부 기능 변경 없음`, `내부 구조 개선`, `가독성/유지보수성 향상`
     * **개념**: 프로그램의 동작(기능)은 유지한 채, 내부 소스 코드를 깔끔하게 정돈하고 가독성을 높이는 작업.
  4. **`짝 프로그래밍 (Pair Programming)`**:
     * 2명의 개발자가 한 대의 컴퓨터에서 한 명은 코딩, 한 명은 검토하는 원칙.
  5. **`지속적 통합 (Continuous Integration, CI)`**:
     * 매일 수차례씩 작성한 코드를 모아 자동 빌드 및 테스트 수행.

## 188. 서비스 운영 및 인프라 구축 - 응용 소프트웨어 기초 기술 (LAN MAC 매체 접근 제어 3대 대표 기법 CSMA/CD, 토큰 버스 Token Bus, 토큰 링 Token Ring)

### 문제
다음 LAN 의 MAC(Media Access Control) 매체 접근 제어 방식 3가지에 대한 설명이다. ①, ②, ③에 해당하는 접근 제어 방식 명칭을 쓰시오.
* ( ① ): 버스형 유선 이더넷(Ethernet) 환경에서 회선 상태를 감지(Carrier Sense)하다가, 충돌(Collision)이 발생하면 이를 감지(Detection)하여 일정 시간 대기 후 재전송하는 방식.
* ( ② ): 물리적으로는 버스(Bus) 구조를 형성하지만 논리적으로는 링(Ring) 구조를 이루어 '토큰(Token)'을 순차적으로 전달하는 매체 접근 제어 방식.
* ( ③ ): 물리적으로 링(Ring) 형태의 폐순환망 구조를 이루어 '토큰(Token)'을 순차적으로 전달하여 데이터 송신 권한을 부여하는 매체 접근 제어 방식.

### 정답 및 풀이 결과
* **작성한 답**: `① CSMA/CD, ② 토큰 버스, ③ 토큰 링` (이더넷 충돌 감지 CSMA/CD 및 물리 버스-논리 링 토큰 버스, 물리 링 토큰 링 3가지 매체 접근 제어 방식을 100% 바르게 도출하여 정답!)
* **모범 답안**: **`① CSMA/CD`**, **`② 토큰 버스 (Token Bus)`**, **`③ 토큰 링 (Token Ring)`**
* **결과**: **정답 (100% 매칭)**

### 핵심 해설
이 문제는 컴퓨터 네트워크 데이터 링크 계층(2계층)의 핵심인 **LAN 매체 접근 제어(MAC, Media Access Control) 3대 기법인 CSMA/CD, 토큰 버스(Token Bus), 토큰 링(Token Ring)**의 정의와 물리적/논리적 구조 특성을 정확히 구분할 수 있는지 평가하는 기출 문제입니다.

* **LAN MAC 3대 매체 접근 제어 방식 완전 해체 족보**:
  1. **`CSMA/CD (Carrier Sense Multiple Access with Collision Detection)` (지문 ①번)**:
     * **핵심 키워드**: `유선 이더넷 (Ethernet)`, `충돌 감지 (Collision Detection)`
     * **특징**: 버스형 유선망에서 회선 사용 여부를 감지하다가 충돌 발생 시 감지하고 임의 시간 대기 후 재전송.
  2. **`토큰 버스 (Token Bus)` (지문 ②번)**:
     * **핵심 키워드**: `물리적 버스(Bus) + 논리적 링(Ring)`, `토큰 전달`
     * **특징**: 물리적으로는 버스 형태 형태지만 논리적인 링 구조를 만들어 토큰(Token)을 전달하며 충돌을 예방.
  3. **`토큰 링 (Token Ring)` (지문 ③번)**:
     * **핵심 키워드**: `물리적 링(Ring)`, `토큰 전달`, `IEEE 802.5`
     * **특징**: 물리적으로 단방향 링 구조를 형상화하여 토큰을 가진 노드가 송신 권한을 확보 (충돌 없음 Collision-Free).

## 189. 서비스 운영 및 인프라 구축 - 응용 소프트웨어 기초 기술 (FLSM 서브넷 분할 IP Subnet-Zero 적용 4번째 서브넷 브로드캐스트 IP 주소 산출 192.168.54.127)

### 문제
192.168.54.0/24인 IP 주소를 FLSM 방식으로 6개의 서브넷(Subnet)으로 분할하고, IP Subnet-Zero를 적용했다. 이때 서브넷팅(Subnetting)된 네트워크 중 4번째 서브넷(Subnet)의 브로드캐스트 IP 주소(10진수)를 쓰시오.

### 정답 및 풀이 결과
* **작성한 답**: `192.168.54.127` (6개 서브넷 분할을 위해 3비트를 빌려 서브넷당 32개 블록 크기를 구하고, Subnet-Zero 000, 001, 010, 011 비트 대입을 통해 4번째 서브넷 범위인 .96~.127 의 브로드캐스트 주소 192.168.54.127 을 100% 바르게 산출하여 정답 도출 성공)
* **모범 답안**: **`192.168.54.127`**
* **결과**: **정답 (100% 매칭)**

### 핵심 해설
이 문제는 컴퓨터 네트워크의 **서브넷팅(Subnetting) 연산 법칙**과 **IP Subnet-Zero(첫 번째 0번 서브넷부터 사용) 적용**, 그리고 **n번째 서브넷의 네트워크 주소 및 브로드캐스트 주소 산출 과정**을 정밀하게 트레이싱할 수 있는지 평가하는 정보처리기사 최고 난이도 킬러 수치 계산 문제입니다.

* **단계별 서브넷팅 트레이싱 및 정밀 연산 과정**:
  1. **서브넷 비트 수 구하기**:
     * 6개 이상의 서브넷이 필요하므로 $2^n \ge 6 \implies n = 3$ 비트를 서브넷 비트로 할당합니다.
  2. **서브넷 1개당 IP 블록 크기 (Block Size) 산출**:
     * 호스트 비트 = $8 - 3 = 5$ 비트.
     * 서브넷 1개당 IP 개수 = $2^5 =$ **`32 개`** (서브넷 마스크: `255.255.255.224` `/27`)
  3. **IP Subnet-Zero 적용 4개 서브넷 범위 타임라인**:
     * **1번째 서브넷 (비트 000)**: `192.168.54.0` ~ `192.168.54.31` (네트워크: .0 / 브로드캐스트: .31)
     * **2번째 서브넷 (비트 001)**: `192.168.54.32` ~ `192.168.54.63` (네트워크: .32 / 브로드캐스트: .63)
     * **3번째 서브넷 (비트 010)**: `192.168.54.64` ~ `192.168.54.95` (네트워크: .64 / 브로드캐스트: .95)
     * **4번째 서브넷 (비트 011)**: **`192.168.54.96` ~ `192.168.54.127`** (네트워크: .96 / 브로드캐스트: **`.127`**)
  4. **4번째 서브넷의 브로드캐스트 IP 주소**:
     * 4번째 서브넷 범위의 가장 마지막 주소 ➔ **`192.168.54.127`**

## 190. 서비스 운영 및 인프라 구축 - 응용 소프트웨어 기초 기술 (데이터 링크 계층 오류 제어 ARQ 3대 기법 - Stop-and-Wait ARQ 정지-대기 & Go-Back-N ARQ)

### 문제
* ( ① ) ARQ 방식은 한 개의 프레임을 전송하고, 수신 측으로부터 ACK 및 NAK 신호를 수신할 때까지 정보 전송을 중지하고 기다리는 방식이다.
* ( ② ) ARQ 방식은 데이터 프레임을 연속적으로 전송하는 과정에서 NAK를 수신하게 되면, 오류가 발생한 프레임 이후에 전송된 모든 데이터 프레임을 재전송하는 방식이다.

### 정답 및 풀이 결과
* **작성한 답**: `① FEC, ② BEC` (오류 제어 대분류인 전진 오류 수정 FEC 와 후진 오류 수정 BEC 개념을 연상하였으나, BEC 하위의 자동 재전송 요구 ARQ 3대 구체적 프로토콜 명칭 미인지로 오답)
* **모범 답안**: **`① Stop-and-Wait ARQ (정지-대기 ARQ)`**, **`② Go-Back-N ARQ`**
* **결과**: **오답 (ARQ 세부 방식 명칭 Stop-and-Wait / Go-Back-N 미기재)**

### 핵심 해설
이 문제는 컴퓨터 네트워크 데이터 링크 계층(2계층)에서 프레임 전송 시 오류가 발생하면 수신 측의 요청에 의해 데이터를 재전송하는 **ARQ(Automatic Repeat Request, 자동 재전송 요구)** 세부 방식의 작동 메커니즘을 정확히 구분하고 있는지 평가하는 빈출 기출 문제입니다.

* **ARQ (자동 재전송 요구) 3대 기법 완전 해체 족보**:
  1. **`Stop-and-Wait ARQ (정지-대기 ARQ)` (지문 ①번)**:
     * **키워드**: `프레임 1개 전송 후 대기`, `ACK/NAK 수신 시까지 전송 중지`
     * **원리**: 송신 측이 한 번에 딱 1개의 프레임만 보내고, 수신 측으로부터 응답(ACK/NAK)이 돌아올 때까지 기다린 후 다음 프레임을 송신하는 가장 단순한 방식.
  2. **`Go-Back-N ARQ` (지문 ②번)**:
     * **키워드**: `연속 전송 중 NAK 발생`, `오류 발생 프레임 이후 모든 프레임 손수 재전송`
     * **원리**: 슬라이딩 윈도우를 이용해 연속으로 보내다가 NAK(오류)를 받으면, **오류가 난 프레임부터 그 뒤에 보냈던 모든 프레임을 되돌아가서 전부 다시 전송**.
  3. **`선택적 재전송 ARQ (Selective Repeat ARQ)`**:
     * **키워드**: `오류 발생한 특정 프레임만 선별 재전송`
     * **원리**: Go-Back-N 과 달리 **오류가 발생한 그 프레임 1개만 골라서 재전송**하는 방식 (수신 측 버퍼 필요).
* **★ [수험생 오해 해소] 오류 제어 전체 대분류 체계**:
  * **`FEC (Forward Error Correction)`**: 수신 측이 해밍코드 등을 이용해 **스스로 오류를 정정** (재전송 X)
  * **`BEC (Backward Error Correction)`**: 수신 측이 송신 측에 **재전송을 요청하는 ARQ 방식** (본 문제의 상위 개념!)

## 191. 서비스 운영 및 인프라 구축 - 응용 소프트웨어 기초 기술 (운영체제 OS 2대 핵심 실행 단위 - 프로세스 Process & 스레드 Thread)

### 문제
* ( ① )은/는 CPU에 의해 처리되는 사용자 프로그램, 시스템 프로그램, 실행 중인 프로그램으로 작업(Job) 또는 태스크(Task)라고도 한다.
* ( ② )은/는 독립적으로 수행되는 순차적인 제어의 흐름이며 ( ① )(에)서 실행 제어만 분리한 실행 단위이다.

### 정답 및 풀이 결과

* **모범 답안**: **`① 프로세스 (Process)`**, **`② 스레드 (Thread)`**
* **결과**: **오답 (①번과 ②번 위치 교정 필요)**

### 핵심 해설
이 문제는 운영체제(OS) 관리의 가장 기본이 되는 **프로세스(Process)**와 **스레드(Thread)**의 정의 및 두 개념 간의 포섭 관계를 정확히 이해하고 구분할 수 있는지 평가하는 빈출 용어 기입형 문제입니다.

* **프로세스 (Process) vs 스레드 (Thread) 완전 해체 족보**:
  1. **`프로세스 (Process)` (지문 ①번)**:
     * **키워드**: `실행 중인 프로그램 (Program in Execution)`, `작업(Job)`, `태스크(Task)`
     * **개념**: 디스크에 있는 프로그램이 메인 메모리에 탑재되어 **CPU 자원을 할당받아 실행되고 있는 주체 (독립된 메모리 영역 Code, Data, Heap, Stack 보유)**.
  2. **`스레드 (Thread)` (지문 ②번)**:
     * **키워드**: `프로세스 내의 실행 흐름/단위`, `경량 프로세스 (Light-Weight Process)`, `독립된 Stack/Register 만 보유`
     * **개념**: 프로세스 하나 내부에서 **실행 제어(Flow of Control)만을 따로 분리해 낸 최소 단위**로, 같은 프로세스 내의 스레드들끼리는 메모리(Code, Data, Heap)를 공유함.

## 192. Python - 프로그래밍 언어 활용 (문자열 알파벳 빈도수 딕셔너리 카운팅 calc 및 max(r, key=r.get) 최대 빈도 Key 추출 출력 o:2)

### 문제
아래 파이썬 코드를 실행했을 때 출력되는 결과를 쓰시오. (단, 코드 블록 HTML 내에서 바르게 파이썬 코드로 디해싱하여 풀이한다.)
```python
def calc(text):
  cnt = {}
  text = text.lower()
  for c in text:
    if c.isalpha(): 
      if c in cnt:
        cnt[c] += 1
      else:
        cnt[c] = 1
  return cnt

text = "soojebi"
r = calc(text)
m = max(r, key=r.get)
print(f"{m}:{r[m]}")
```

### 정답 및 풀이 결과
* **작성한 답**: `o:2` (calc 함수를 통한 "soojebi" 알파벳 빈도수 딕셔너리 카운팅 r={'s':1, 'o':2, 'j':1, 'e':1, 'b':1, 'i':1} 산출과 max(r, key=r.get) 을 통한 최대 빈도 Key 'o' 추출 및 f-string 출력 형식 o:2 를 100% 바르게 도출하여 정답!)
* **모범 답안**: **`o:2`**
* **결과**: **정답 (100% 매칭)**

### 핵심 해설
이 문제는 파이썬의 **딕셔너리(Dictionary) 자료형을 활용한 문자열 빈도수 카운팅 알고리즘**과, **`max(r, key=r.get)` 구문을 활용한 Value 기준 최대 Key 추출 테크닉**, 그리고 **f-string 포맷팅 출력**을 정밀하게 트레이싱할 수 있는지 평가하는 기출 문제입니다.

* **단계별 라인 트레이싱 및 연산 분석**:
  1. **`calc("soojebi")` 실행**:
     * 문자열 `"soojebi"` 의 각 알파벳 빈도 카운트 ➔ `r = {'s': 1, 'o': 2, 'j': 1, 'e': 1, 'b': 1, 'i': 1}`
     * `'o'` 가 총 2개 포함되어 빈도수 최댓값을 가짐!
  2. **`m = max(r, key=r.get)` 실행 (★ 파이썬 킬러 테크닉)**:
     * `max(dict, key=dict.get)` 은 딕셔너리의 Key들 중 **Value(`dict.get(key)`)가 가장 큰 Key**를 찾아 리턴함!
     * 가장 큰 Value(2)를 가진 Key 인 **`'o'`** 가 변수 `m` 에 대입됨.
  3. **`print(f"{m}:{r[m]}")` 최종 출력**:
     * `m` = `'o'`, `r[m]` = `r['o']` = `2`
     * f-string 변수 바인딩 ➔ **`o:2`**
  4. **★ [수험생 질답 노트] `max(r, key=r.get)` 의 내부 파이썬 동작 메커니즘**:
     * 그냥 `max(r)` 을 호출하면 딕셔너리의 Key 문자열 자체('s', 'o', 'j' 등의 알파벳 아스키 순서)를 비교함.
     * `key=r.get` 함수 매개변수를 전달하면, 파이썬이 내부적으로 각 Key 들을 순회할 때 **크기를 비교(우선순위)하는 기준값으로는 `r.get(key)` 의 리턴값인 숫자(Value: 1, 2, 1)를 사용**함!
     * 비교 후 **최종 리턴(Return)할 때는 대입할 원본 대상인 Key `'o'` 를 반환**해 주는 아주 유용한 파이썬의 고급 함수 기작임.

## 193. DB - SQL 응용 (인라인 뷰 Inline View 서브쿼리 필터링 & 이너 조인 & MAX 집계 함수 결과 333)

### 문제
다음 SQL 실행 시 [결과]의 밑줄에 들어갈 알맞은 값을 쓰시오.
* **[도서] 테이블**:
  - `(111, '운영체제')`, `(222, '자료구조')`, `(333, '데이터베이스')`, `(444, '소프트웨어공학')`
* **[도서가격] 테이블**:

* **[쿼리]**:
```sql
SELECT MAX(A.책번호) AS 책번호
  FROM 도서 A, (SELECT 책번호 FROM 도서가격 WHERE 가격 = 20000) B
 WHERE A.책번호 = B.책번호
```

### 정답 및 풀이 결과
* **작성한 답**: `333` (인라인 뷰 서브쿼리 B 에서 가격=20000 인 책번호 111, 333 을 필터링하고 도서 테이블 A와 조인 후 MAX(책번호) 인 333 을 100% 바르게 도출하여 정답 도출 성공)
* **모범 답안**: **`333`**
* **결과**: **정답 (100% 매칭)**

### 핵심 해설
이 문제는 SQL의 **FROM 절에 위치하는 인라인 뷰(Inline View) 서브쿼리의 독립 실행**과, **조인 조건(`A.책번호 = B.책번호`)을 통한 튜플 결합**, 그리고 **`MAX()` 그룹 집계 함수의 최댓값 추출 과정**을 바르게 트레이싱할 수 있는지 평가하는 기출 문제 유형입니다.

* **단계별 쿼리 트레이싱 및 연산 분석**:
  1. **FROM 절 인라인 뷰 `B` 서브쿼리 우선 실행**:
     * `SELECT 책번호 FROM 도서가격 WHERE 가격 = 20000`
     * `도서가격` 테이블 중 가격이 `20000` 인 행의 책번호 ➔ **`111`**, **`333`** 튜플 집합 생성.
  2. **`WHERE A.책번호 = B.책번호` 이너 조인 매칭**:
     * `도서 A` 테이블과 `서브쿼리 B` 의 공통 책번호 매칭 ➔ **`111`**, **`333`**
  3. **`SELECT MAX(A.책번호)` 최댓값 연산**:
     * 매칭된 책번호 중 최댓값 `MAX(111, 333)` ➔ **`333`**

## 194. DB - SQL 응용 (INNER JOIN & GROUP BY 부서별 그룹화 & HAVING COUNT >= 2 필터링 결과 총무팀: 2)

### 문제
아래 테이블 [EMP], [DEPT] 와 [쿼리]를 실행했을 때 출력되는 결과(부서이름, 인원수)를 쓰시오.
* **[EMP] 테이블**:
  - `(101, '홍길동', 10001)`, `(102, '김철수', 10002)`, `(103, '이만기', 10003)`, `(104, '박지훈', 10001)`
* **[DEPT] 테이블**:
  - `(10001, '총무팀')`, `(10002, '전산팀')`, `(10003, '회계팀')`
* **[쿼리]**:
```sql
SELECT D.dept_nm AS 부서이름, COUNT(E.emp_no) AS 인원수
  FROM DEPT D JOIN EMP E
    ON D.dept_id = E.dept_id
 GROUP BY D.dept_nm
HAVING COUNT(E.emp_no) >= 2;
```

### 정답 및 풀이 결과
* **작성한 답**: `부서이름 총무팀 / 인원수 2` (DEPT와 EMP 간의 이너 조인 후 D.dept_nm 부서명 기준 그룹화 카운트 총무팀 2명, 전산팀 1명, 회계팀 1명 산출과 HAVING COUNT >= 2 조건 필터링을 거쳐 총무팀: 2 결과를 100% 바르게 도출하여 정답!)
* **모범 답안**:
  | 부서이름 | 인원수 |
  | :---: | :---: |
  | **총무팀** | **2** |
* **결과**: **정답 (100% 매칭)**

### 핵심 해설
이 문제는 **ANSI 표준 SQL의 INNER JOIN(ON 절)**과 **`GROUP BY` 절을 통한 소그룹 집계**, 그리고 **`HAVING` 절을 통한 그룹 조건 필터링**의 연산 체계 및 **SQL 6대 구문 순서**를 정확히 트레이싱할 수 있는지 평가하는 기출 문제입니다.

* **단계별 쿼리 트레이싱 및 연산 분석**:
  1. **`FROM DEPT D JOIN EMP E ON D.dept_id = E.dept_id` 실행**:
     * `홍길동(101)`, `박지훈(104)` ➔ `10001 (총무팀)`
     * `김철수(102)` ➔ `10002 (전산팀)`
     * `이만기(103)` ➔ `10003 (회계팀)`
  2. **`GROUP BY D.dept_nm` 부서별 카운트 집계**:
     * **`총무팀`**: 홍길동(101), 박지훈(104) ➔ **`COUNT = 2`**
     * **`전산팀`**: 김철수(102) ➔ **`COUNT = 1`**
     * **`회계팀`**: 이만기(103) ➔ **`COUNT = 1`**
  3. **`HAVING COUNT(E.emp_no) >= 2` 필터링**:

## 195. Python - 프로그래밍 언어 활용 (위치 매개변수 바인딩 함정 & try-else-finally 구문 & f-string 실수 포맷팅 rate:2.00, value:202)

```python
def calculate(value, new_value=None, increase_rate=2):

value = 100

try:
```

## 196. C언어 - 프로그래밍 언어 활용 (단순 연결 리스트 오름차순 정렬 삽입 insert & delete_node(8) 노드 삭제 후 순회 출력 3510)

### 문제
아래 C언어 코드를 실행했을 때 출력되는 결과를 쓰시오. (단, 코드 블록 HTML 내에서 바르게 C언어 코드로 디해싱하여 풀이한다.)
```c
#include <stdio.h>
#include <stdlib.h>

typedef struct _node {
  int data;
  struct _node *next;
} node;

node *head, *tail;

void init_list(void){
  head = (node *)malloc(sizeof(node));
  tail = (node *)malloc(sizeof(node));
  head->next = tail;
  tail->next = NULL;
}

node *insert(int k){
  node *p = head;
  node *new_node = (node *)malloc(sizeof(node));
  new_node->data = k;
  while(p->next != tail && p->next->data < k) {
    p = p->next;
  }
  new_node->next = p->next;
  p->next = new_node;
  return new_node;
}

void print_list(node *t){
  while(t != tail) {
    printf("%d", t->data);
    t = t->next;
  }
}

int delete_node(int k){
  node *p = head;
  node *temp;
  while(p->next != tail && p->next->data != k) {
    p = p->next;
  }
  if(p->next == tail) {
    return -1; 
  }
  temp = p->next;
  p->next = temp->next;
  free(temp);
  return 0; 
}

int main(){
  init_list();
  insert(10);
  insert(5);
  insert(8);
  insert(3);
  delete_node(8);
  print_list(head->next);
  return 0;
}
```

### 정답 및 풀이 결과
* **작성한 답**: `3510` (insert 함수의 오름차순 정렬 삽입 트레이싱으로 head->3->5->8->10->tail 구성과 delete_node(8) 노드 삭제 후 head->3->5->10->tail 연결 리스트 상태에서 print_list 출력을 수행하여 3510 을 100% 바르게 도출하여 정답 도출 성공)
* **모범 답안**: **`3510`**
* **결과**: **정답 (100% 매칭)**

### 핵심 해설
이 문제는 C언어의 **동적 메모리 할당(`malloc`, `free`)과 구조체 포인터(`struct _node *`)를 활용한 단순 연결 리스트(Singly Linked List)**에서 **오름차순 정렬 삽입 알고리즘**과 **특정 노드 탐색 및 삭제**, 그리고 **포인터 순회 출력**을 정밀하게 트레이싱할 수 있는지 평가하는 C언어 고난도 킬러 문제입니다.

* **단계별 메인 코드 포인터 트레이싱 과정**:
  1. **`init_list()` 더미 노드 초기화**:
     * `head -> tail -> NULL`
  2. **`insert()` 호출 (오름차순 자동 정렬 삽입)**:
     * `insert(10)` ➔ `head -> [10] -> tail`
     * `insert(5)` ➔ `head -> [5] -> [10] -> tail`
     * `insert(8)` ➔ `head -> [5] -> [8] -> [10] -> tail`
     * `insert(3)` ➔ `head -> [3] -> [5] -> [8] -> [10] -> tail`
     * (※ `p->next->data < k` 조건에 의해 작은 값이 앞서도록 오름차순 정렬됨!)
  3. **`delete_node(8)` 호출 (노드 [8] 탐색 및 삭제)**:
     * `p` 가 `[5]` 노드를 가리킬 때 `p->next` 가 `[8]` 임을 감지!
     * `temp = [8]`, `p->next = [10]`, `free([8])`
     * **삭제 후 연결 상태**: **`head -> [3] -> [5] -> [10] -> tail`**
  4. **`print_list(head->next)` 순회 출력**:
     * `[3]` ➔ `3` 출력
     * `[5]` ➔ `5` 출력
     * `[10]` ➔ `10` 출력
     * 최종 출력 문자열 ➔ **`3510`**

## 197. C언어 - 프로그래밍 언어 활용 (문자열 대칭 인덱스 요소 교환 반전 알고리즘 fn("soojebi") 결과 ibejoos)

```c
#include <stdio.h>

void fn(char *str) {
  char temp;

    temp = str[i];
    str[i] = str[length-i-1];

}    

int main() {

}
```

### 핵심 해설
이 문제는 C언어의 **`strlen()` 함수를 이용한 문자열 길이 측정**과, **양 끝 대칭 인덱스(`i` ↔ `length-i-1`) 간의 `temp` 변수를 활용한 문자 값 맞교환(Swap) 알고리즘**을 정확히 이해하고 트레이싱할 수 있는지 평가하는 기출 문제 유형입니다.

     * **`ibejoos`**

## 198. Java - 프로그래밍 언어 활용 (다중 catch 블록 예외 발생 트레이싱 - ArrayIndexOutOfBoundsException 우선 발생 "B" & finally "E" 결과 BE)

아래 자바 코드를 실행했을 때 출력되는 결과를 쓰시오. (단, 코드 블록 HTML 내에서 바르게 자바 코드로 디해싱하여 풀이한다.)

```java
public class Soojebi {
```

  1. **`try` 블록 1번째 문장 `System.out.print(arr[5]);` 실행**:

  3. **`finally` 블록 실행**:

     * **`BE`**

## 199. C언어 - 프로그래밍 언어 활용 (3차원 배열 x[4][4][4] & 배열 포인터 산술 & 음수 인덱스 c[2][-1][-2] 정밀 포인터 트레이싱 74 71\n88 84)

### 문제
아래 C언어 코드를 실행했을 때 출력되는 결과를 쓰시오. (단, 코드 블록 HTML 내에서 바르게 C언어 코드로 디해싱하여 풀이한다.)
```c
#include <stdio.h>

int main() {
  int x[4][4][4];
  int i, j, k;
  int *a, (*b)[4], (*c)[4][4];

  for(i=0; i<4; i++){
    for(j=0; j<4; j++){
      for(k=0; k<4; k++){
        x[i][j][k] = i*32 + j*8 + k*2;
      }
    }
  }

  a = x[2][1];
  b = *(x+3);
  c = x+1;

  printf("%d %d\n", *(a+1), a[-1]+1);
  printf("%d %d\n", **(b-1), c[2][-1][-2]);

  return 0;
}
```

### 정답 및 풀이 결과
* **작성한 답**: `74 71\n88 84` (3차원 배열 요소 계산 공식 i*32 + j*8 + k*2 와 포인터 a, b, c 차원별 오프셋 이동 및 음수 인덱스 a[-1]=x[2][0][3]=70, **(b-1)=x[2][3][0]=88, c[2][-1][-2]=x[2][2][2]=84 정밀 계산으로 74 71 / 88 84 100% 도출 성공)
* **모범 답안**:
```text
74 71
88 84
```
* **결과**: **정답 (100% 매칭)**

### 핵심 해설
이 문제는 C언어의 **3차원 배열(`int x[4][4][4]`)의 1차원 연속 메모리 구조**와 **차원별 포인터 변수(`int *a`, `int (*b)[4]`, `int (*c)[4][4]`)의 오프셋 이동**, 그리고 **음수 인덱스(`a[-1]`, `c[2][-1][-2]`) 포인터 산술**을 정밀하게 트레이싱할 수 있는지 평가하는 정보처리기사 최고 난이도 극악 킬러 문제입니다.

* **초정밀 단계별 포인터 트레이싱 계산 과정**:
  1. **`*(a+1)` 및 `a[-1]+1` 연산**:
     * `a = x[2][1]` ➔ `&x[2][1][0]` (값 72)
     * `*(a+1)` = `x[2][1][1]` ➔ $2 \times 32 + 1 \times 8 + 1 \times 2 = 64 + 8 + 2 =$ **`74`**
     * `a[-1]` = `*(a - 1)` ➔ `a` 바로 직전 정수인 `x[2][0][3]` (값 70)
     * `a[-1] + 1` = $70 + 1 =$ **`71`**
  2. **`**(b-1)` 및 `c[2][-1][-2]` 연산**:
     * `b = *(x+3)` ➔ `&x[3][0]` (크기 4인 배열 포인터 `int (*)[4]`)
     * `b - 1` ➔ 4개 정수 묶음 하나 전인 `x[2][3]` (주소 `&x[2][3][0]`)
     * `**(b-1)` = `x[2][3][0]` ➔ $2 \times 32 + 3 \times 8 + 0 \times 2 = 64 + 24 + 0 =$ **`88`**
     * `c = x+1` ➔ `c[2][-1][-2]` ➔ `x[3][-1][-2]` ➔ `x[2][3][-2]` ➔ 정수 2개 전인 `x[2][2][2]`
     * `x[2][2][2]` = $2 \times 32 + 2 \times 8 + 2 \times 2 = 64 + 16 + 4 =$ **`84`**
  3. **최종 출력**:
     ```text
     74 71
     88 84
     ```
  4. **★ [수험생의 4단계 초정밀 포인터 수식 전개 노트]**:
     * `*(a+1)` = `*(x[2][1]+1)` = `x[2][1][1]` = `2*32 + 1*8 + 1*2` = `64 + 8 + 2` = **`74`**
     * `a[-1] + 1` = `x[2][1][-1] + 1` = `x[2][0][3] + 1` = `2*32 + 0 + 3*2 + 1` = `64 + 6 + 1` = **`71`**
     * `**(b-1)` = `**((x+3)-1)` = `*x[3][-1]` = `x[2][3][0]` = `2*32 + 3*8` = `64 + 24` = **`88`**
     * `c[2][-1][-2]` = `(x+1)[2][-1][-2]` = `x[3][-1][-2]` = `x[2][3][-2]` = `x[2][2][2]` = `2*32 + 2*8 + 2*2` = `64 + 16 + 4` = **`84`**

## 200. 🎉 [200번 영광의 기념비적 문항] C언어 - 프로그래밍 언어 활용 (재귀 호출 Call Stack 진입/복귀 LIFO 순회 & 전위 감증 x[--y] 트레이싱 킬러)

### 문제
아래 C언어 코드를 실행했을 때 출력되는 결과를 쓰시오. (단, 코드 블록 HTML 내에서 바르게 C언어 코드로 디해싱하여 풀이한다.)
```c
#include <stdio.h>

int t = 0;

void fn(int x[], int y) {
  int z = t;
  t++;
  printf("A: %d\n", z);
  if (y <= 0) return;
  printf("B: %d\n", x[--y]);
  fn(x, y);
  printf("C: %d\n", z);
}

int main() {
  int x[4] = {2, 1, 3, 7};
  int y = 4;
  fn(x, y);
  return 0;
}
```

### 정답 및 풀이 결과
* **작성한 답**:
```text
A:0
B:7
A:1
B:3
A:2
B:1
A:3
B:2
A:4
C:3
C:2
C:1
C:0
```
  (재귀 함수 fn 호출 시 전역변수 t와 지역변수 z의 프레임별 보관, --y 감소 후 배열 x 인덱스 접근, y<=0 return 기지 조건 도달 후 호출 스택 LIFO 역순 C:3 -> C:2 -> C:1 -> C:0 복귀 출력을 100% 바르게 도출하여 정답!)
* **모범 답안**:
```text
A: 0
B: 7
A: 1
B: 3
A: 2
B: 1
A: 3
B: 2
A: 4
C: 3
C: 2
C: 1
C: 0
```
* **결과**: **정답 (100% 매칭)**

### 핵심 해설
이 문제는 C언어의 **재귀 호출(Recursive Call) 시 메모리 스택 프레임(Stack Frame)에 저장되는 전역 변수(`t`)와 지역 변수(`z`)의 수명 스코프**, **전위 감소 연산자(`--y`)**, **기지 조건(`if (y <= 0) return;`)**, 그리고 **호출 스택 복귀 시 LIFO(후입선출) 역순 출력 과정**을 정밀하게 트레이싱할 수 있는지 평가하는 기출 문제 유형입니다.

* **초정밀 단계별 호출 스택 (Call Stack) 트레이싱 과정**:
  1. **`fn(x, 4)` 1회차 호출**: `z=0`, `t=1` ➔ **`A: 0`** / `--y=3` ➔ `x[3]=7` ➔ **`B: 7`** ➔ `fn(x, 3)` 호출
  2. **`fn(x, 3)` 2회차 호출**: `z=1`, `t=2` ➔ **`A: 1`** / `--y=2` ➔ `x[2]=3` ➔ **`B: 3`** ➔ `fn(x, 2)` 호출
  3. **`fn(x, 2)` 3회차 호출**: `z=2`, `t=3` ➔ **`A: 2`** / `--y=1` ➔ `x[1]=1` ➔ **`B: 1`** ➔ `fn(x, 1)` 호출
  4. **`fn(x, 1)` 4회차 호출**: `z=3`, `t=4` ➔ **`A: 3`** / `--y=0` ➔ `x[0]=2` ➔ **`B: 2`** ➔ `fn(x, 0)` 호출
  5. **`fn(x, 0)` 5회차 호출 (기지 조건 도달)**: `z=4`, `t=5` ➔ **`A: 4`** / `if (0 <= 0)` 참 ➔ **`return;` (복귀 시작!)**
  6. **스택 프레임 LIFO 역순 복귀 (Unwinding)**:
     * 4회차 호출(`z=3`) 복귀 ➔ **`C: 3`**
     * 3회차 호출(`z=2`) 복귀 ➔ **`C: 2`**
     * 2회차 호출(`z=1`) 복귀 ➔ **`C: 1`**
     * 1회차 호출(`z=0`) 복귀 ➔ **`C: 0`**

## 201. C언어 - 프로그래밍 언어 활용 (진법 리터럴 0x 16진수 & 0 8진수 & 0b 2진수 및 서식 지정자 %x 16진수 & %o 8진수 교차 포맷팅 1111 249 f \n 10421 1111 17)

### 문제
아래 C언어 코드를 실행했을 때 출력되는 결과를 쓰시오. (단, 코드 블록 HTML 내에서 바르게 C언어 코드로 디해싱하여 풀이한다.)
```c
#include <stdio.h>

int main(){
  int a = 0x1111;
  int b = 01111;
  int c = 0b1111;
  printf("%x %x %x\n", a, b, c);
  printf("%o %o %o\n", a, b, c);
  return 0;
}
```

### 정답 및 풀이 결과
* **작성한 답**:
```text
1111 249 f
10421 1111 17
```
  (진법 리터럴 0x1111=4369, 01111=585, 0b1111=15 의 비트 구조 해석과 %x 16진수 소문자 표기 1111, 249, f 및 %o 8진수 표기 10421, 1111, 17 상호 변환을 100% 바르게 도출하여 정답!)
* **모범 답안**:
```text
1111 249 f
10421 1111 17
```
* **결과**: **정답 (100% 매칭)**

### 핵심 해설
이 문제는 C언어에서 **상수 리터럴의 접두사에 따른 진법 구분(`0x` 16진수, `0` 8진수, `0b` 2진수)**과, **`printf()` 서식 지정자(`%x` 16진수 소문자, `%o` 8진수)**를 활용한 **2진수 비트 그룹핑 16진수(4비트)/8진수(3비트) 상호 진법 교차 출력**을 트레이싱할 수 있는지 평가하는 기출 문제입니다.

* **변수별 10진수/2진수 정밀 진법 변환**:
  1. **`a = 0x1111` (16진수)**:
     * 10진수 값: $16^3 + 16^2 + 16^1 + 16^0 = 4096 + 256 + 16 + 1 = 4369$
     * `%x` (16진수): **`1111`**
     * `%o` (8진수 3비트 그룹): `001 000 100 010 001 001` (2진수) ➔ **`10421`**
  2. **`b = 01111` (8진수)**:
     * 10진수 값: $8^3 + 8^2 + 8^1 + 8^0 = 512 + 64 + 8 + 1 = 585$
     * `%o` (8진수): **`1111`**
     * `%x` (16진수 4비트 그룹): `0000 0010 0100 1001` (2진수) ➔ **`249`**
  3. **`c = 0b1111` (2진수)**:
     * 10진수 값: $8 + 4 + 2 + 1 = 15$
     * `%x` (16진수 소문자): 15 ➔ **`f`**
     * `%o` (8진수 3비트 그룹): `001 111` ➔ **`17`**
  4. **최종 출력**:
     ```text
     1111 249 f
     10421 1111 17
     ```
