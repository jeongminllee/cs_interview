# 1. C언어 포인터 및 배열 연산

## 문제
아래 코드를 확인하여 알맞은 출력값을 작성하시오.

```c
#include <stdio.h>

double arr1(int p[], int len) {
    double av = 0;
    int i;
    for (i = 0; i < len; i++) {
        av += (double) p[i];
    }
    return av / len;
}

double arr2(int * p, int len) {
    double av = 0;
    int i;
    for (i = 0; i < len; i++) {
        av += (double)(*(p + i));
    }
    return av / len;
}

int main() {
    int arr[10] = {80, 20, 50, 55, 45, 95, 55, 10, 40, 80};
    int len = 10;
    printf("%.2f", arr1(arr, len) + arr2(arr, len));
    return 0;
}
```

## 정답
**`106.00`**

## 해설
1. `arr` 배열의 원소 합은 $80+20+50+55+45+95+55+10+40+80 = 530$ 이다.
2. `arr1` 함수는 배열의 형태(`int p[]`)로 인자를 받아 평균값을 계산한다. 총합 530을 `len`(10)으로 나눈 값인 `53.0`을 반환한다.
3. `arr2` 함수는 포인터 형태(`int * p`)로 인자를 받아 역참조(`*(p + i)`)를 이용해 합산한 뒤 평균값을 계산한다. 동일하게 `53.0`을 반환한다.
4. `main` 함수에서 두 반환값의 합인 `53.0 + 53.0 = 106.0`을 출력 형식 `%.2f`에 맞춰 소수점 둘째 자리까지 출력하므로 `106.00`이 된다.

# 2. 디자인 패턴 유형

## 문제
다음 설명에 해당하는 디자인 패턴의 유형을 괄호( ) 안에 쓰시오.
- ( ㄱ ) 패턴은 기능의 클래스 계층과 구현의 클래스 계층을 연결
- ( ㄴ ) 패턴은 한 객체의 상태가 바뀌면 그 객체에 의존하는 다른 객체들에게 연락이 가고 자동으로 내용이 갱신되는 방식으로 일대다 의존성을 정의

## 정답
- **ㄱ: 브리지 (Bridge)**
- **ㄴ: 옵저버 (Observer)**

## 해설
- **Bridge 패턴**: 기능 계층과 구현 계층을 분리하여 각 계층이 독립적으로 확장될 수 있도록 하는 구조 패턴이다.
- **Observer 패턴**: 한 객체의 상태 변화에 따라 다른 의존 객체들이 상호작용 없이 자동으로 갱신을 통지받는 행위 패턴이다.

# 6. HDLC 프레임 및 모드

## 문제
HDLC는 비트 중심의 데이터 링크 제어 프로토콜로, 프레임 단위로 데이터를 전송하며 흐름 제어 및 오류 복구 기능을 제공한다. 다음 설명을 읽고 알맞은 용어를 쓰시오. [확인 필요]
1. HDLC 프레임의 구성 단위로, 실제 사용자 데이터를 전송하는 프레임
2. 데이터 링크의 흐름을 관리하고 오류 제어 및 통신 상태를 감시하는 프레임
3. 순서 번호 없이 링크 설정, 해제, 모드 설정 등 제어 기능을 수행하는 프레임
4. 두 국(Station)이 동등한 위치에서 서로 명령과 응답을 주고받는 모드
5. 종국(Secondary)이 주국(Primary)의 허가 없이도 자발적으로 응답을 전송할 수 있는 모드

## 정답
1. **정보 프레임 (I-Frame)**
2. **감독 프레임 (S-Frame)**
3. **비번호 프레임 (U-Frame)**
4. **비동기 균형 모드 (ABM)**
5. **비동기 응답 모드 (ARM)**

## 해설
- **정보(I) 프레임**: 사용자 데이터를 실어 나르고, 송수신 확인 메커니즘을 포함한다.
- **감독(S) 프레임**: 오류 제어와 흐름 제어를 담당하며, 정보 프레임 전송 감시를 수행한다.
- **비번호(U) 프레임**: 순서 번호가 없어 링크 수립 및 종료 등 링크 제어 용도로 쓰인다.
- **ABM (Asynchronous Balanced Mode)**: 두 국이 혼합국으로 동작하여 대등하게 제어를 수행한다.
- **ARM (Asynchronous Response Mode)**: 종국이 주국의 허가 없이 비동기적으로 응답 프레임을 개시할 수 있다.

# 7. Java 상속과 다형성

## 문제
아래 코드를 확인하여 알맞는 출력값을 작성하시오.

```java
class A {
    String f(Object x) {
        return "1";
    }
    String g() {
        return f("a");
    }
}

class B extends A {
    String f(Object x) {
        return "2";
    }
    String f(String x) {
        return "3";
    }
}

public class Main {
    public static void main(String[] args) {
        A a = new B();
        System.out.println(a.g());
    }
}
```

## 정답
**`2`**

## 해설
1. `A a = new B();`에 의해 객체 타입은 `B`이지만 참조 변수 타입은 `A`이다.
2. `a.g()`를 호출하면 상속된 `g()` 메서드가 실행되는데, 이때 내부적으로 `f("a")`를 호출한다.
3. 컴파일 타임에 `A` 클래스 내부에서 `f("a")`를 분석할 때 바인딩 가능한 메서드는 `f(Object x)` 뿐이다. (`A` 클래스에는 `f(String)`이 정의되어 있지 않음)
4. 런타임에 동적 디스패치(Dynamic Dispatch)가 일어날 때, 객체 `B`에 재정의(Override)된 `f(Object x)`를 실행하게 된다.
5. 따라서 `B` 클래스의 `f(Object x)`가 반환하는 `"2"`가 출력된다. `B`에 오버로딩된 `f(String x)`인 `"3"`은 호출 경로에 속하지 못한다.

i = input()
x = []
for word in i.split():

3. 리스트 컴프리헨션 내부 조건 `if c not in 'ong'`에 의해 `'o'`, `'n'`, `'g'` 문자 중 하나에 해당하면 필터링된다.
4. `"veDnamuH"`에서 알파벳 `'n'`이 제거되어 최종적으로 `"veDamuH"`가 출력된다.

# 9. SQL 행 개수 계산

## 문제
아래 조건을 참고하여 각 SQL 구문을 실행했을 때 반환되는 행(Row)의 수를 쓰시오. (단, DEPT 칼럼은 학과명이다.)

> **[테이블 조건]**
> STUDENT 테이블에는 다음 세 학과의 학생 정보가 저장되어 있다.
> 컴퓨터과 50명 . 인터넷과 100명 . 사무자동화과 50명

[SQL 구문]
1. `SELECT DEPT FROM STUDENT;`
2. `SELECT DISTINCT DEPT FROM STUDENT;`
3. `SELECT COUNT(DISTINCT DEPT) FROM STUDENT WHERE DEPT = '컴퓨터과';`

## 정답
1. **`200`**
2. **`3`**
3. **`1`**

## 해설
1. 총 학생 수는 컴퓨터과(50) + 인터넷과(100) + 사무자동화과(50) = 200명이다. 중복 제거 없이 모든 행의 학과명을 가져오므로 **200개**의 행이 반환된다.
2. `DISTINCT` 키워드로 인해 학과명의 중복이 제거되어 고유한 학과 명칭 목록(컴퓨터과, 인터넷과, 사무자동화과)만 가져오므로 **3개**의 행이 반환된다.
3. 조건절 `WHERE DEPT = '컴퓨터과'`에 해당하는 고유 학과는 '컴퓨터과' 1개뿐이다. `COUNT()` 집계 함수는 행의 개수 자체(단일 값)를 표현하는 1개의 행을 반환한다. (만약 반환되는 행의 수가 아니라 카운팅된 값 자체를 묻더라도 `1`이다.)

CREATE TABLE PLAYER (

## 정답

`CONSTRAINT [제약조건명] FOREIGN KEY ([로컬칼럼]) REFERENCES [참조테이블]([참조칼럼])`
따라서 각 빈칸에 알맞은 키워드는 차례대로 `CONSTRAINT`, `FOREIGN`, `TEAM_ID`, `REFERENCES`, `TEAM_ID2`가 된다.

# 11. IP 주소와 서브넷 마스크 연산

## 문제
두 호스트 a, b의 IP 주소와 서브넷 마스크가 주어졌을 때, 각 호스트가 속한 네트워크 주소를 CIDR 표기법으로 쓰시오.
- 호스트 a의 IP 주소: `192.168.11.20`
- 호스트 b의 IP 주소: `192.168.12.200`
- 서브넷 마스크: `255.255.254.0`

## 정답
- **a: `192.168.10.0/23`**
- **b: `192.168.12.0/23`**

## 해설
1. 서브넷 마스크 `255.255.254.0`을 2진수로 변환하면 다음과 같다.
   `11111111.11111111.11111110.00000000` (앞에서부터 23비트가 네트워크 영역을 나타내므로 접미사는 `/23`이 된다)
2. 3번째 옥텟 단위로 호스트 주소와 서브넷 마스크에 대해 비트 논리곱(AND) 연산을 수행한다.
   - **호스트 a** (11): `00001011` AND `11111110` (서브넷) = `00001010` (10)
     따라서 네트워크 주소는 `192.168.10.0/23` 이 된다.
   - **호스트 b** (12): `00001100` AND `11111110` (서브넷) = `00001100` (12)
     따라서 네트워크 주소는 `192.168.12.0/23` 이 된다.

# 12. C언어 구조체 및 포인터 함수

## 문제
아래 코드를 확인하여 알맞는 출력값을 작성하시오.

```c
struct fns {
    int *(*fn)(int*);
} mine;

int *dummy(int *d) {
    return d + 1;
}

int main() {
    struct fns mine;
    int n[] = {16, 32};
    mine.fn = dummy;
    printf("%x", *mine.fn(n));
    return 0;
}
```

## 정답
**`20`**

## 해설
1. `n`은 정수형 배열 `{16, 32}`를 가리킨다. (10진수 32는 16진수로 `0x20`이다.)
2. `dummy` 함수는 주소 `d`에 `1`을 더한 값을 리턴한다. `int` 포인터 연산이므로 `d + 1`은 배열의 다음 원소 주소인 `&n[1]`이 된다.
3. `mine.fn(n)`의 결과값은 `n + 1` 주소이다. 이를 역참조(`*`)하므로 `n[1]`의 값인 `32`를 얻는다.
4. `printf("%x", ...)`에 의해 `32`를 16진수로 출력하므로 `20`이 화면에 표시된다.

# 14. Python 재귀 및 얕은 복사

## 문제
아래 파이썬 코드를 실행했을 때 출력되는 값을 쓰시오.

```python
def f(a):
    m = [[x] for x in a]
    b = m[:]
    for i in range(len(b) - 1):
        b[i+1] += b[i]
    return sum(len(x) for x in m)

print(f([1, 2, 3, 4]))
```

## 정답
**`10`**

## 해설
1. `m`은 `[[1], [2], [3], [4]]` 2차원 리스트이다.
2. `b = m[:]`을 수행하면 얕은 복사(Shallow Copy)가 일어나 `b`는 `m`과 별개의 리스트 객체가 되지만, 내부의 각 서브 리스트 객체(`[1]`, `[2]` 등)는 `m`과 `b`가 동일한 메모리 주소를 참조하게 된다.
3. 루프 동작 분석:
   - **i = 0**: `b[1] += b[0]` -> `b[1]`은 `[2] + [1] = [2, 1]`이 된다. 이때 `m[1]` 또한 동기화되어 `[2, 1]`로 바뀐다.
   - **i = 1**: `b[2] += b[1]` -> `b[2]`는 `[3] + [2, 1] = [3, 2, 1]`이 된다. `m[2]` 역시 `[3, 2, 1]`로 바뀐다.
   - **i = 2**: `b[3] += b[2]` -> `b[3]`은 `[4] + [3, 2, 1] = [4, 3, 2, 1]`이 된다. `m[3]` 역시 `[4, 3, 2, 1]`이 된다.
4. 결과적으로 `m` 리스트는 `[[1], [2, 1], [3, 2, 1], [4, 3, 2, 1]]`이 된다.
5. 각 원소의 길이는 1, 2, 3, 4이므로 이들의 총합은 **10**이다.

# 17. Java 연산 우선순위와 문자열 결합

public class Main {
    public static void main(String[] args) {

        int x2 = 2;

# 18. SQL JOIN 및 AVG 그룹연산

## 문제
아래 코드를 확인하여 알맞는 출력값을 작성하시오.

```sql
SELECT COUNT(*)
FROM employee e
JOIN dept d ON e.dep_id = d.dept_id
WHERE d.budget > (
    SELECT AVG(budget) FROM dept
);
```

## 정답
**`2`**

해당 쿼리는 부서 테이블의 예산 평균값(`AVG(budget)`)보다 예산이 더 큰 부서에 소속된 사원들의 총합 수를 연산하는 집계 쿼리이다. 서브쿼리를 통해 산출된 평균 예산 조건보다 예산이 많은 부서의 사원 수를 세어 최종 행의 개수 `2`를 반환한다.

# 19. 소프트웨어 테스트 스텁/드라이버

## 문제
다음은 통합 테스트에서 사용되는 더미 모듈에 대한 설명이다. 괄호 안에 들어갈 알맞은 용어를 쓰시오.
- ( 1 ) 은/는 하위 모듈을 대신하여 단순한 결과값만 반환하도록 임시로 작성된 더미 모듈로, 하향식 통합 테스트 수행 시 필요하다.
- ( 2 ) 은/는 상위 모듈을 대신하여 하위 모듈의 데이터 입력과 출력을 확인하기 위한 더미 모듈로, 상향식 통합 테스트 수행 시 필요하다.

## 정답
- **1: 스텁 (Stub)**
- **2: 드라이버 (Driver)**

## 해설
- **스텁(Stub)**: 상위 모듈을 테스트하기 위해 아직 개발되지 않은 가상의 하위 모듈 역할을 하는 모듈이다. (하향식 - Top Down 방식)
- **드라이버(Driver)**: 개발된 하위 모듈을 호출하고 파라미터를 넘겨주기 위한 가상의 상위 모듈 역할을 하는 모듈이다. (상향식 - Bottom Up 방식)

# 20. 모듈 응집도 유형

## 문제
다음은 응집도의 유형에 대한 설명이다. 괄호 안에 들어갈 알맞은 용어를 쓰시오.
- ( 1 ) 은/는 모듈이 다수의 관련 기능을 가질 때, 모듈 안의 구성요소들이 그 기능을 순차적으로 수행하는 경우의 응집도이다.
- ( 2 ) 은/는 동일한 입력과 출력을 사용하여 서로 다른 기능을 수행하는 활동들이 모여 있는 경우의 응집도이다.
- ( 3 ) 은/는 모듈 내부의 모든 기능이 단일한 목적을 위해 수행되는 경우의 응집도이다.

## 정답
- **1: 절차적 응집도 (Procedural Cohesion)**
- **2: 교환적 응집도 / 통신적 응집도 (Communication Cohesion)**
- **3: 기능적 응집도 (Functional Cohesion)**

## 해설
응집도의 강도 순서(낮음 -> 높음) 및 유형은 다음과 같다.
- **우연적 (Coincidental)**: 아무 관계 없는 요소들이 모임
- **논리적 (Logical)**: 유사한 성격의 기능들이 한 모듈에서 수행됨
- **시간적 (Temporal)**: 특정 시간대에 처리되는 기능들이 모임
- **절차적 (Procedural)**: 모듈 내부 구성요소들이 순차적 단계에 따라 실행됨
- **교환적 / 통신적 (Communication)**: 동일한 입출력 데이터를 사용하여 다른 기능을 수행함
- **순차적 (Sequential)**: 한 구성요소의 출력이 다음 구성요소의 입력으로 쓰임
- **기능적 (Functional)**: 모듈 내부 모든 기능이 하나의 단일 목표 및 기능을 위해 완벽하게 통합됨
