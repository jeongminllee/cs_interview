# 5. 조건문 (Python Conditional Statements)

## 5.1 if문과 코드 블록 (Code Blocks)

조건문은 특정 조건의 참/거짓 판정 여부에 따라 코드 실행의 흐름(Flow)을 다르게 제어하는 구문입니다.

### (1) if문의 기본 구조
```python
if 조건식:
    실행문(코드 블록)
```
* **if 키워드**: 조건문 선언을 시작하는 한정 기호.
* **조건식**: 논리 판정의 기준. 비교 연산이나 논리 연산 등을 통해 최종적으로 `True` 또는 `False` 값을 출력함.
* **콜론 (`:`)**: 조건식의 선언이 종료되고, 하위에 실행될 **코드 블록이 시작됨**을 파이썬 인터프리터에 지시하는 표식.
* **실행문**: 조건식의 결과가 `True`인 경우 가동되는 명령.

### (2) 들여쓰기(Indentation)를 통한 코드 블록 제어
파이썬은 타 언어(C/C++, Java)처럼 중괄호(`{ }`)로 코드의 단위를 묶지 않고, **오직 들여쓰기(공백)의 수준**만으로 코드 블록의 포함 종속 여부를 결정합니다.
* **4칸 들여쓰기**: 통상 키보드의 `[Tab]` 또는 스페이스 공백 4칸을 활용하여 들여씁니다.
* **일치성 규칙**: 같은 코드 블록에 속한 모든 명령어 행들은 **한 치의 오차도 없이 들여쓰기 공백 칸 수가 정확하게 일치**해야 합니다. 공백 개수가 한 칸이라도 들쭉날쭉하면 `IndentationError` 구문 오류가 발생하여 컴파일 단계에서 프로그램 가동이 중단됩니다.
* **코드 블록의 종료**: 들여쓰기를 더 이상 하지 않고 원래의 라인으로 복귀한 행부터 코드 블록이 완결 종료된 것으로 간주합니다.
* **한 줄 코드 블록**: 실행해야 할 명령이 단 한 줄뿐이라면, 굳이 줄바꿈하지 않고 콜론(`:`) 오른쪽에 즉각 기재하여 코드를 컴팩트하게 구성할 수 있습니다.
  ```python
  if score >= 90: print("우수")
  ```

## 5.2 양자택일(if~else)과 pass 키워드

### (1) if~else문
조건식이 참인 경우의 실행 경로와 거짓인 경우의 실행 경로 2가지 중 하나를 강제 선택하는 구조입니다.
```python
if score >= 60:
    print("합격")
else:
    print("불합격")
```

### (2) `pass` 키워드의 물리적 역할
프로그램 코딩 및 로직 설계 단계에서, 조건문 구조는 만들어 두었으나 **구체적으로 실행할 명령어 내용을 아직 결정하지 못했을 때** 사용하는 키워드입니다.
* **필요성**: 파이썬은 콜론(`:`) 아래의 들여쓰기 코드 블록 내부에 유효한 명령어가 한 줄도 기재되지 않으면 문법 규칙 위반으로 판단하여 즉각 구문 에러를 냅니다. 
* **해결책**: 아무 연산도 수행하지 않고 다음 단계로 흐름을 그냥 토스하는 `pass` 문을 삽입해 둠으로써 컴파일 에러 발생을 미연에 방지합니다.
  ```python
  if score >= 60:
      pass # 추후 알림 전송 로직 구현 예정 (에러 방지)
  else:
      print("불합격 통보")
  ```

## 5.3 다중 비교 (if~elif) 설계 시 주의사항

여러 개의 다양한 조건을 순차적으로 검증하여 다중 분기해야 하는 경우 `if ~ elif ~ else` 구문을 사용합니다.

### (1) 조건식 검사 순서의 중요성
인터프리터는 상단 `if`문부터 순차적으로 내려오며 조건식을 평가하고, **최초로 참(`True`)을 만족하는 단 하나의 실행 블록만 구동한 뒤 전체 조건문을 즉시 이탈**합니다. 따라서 조건들의 검사 순서 배치가 잘못 설계되면 하위 조건식이 아예 실행 기회를 잃는 논리적 오류가 초래됩니다.

#### [오류 사례] 학점 분류 시 잘못된 순서 설계
```python
# score = 95인 상황
if score >= 60: # 최초 조건 만족 -> 실행문 가동 후 이탈
    print('D')
elif score >= 70:
    print('C')
elif score >= 80:
    print('B')
elif score >= 90:
    print('A')
```
* *결과*: `score = 95`는 원래 `A`학점이어야 하지만, 맨 위 `score >= 60` 조건을 가장 먼저 참으로 통과해 버리므로 화면에는 잘못된 `'D'`가 출력됩니다.

### (2) 해결 전략
* **엄격한 하강 순서 배치**: 조건식을 설계할 때 가장 큰 범위 혹은 가장 높은 경계치 조건부터 내림차순(또는 오름차순) 순서대로 꼼꼼히 배치해야 합니다. (예: `score >= 90`을 가장 맨 위에 위치시킴)
* **논리 연산을 통한 범위의 명시적 고정**: 순서 배치 실수에 따른 오류를 미연에 방지하기 위해, 다음과 같이 논리 연산자(`and`)를 사용해 영역의 상한과 하한을 개별 조건식마다 직접 강제하는 안전 설계를 가할 수 있습니다.
  ```python
  if score >= 90:
      print('A')
  elif score >= 80 and score < 90:
      print('B')
  elif score >= 70 and score < 80:
      print('C')
  ```

## 5.4 중첩 조건문 (Nested Conditionals)

조건문 내부의 실행문 코드 블록 영역 안에 또 다른 제2의 조건문을 복수 계층으로 구성하는 설계 구조입니다.
* 바깥쪽 조건문이 참으로 평가되어 진입한 상태에서만 안쪽의 두 번째 조건 검사가 수행되므로, 세밀한 조건 필터링 단계 설계에 최적입니다. 단, 중첩이 너무 깊어지면 들여쓰기 공백이 과도하게 누적되어 코드 가독성이 나빠집니다.
