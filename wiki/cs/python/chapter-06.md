# 6. 반복문 (Python Loop Statements)

## 6.1 반복문의 필요성과 종류
반복문(Loop Statement)은 프로그램 내에서 동일하거나 유사한 명령을 조건에 따라 여러 번 반복하여 실행하도록 제어하는 제어문입니다. 반복 처리를 통하지 않으면 코드의 중복이 심해지고, 유지보수가 불가능할 정도로 코드 라인수가 증가하게 됩니다.

### (1) 반복문의 필요성
* **코드 가독성 및 효율성 극대화**: 예컨대 `"안녕하세요."`라는 문자열을 화면에 100번 출력해야 할 때, 반복문이 없다면 `print()` 함수를 100번 수동으로 작성해야 합니다. 만약 출력 횟수가 10,000번으로 늘어나거나 출력 메시지가 변경된다면 막대한 리소스 낭비가 발생합니다. 반복문을 사용하면 단 2~3줄의 코드로 이를 깔끔하게 처리할 수 있습니다.
* **동적 제어 가능**: 고정된 횟수가 아니라 사용자 입력이나 프로그램 실행 상태(파일 끝에 도달할 때까지 읽기, 특정 데이터가 검색될 때까지 탐색 등)에 따라 유연하게 반복 횟수를 조정할 수 있습니다.

### (2) 파이썬 반복문의 분류
파이썬은 제어 방식과 목적에 따라 크게 두 가지의 반복문을 제공합니다.
1. **for문 (횟수 및 컨테이너 순회 중심)**: 정해진 횟수만큼 반복하거나 리스트, 튜플, 문자열 등 반복 가능한 객체(Iterable)의 요소를 차례대로 꺼내어 실행 블록을 반복할 때 사용합니다.
2. **while문 (조건 조건문 중심)**: 지정한 조건식의 판정 결과가 참(`True`)인 동안 실행 블록을 계속해서 반복하며, 조건식이 거짓(`False`)이 되는 순간 반복 루프를 즉시 탈출합니다.

## 6.2 for문과 range() 함수

### (1) for문의 기본 구조와 작동 원리
```python
for 아이템 in 이터러블:
    실행문(코드 블록)
```
* **for~in 키워드**: 반복 순회 제어를 시작하는 한정 지시자입니다.
* **이터러블(Iterable)**: 리스트(List), 튜플(Tuple), 딕셔너리(Dictionary), 세트(Set), 문자열(String) 등 내부 요소를 차례대로 하나씩 꺼낼 수 있는 **반복 가능한 모든 객체**를 의미합니다.
* **아이템(Item)**: 이터러블 객체가 매 루프마다 꺼내어 제공하는 단일 데이터를 임시로 저장하는 변수(루프 제어 변수)입니다.
* **콜론(`:`)과 들여쓰기**: 코드 블록의 시작을 의미하며, 하위 실행문들은 반드시 동일한 수준으로 들여쓰기(통상 4칸 공백)되어야 합니다.

#### [원리 분석] 문자열 이터러블 순회
문자열은 각 문자들이 순서대로 나열된 이터러블 객체입니다. `for char in "Python":` 과 같이 작성하면, 변수 `char`에는 `'P'`, `'y'`, `'t'`, `'h'`, `'o'`, `'n'`이 루프마다 차례대로 한 자씩 할당되며, 실행문은 문자열의 길이인 총 6번 실행됩니다.
```python
for char in "Hello":
    print(char, end=" ")
# 출력 결과: H e l l o 
```

### (2) range() 함수를 통한 반복 범위 제어
`range()` 함수는 연속된 정수들의 시퀀스를 생성하는 파이썬 내장 이터러블 객체 제조기입니다. 메모리를 효율적으로 아끼기 위해 숫자를 한꺼번에 메모리에 올리지 않고, 루프가 돌 때마다 실시간으로 숫자를 생성하여 반환합니다.

`range()` 함수는 입력받는 인자(Argument)의 개수에 따라 총 3가지 형태로 작동합니다.

#### 1) 인자가 3개인 경우: `range(시작값, 끝값, 단계)`
* **구조**: `range(start, stop, step)`
* **규칙**: **시작값(start)**부터 **끝값(stop) 직전**까지 **단계(step)**의 간격으로 정수를 생성합니다.
* **주의**: 끝값 자체는 절대로 결과 시퀀스에 포함되지 않습니다. (수식적 범위: $start \le x < stop$)
```python
# 1부터 9까지 2씩 증가
for i in range(1, 10, 2):
    print(i, end=" ")
# 출력 결과: 1 3 5 7 9
```

#### 2) 인자가 2개인 경우: `range(시작값, 끝값)` (단계 생략)
* **구조**: `range(start, stop)`
* **규칙**: 단계(step) 인자가 생략되면 기본값인 `1`이 자동으로 적용되어 시작값부터 1씩 증가합니다.
```python
# 1부터 5까지 1씩 증가
for i in range(1, 6):
    print(i, end=" ")
# 출력 결과: 1 2 3 4 5
```

#### 3) 인자가 1개인 경우: `range(끝값)` (시작값, 단계 생략)
* **구조**: `range(stop)`
* **규칙**: 시작값은 기본값인 `0`, 단계는 기본값인 `1`이 적용되어, **0부터 (끝값 - 1)**까지 1씩 증가하는 정수열을 만듭니다.
```python
# 0부터 4까지 1씩 증가 (총 5회 반복)
for i in range(5):
    print(i, end=" ")
# 출력 결과: 0 1 2 3 4
```

### (3) 역방향(음수 단계) 반복 제어 규칙
단계를 음수(`-`)로 지정하면 숫자가 작아지는 감산 형태의 역방향 순회 루프를 구축할 수 있습니다.
* **증가 방향 (step > 0)**: 시작값에서 계속 더해가므로, `시작값 < 끝값` 이어야 정상 작동합니다. 반복 범위는 **(끝값 - 1)**까지입니다.
* **감소 방향 (step > 0)**: 시작값에서 계속 빼가므로, `시작값 > 끝값` 이어야 작동합니다. 반복 범위는 **(끝값 + 1)**까지입니다.
```python
# 5부터 1까지 1씩 감소
for i in range(5, 0, -1):
    print(i, end=" ")
# 출력 결과: 5 4 3 2 1
```
> [!WARNING]
> 역방향 범위 지정 시 `range(1, 5, -1)`과 같이 시작값이 끝값보다 작으면서 단계를 음수로 설정하면, 조건 만족이 불가능하므로 아무런 값도 생성하지 않고 즉시 루프가 끝납니다. (에러는 발생하지 않음)

## 6.3 while문

### (1) while문의 기본 구조
while문은 조건식이 참(`True`)을 만족하는 한 무한히 반복을 수행하는 조건 제어 루프입니다.
```python
while 조건식:
    실행문(코드 블록)
```
* **실행 순서**:
  1. 먼저 상단의 **조건식**을 평가합니다.
  2. 결과가 `True`이면 하위 코드 블록의 **실행문**을 가동합니다.
  3. 실행문이 끝나면 다시 상단으로 돌아가 **조건식**을 재평가합니다.
  4. 조건식이 `False`로 판정되는 순간 하위 실행문을 건너뛰고 while문 블록을 탈출합니다.

### (2) 제어 변수(Counter Variable) 관리 규칙
while문을 안전하게 제어하기 위해서는 다음 세 가지 필수 요소를 모두 갖춰야 합니다.
1. **제어 변수 초기화**: 루프 진입 전에 조건식에서 평가받을 변수를 미리 선언해야 합니다.
2. **조건식 평가**: 제어 변수의 값이 범위를 벗어났는지 검증할 수 있는 조건식을 작성해야 합니다.
3. **조건 변화식**: 실행문 블록 내부에 **제어 변수의 값을 변경시키는 코드**가 반드시 존재해야 합니다. (이 부분이 누락되면 무한 루프에 빠지게 됨)

```python
num = 1  # 1. 제어 변수 초기화
while num <= 5:  # 2. 조건식 평가
    print(num, end=" ")
    num += 1  # 3. 조건 변화식 (1씩 증가시켜 언젠가 num이 6이 되어 탈출하도록 유도)
# 출력 결과: 1 2 3 4 5
```

## 6.4 for문과 while문의 비교 분석

### (1) 동일 조건 구현 비교 (1부터 5까지 출력)
| 구분 | for문 구현 | while문 구현 |
| :--- | :--- | :--- |
| **코드** | ```python for i in range(1, 6): print(i) ``` | ```python i = 1 while i <= 5: print(i) i += 1 ``` |
| **변수 제어** | `range()` 내에서 시작, 끝, 단계가 일괄 관리됨. | 시작(루프 외부), 조건(while문), 단계(루프 내부)로 산재되어 관리됨. |

### (2) 차이점과 적정 선택 기준
* **for문이 적합한 경우**:
  * 반복할 횟수가 정해져 있거나, 데이터셋(리스트, 파일 줄 등)의 크기가 정해진 경우.
  * 복잡한 변수 증감 로직 없이 간결하게 순차 처리를 원할 때.
* **while문이 적합한 경우**:
  * 정확히 몇 번 돌려야 할지는 모르지만, **특정 조건이 충족될 때까지(또는 충족되지 않을 때까지)** 계속 가동해야 할 때.
  * 예: 사용자가 올바른 암호를 입력할 때까지 반복, 파일의 끝(`EOF`)에 도달할 때까지 읽기 등.

## 6.5 무한 루프 (Infinite Loops)
무한 루프란 조건문 탈출을 처리하는 로직이 없거나 조건식의 결과가 언제나 참(`True`)이어서 프로그램이 영원히 반복 실행을 멈추지 않는 비정상적인 상태를 의미합니다.

### (1) 발생 원인
* **의도치 않은 누락**: while문 실행부 내에 제어 변수를 업데이트하는 코드(`num += 1` 등)를 빼먹은 경우.
* **조건 설계 에러**: `while num < 10:` 조건인데 루프 내부에서 `num`을 계속 감소시켜 조건식이 계속 참이 되도록 만든 경우.
* **의도적 무한 루프**: 의도적으로 가동 상태를 유지해야 하는 시스템 서버, 게임 루프 등의 경우 `while True:` 형태로 구성 후 내부에서 `break` 키워드로 탈출 통로를 개방합니다.

```python
# 무한 루프 예시 (제어 변수 변화가 없음)
num = 1
while num < 3:
    print("안녕하세요.")
    # num += 1 이 없어서 num은 영원히 1임 -> 무한 루프
```
> [!CAUTION]
> 무한 루프에 빠진 경우 CPU 자원을 100% 점유하여 시스템 마비를 초래하므로 주의해야 합니다. 로컬 콘솔 환경에서 무한 루프가 발생하면 `Ctrl + C` 단축키를 눌러 인터프리터에 강제 인터럽트(KeyboardInterrupt)를 인가하여 프로세스를 강제 종료해야 합니다.

## 6.6 반복 제어 (break와 continue)

### (1) continue 키워드 (남은 루프 건너뛰기)
반복 블록 내에서 `continue`를 만나면, **그 뒤에 오는 모든 실행문들의 가동을 생략(Skip)하고 루프의 맨 처음으로 즉시 점프**합니다.
* for문: 다음 아이템을 변수에 담는 단계로 점프.
* while문: 상단 조건식 재평가 단계로 점프.

#### [예제] 1부터 10까지의 정수 중 홀수들의 누적 합 구하기
```python
total = 0
for num in range(1, 11):
    if num % 2 == 0:  # 짝수이면
        continue      # 아래 total += num 을 건너뛰고 다음 숫자로 점프
    total += num
print(total)
# 출력 결과: 25 (1 + 3 + 5 + 7 + 9)
```

### (2) break 키워드 (루프 강제 탈출)
루프 내부에서 `break`를 만나는 즉시, 반복 횟수나 남은 조건 판정 여부와는 아무런 상관없이 **가장 가까운 바깥쪽 반복문 블록을 즉시 완전히 탈출**하여 반복문 하위의 첫 코드로 진입합니다.

#### [예제] 1부터 10까지의 정수를 더해나가되, 누적 합이 30 이상이 될 때의 그 순간의 정수를 구하기
```python
total = 0
for num in range(1, 11):
    total += num
    if total >= 30:
        print(f"합이 {total}에 도달했을 때의 정수는 {num}")
        break  # 루프 강제 종료
# 출력 결과: 합이 36에 도달했을 때의 정수는 8
```

## 6.7 중첩 반복문 (Nested Loops)
반복문 실행 블록 내부에 또 다른 반복문이 위치하여 다차원 순회를 처리하는 제어 구조입니다. 바깥쪽 반복문이 1회 실행될 때마다, 안쪽 반복문은 지정된 횟수만큼 완전히 처음부터 끝까지 전체 루프를 돌게 됩니다.

따라서 전체 실행 횟수는 `(바깥 반복 횟수) * (안쪽 반복 횟수)` 가 됩니다.

### (1) 별(*) 찍기 예제 (행별 개수 제어)
* **목표**: 1행에는 `*` 1개, 2행에는 2개, ..., 5행에는 5개를 출력하는 역삼각형 구조 만들기.
```python
for i in range(1, 6):          # 바깥 루프: 행 제어 (1행 ~ 5행)
    for j in range(i):         # 안쪽 루프: 각 행당 출력할 별의 개수 (0부터 i-1까지, 즉 i개)
        print("*", end="")
    print()                    # 행이 바뀔 때 개행 처리
```
```text
*
**
***
****
*****
```

## 6.8 실전 문제 해결 (Practical Exercises)

### (1) 369 게임 만들기
* **요구사항**: 1부터 99까지 1씩 증가하면서 숫자에 3, 6, 9가 하나라도 포함될 때마다 숫자 대신 `'짝!'`을 출력하고, 그렇지 않으면 숫자를 그대로 출력하는 프로그램.
* **분석**: 숫자를 문자열로 변환한 뒤 `'3'`, `'6'`, `'9'`가 해당 문자열 내에 존재하는지(`in` 연산자 사용) 또는 카운트하여 판별합니다.
```python
for num in range(1, 100):
    num_str = str(num)

    if count > 0:
```

### (2) 열차 교차 시간 알아내기
* **요구사항**: 오전 9시부터 오후 6시(18시)까지 한빛역에서 3대의 열차(각각 A, B, C)가 출발합니다. 각 열차의 운행 주기(예: A열차는 10분, B열차는 15분, C열차는 20분 간격으로 출발)가 다를 때, 세 열차가 역에서 동시에 교차(동시 출발)하는 시각을 모두 알아내는 프로그램을 작성하십시오.
* **분석**: 
  - 오전 9시를 기준 시간 $0$분으로 둡니다.
  - 오후 6시는 기준 시간으로부터 총 $9 \text{시간} \times 60 \text{분} = 540$분 흐른 시점입니다.
  - 임의의 시간 $t$분($0 \le t \le 540$)에 대해, $t$가 세 열차 운행 주기의 공배수(즉, 각각의 주기로 나누어떨어지는 시점)일 때 동시 교차하게 됩니다.
```python
# 열차 출발 주기 (분 단위)
cycle_A = 10
cycle_B = 15
cycle_C = 20

start_hour = 9
end_hour = 18
total_minutes = (end_hour - start_hour) * 60  # 540분

for t in range(0, total_minutes + 1):
    if (t % cycle_A == 0) and (t % cycle_B == 0) and (t % cycle_C == 0):
        # 교차하는 분 단위를 시간과 분으로 재환산
        curr_hour = start_hour + (t // 60)
        curr_minute = t % 60
        print(f"동시 교차 시간: {curr_hour:02d}시 {curr_minute:02d}분")
```

### (3) 로그인 기능 만들기
* **요구사항**:
  - 사용자에게 비밀번호 입력을 받습니다.
  - 올바른 암호는 `'hanbitac'`입니다.
  - 암호가 틀렸다면 `"암호를 다시 확인하세요!"`를 출력하고 다시 비밀번호 입력을 요청합니다.
  - 암호가 올바르다면 `"로그인 됐습니다."`를 출력하고 프로그램을 안전히 종료합니다.
  - 단, 로그인 시도 횟수가 5회 이상을 초과하면 `"로그인 실패!! 횟수 초과!!!"` 메시지를 출력하고 루프를 탈출 및 강제 종료합니다.
```python
correct_pw = 'hanbitac'
fail_count = 0
max_attempts = 5

while True:
    user_pw = input("비밀번호를 입력하세요: ")
    
    if user_pw == correct_pw:
        print("로그인 됐습니다.")
        break
    else:
        fail_count += 1
        print("암호를 다시 확인하세요!")
        
        if fail_count >= max_attempts:
            print("로그인 실패!! 횟수 초과!!!")
            break
```
