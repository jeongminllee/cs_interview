# 2. 파이썬 첫걸음 (Introduction to Python and Environment)

## 2.1 파이썬 개론

### (1) 파이썬의 역사 및 개요
파이썬(Python)은 1991년 네덜란드의 개발자 귀도 반 로섬(Guido van Rossum)이 발표한 인터프리터식 객체지향 프로그래밍 언어입니다. 구글을 비롯한 글로벌 IT 기업들이 메인 시스템 언어로 채택하며 데이터 과학, AI, 웹 인프라 등 현대 소프트웨어 산업 전반의 핵심 언어로 군림하고 있습니다.

### (2) 파이썬의 컴퓨터 과학적 특징
* **무한 정수 처리 능력**: 고정된 하드웨어 비트수(예: 32비트, 64비트)에 의해 정수 상한선 오버플로우가 발생하는 C/C++ 등과 달리, 파이썬은 컴퓨터 시스템 메모리가 허용하는 한 자릿수의 제한 없이 무한 정수(Arbitrary-precision integers)를 연산할 수 있습니다.
* **풍부한 생태계(Library/Module)**: 전 세계 개발자 그룹에 의해 오픈소스로 배포된 라이브러리 생태계가 강력하여 빠른 프로토타이핑이 가능합니다.
* **직관적인 문법**: 인간의 사고 구조에 인접한 간결한 문법 구조를 제공하여 가독성이 높습니다.

## 2.3 파이썬 프로그램의 번역 및 실행 과정

컴퓨터 하드웨어는 $0$과 $1$로 구성된 기계어(Machine Code)만 이해하므로, 우리가 영어식으로 작성한 파이썬 소스 코드(`.py`)를 번역하는 엔진이 필수적입니다.

### (1) 컴파일러와 인터프리터의 동작 차이
* **컴파일러 (Compiler)**: 소스 코드 전체를 한 번에 검사하여 기계어로 구성된 물리적인 독립 실행 파일(`.exe` 등)을 생성하는 방식.
  * *특징*: 소스 변경 시 매번 재컴파일이 필요하지만, 가동 시 기계어가 즉시 실행되므로 구동 속도가 빠름.
* **인터프리터 (Interpreter)**: 별도의 실행 파일을 생성하지 않고, 소스 코드 라인 단위로 한 줄씩 실시간으로 번역하여 즉각 실행하는 방식.
  * *특징*: 소스 수정 시 컴파일 대기 없이 즉시 실행이 가능하여 개발 사이클이 빠르나, 컴파일 방식에 비해 런타임 실행 속도가 다소 느림.

### (2) 파이썬(CPython)의 내부 실행 파이프라인
파이썬은 단순한 인터프리터 언어가 아니며, 내부적으로 **컴파일러와 인터프리터가 혼합 결합된 하이브리드 아키텍처**로 구동됩니다. 파이썬의 표준 구현체인 **CPython**을 기준으로 실행 과정을 도식화하면 다음과 같습니다.

```
 [ 소스 코드 (*.py) ]
          │
          ▼  [ CPython 내장 컴파일러 ]
 [ 바이트 코드 (*.pyc) ]  <─── 가상 머신(PVM)이 인식하는 하드웨어 중립적 중간 코드
          │
          ▼  [ PVM (Python Virtual Machine) 인터프리터 ]
    [ 기계어 (0101) ]
          │
          ▼
   [ CPU 물리 실행 ]
```

1. **바이트 코드(Bytecode) 컴파일**:
   파이썬 소스 코드(`.py`)는 실행 시 먼저 C언어로 구현된 CPython 컴파일러에 의해 가상 머신용 중간 코드인 **바이트 코드(`.pyc`)**로 일괄 컴파일됩니다. 이 바이트 코드는 특정 하드웨어 기계에 종속되지 않는 범용 중간 표현(IR)입니다.
2. **PVM (Python Virtual Machine) 가상머신 구동**:
   가상 CPU 역할을 하는 PVM 내장 인터프리터가 바이트 코드를 라인 단위로 읽어 들이면서, 해당 컴퓨터 CPU가 직접 이해할 수 있는 이진 기계어로 변환하여 전송하고 물리 하드웨어에서 연산을 수행하도록 제어합니다.

이와 같이 파이썬은 **바이트 코드를 사전 생성하는 컴파일 언어인 동시에, 가상 머신 상에서 라인별로 번역을 수행하는 인터프리터 언어**의 특성을 모두 내포하고 있습니다.
