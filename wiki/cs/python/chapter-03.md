# 3. 데이터와 변수 (Data and Variables)

## 3.1 변수와 메모리 관리

### (1) 데이터와 정보의 정의
* **데이터 (Data)**: 물체나 현상을 관찰 및 측정하여 얻어낸 단순한 물리적 값.
* **정보 (Information)**: 개별 데이터들을 가공하고 유기적으로 엮어 의미를 도출하고 관리하기 유용하게 정렬해 놓은 가치 있는 상태.

### (2) 변수 (Variable)와 메모리 주소 매핑
컴퓨터 가동 시 로드되는 모든 데이터는 하드웨어 **메모리(RAM)** 상의 주소(Address) 공간에 개별 보관됩니다. 아파트에 동호수가 있듯이 메모리 공간에도 난해한 이진/16진수 물리 주소가 부여되는데, 사람이 매번 이 기계 주소를 직접 지칭하기 불가능하므로 특정 메모리 공간에 알기 쉬운 별명을 붙여 사용합니다. 이를 **변수(Variable)**라고 칭합니다.

### (3) 변수 선언 및 초기화의 원리
* **변수 선언**: 메모리에 데이터를 담을 수 있는 비어있는 공간을 할당하고 고유한 이름표를 부착하여 데이터 저장 준비를 수행하는 단계.
* **변수 초기화**: 할당된 변수 영역에 **할당 연산자 `=`**를 적용하여 최초의 실제 데이터를 대입하는 단계.
  * *할당 연산자의 대수적 의미*: 할당 연산자 `=`는 수학적 동치($==$)가 아니며, **"우측(R-value)의 값을 좌측(L-value)의 변수가 가리키는 메모리 방에 저장(대입)"**하라는 명령입니다.
  * *주의 사항*: 파이썬에서는 변수를 초기화하지 않고 선언만 가동하여 호출하면 메모리 값 부재로 인해 `NameError` 오류가 야기됩니다. 따라서 파이썬에서 변수는 **선언과 초기화가 하나의 세트**로 묶여 실행되어야 합니다.

## 3.2 변수명 명명 규칙 (Naming Conventions)

가독성 확보 및 구문 에러 예방을 위해 변수명 작성 시 다음 가이드를 준수해야 합니다.
1. **영문자로 명명하며 소문자로 시작**: 클래스(Class) 명명 규칙인 대문자 시작과 물리적으로 구분하기 위해 일반 변수는 항상 소문자로 출발합니다.
2. **낙타 표기법 (CamelCase)**: 두 개 이상의 영단어가 결합하여 변수명을 구성할 때, 첫 단어는 소문자로 두고 두 번째 단어부터 첫 문자만 대문자로 표기하여 가독성을 높입니다. (예: `userPhoneNumber`, `totalSalary`)
3. **예약어 (Reserved Words) 사용 불가**: 파이썬 인터프리터 컴파일 환경에서 이미 고유 명령어로 약속해 둔 단어들(예: `if`, `for`, `while`, `class`, `import`, `True`, `False` 등)은 변수명으로 절대 재정의할 수 없습니다.
4. **특수문자 제한**: **언더바(`_`)**를 제외한 특수문자(`$`, `@`, `!`, `#` 등)는 변수명에 삽입할 수 없습니다.
5. **공백 문자 불가**: 변수명 중간이나 앞뒤에 빈 칸(Space)이 섞이면 인터프리터 번역 과정에서 구문 에러(`SyntaxError`)가 강제 발생합니다.
6. **숫자 위치 제한**: 숫자는 변수명에 포함될 수 있으나, **절대로 첫 번째 글자 자리에 올 수 없습니다.** (예: `3score` $\rightarrow$ 에러, `score3` $\rightarrow$ 정상)

## 3.3 파이썬의 기본 자료형 (Data Types)

메모리를 보존하고 정밀도를 제어하기 위해 파이썬은 변수에 저장되는 값을 자동으로 추론하여 자료형(Data Type)을 지정(Dynamic Typing)합니다.

* **정수형 (int)**: 소수점이 존재하지 않는 임의 정밀도의 정수 자료형.
* **실수형 (float)**: 소수점이 포함된 실수 값.
  * *실수의 정밀도 한계*: 파이썬의 실수형은 64비트 배정도 규격을 적용하므로 **소수점 이하 16번째 자리까지만 정확하게 보존**하고, 17번째 자리 이하의 데이터 값은 부동 소수점 근사 오차로 인해 데이터가 유실(Trashing)됩니다.
* **문자열형 (str)**: 작은따옴표(`' '`) 혹은 큰따옴표(`" "`)로 감싸 정의하는 문자 데이터 집합. 한 글자(Character)와 여러 글자의 문자열을 동일하게 취급합니다.
* **논리형 (bool)**: 참(`True`)과 거짓(`False`)을 나타내는 부울 대수 데이터.

---

## 3.4 자료형 조회 및 형 변환 (Type Casting)

### (1) 자료형 조회 `type()`
임의의 변수에 저장된 실시간 자료형을 검증할 때 내장 함수 `type(변수명)`을 사용합니다.
```python
var = '777'
print(type(var)) # 출력: <class 'str'>
```

### (2) 형 변환 함수의 수학적 조건
* `int()`: 소수점이 포함된 실수를 정수로 캐스팅하면 소수점 이하의 정보는 절사(Truncation) 유실됩니다.
* `bool()`: 다른 타입의 데이터를 논리형으로 변환할 때의 정밀 규칙은 다음과 같습니다.
  * **문자열 $\rightarrow$ 논리형**:
    * 아무 글자도 없는 **빈 문자열 `''`**을 넣을 때만 오직 `False`로 매핑됩니다.
    * 스페이스 공백이 포함된 문자열 `' '`은 빈 문자가 아니므로 변환 시 **`True`**를 반환합니다.
    * 그 외 값이 존재하는 모든 문자열은 `True`로 수렴합니다.
  * **숫자 $\rightarrow$ 논리형**: 오직 $0$ 또는 $0.0$만이 `False`가 되며, $0$이 아닌 양수/음수는 모두 `True`가 됩니다.

## 3.5 데이터 입력 제어 `input()`

사용자 키보드 인터페이스로부터 데이터를 수신하는 핵심 API입니다.

```python
userData = input("나이를 입력하세요: ")
```

* **input()의 데이터형 고정 규칙**:
  `input()` 함수를 통해 입력받아 적재된 데이터는 사용자가 숫자를 타이핑하더라도 **무조건 문자열형(`str`)으로 고정 처리**됩니다.
* **오버헤드 대응**:
  따라서 입력받은 수치를 사칙 산술 연산 등에 즉시 활용하고자 한다면 반드시 명시적인 형 변환 캐스팅 함수를 중첩 적용하여 숫자로 변경해주어야 연산 에러를 막을 수 있습니다.
  ```python
  age = int(input("나이를 입력하세요: ")) # 정수로 형 변환
  ```
