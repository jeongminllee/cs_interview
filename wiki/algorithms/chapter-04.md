# 4. 정렬 (Sorting Algorithms)

## 4.1 정렬 알고리즘의 계산 복잡도 분류

내부 정렬 알고리즘들은 데이터의 배치 상태(정렬 완료 상태 여부)에 따라 비교 연산 횟수가 달라지므로, 최선(Best), 평균(Average), 최악(Worst) 케이스의 점근적 시간 복잡도를 명확히 파악하는 것이 중요합니다.

| 알고리즘 | 최선 시간복잡도 | 평균 시간복잡도 | 최악 시간복잡도 | 공간복잡도 | 특징 및 안정성 (Stable) |
| :--- | :---: | :---: | :---: | :---: | :--- |
| **버블 정렬** | $\mathcal{O}(n)$ | $\mathcal{O}(n^2)$ | $\mathcal{O}(n^2)$ | $\mathcal{O}(1)$ | 인접 원소 비교 교환, 안정 정렬 (Stable) |
| **선택 정렬** | $\mathcal{O}(n^2)$ | $\mathcal{O}(n^2)$ | $\mathcal{O}(n^2)$ | $\mathcal{O}(1)$ | 최솟값 선별 교환, 불안정 정렬 (Unstable) |
| **삽입 정렬** | $\mathcal{O}(n)$ | $\mathcal{O}(n^2)$ | $\mathcal{O}(n^2)$ | $\mathcal{O}(1)$ | 이미 거의 정렬된 상태일 때 최상의 속도, Stable |
| **쉘 정렬** | $\mathcal{O}(n)$ | $\mathcal{O}(n^{1.5})$ | $\mathcal{O}(n^2)$ | $\mathcal{O}(1)$ | 삽입 정렬의 Gap 기반 응용 개량형, Unstable |
| **퀵 정렬** | $\mathcal{O}(n \log n)$ | $\mathcal{O}(n \log n)$ | $\mathcal{O}(n^2)$ | $\mathcal{O}(\log n)$ | 분할 정복 기법, 평균 속도 최상, Unstable |
| **힙 정렬** | $\mathcal{O}(n \log n)$ | $\mathcal{O}(n \log n)$ | $\mathcal{O}(n \log n)$ | $\mathcal{O}(1)$ | 완전 이진트리 힙 구성, 항상 균등한 성능, Unstable |
| **병합 정렬** | $\mathcal{O}(n \log n)$ | $\mathcal{O}(n \log n)$ | $\mathcal{O}(n \log n)$ | $\mathcal{O}(n)$ | 분할 정복 기법, 추가 메모리 필요, Stable |
| **기수 정렬** | $\mathcal{O}(d \cdot n)$ | $\mathcal{O}(d \cdot n)$ | $\mathcal{O}(d \cdot n)$ | $\mathcal{O}(n + k)$ | 자릿수별 분배 비교 배제, 값 분포 지배, Stable |

## 4.2 주요 정렬 알고리즘 작동 정밀 추적

### (1) 버블 정렬 (Bubble Sort) 패스별 추적
* **초기 데이터**: `[9, 6, 7, 3, 5]` (오름차순 정렬 대상)
* **정밀 추적 과정**:
  * **PASS 1**:
    * $9$와 $6$ 비교 $\rightarrow$ 교환 $\rightarrow$ `[6, 9, 7, 3, 5]`
    * $9$와 $7$ 비교 $\rightarrow$ 교환 $\rightarrow$ `[6, 7, 9, 3, 5]`
    * $9$와 $3$ 비교 $\rightarrow$ 교환 $\rightarrow$ `[6, 7, 3, 9, 5]`
    * $9$와 $5$ 비교 $\rightarrow$ 교환 $\rightarrow$ `[6, 7, 3, 5, 9]` $\rightarrow$ **최대치 9가 맨 끝에 최종 고정됨.**
  * **PASS 2** (앞 4개 데이터 대상):
    * $6$과 $7$ 비교 $\rightarrow$ 정렬 완료 상태이므로 유지 $\rightarrow$ `[6, 7, 3, 5, 9]`
    * $7$와 $3$ 비교 $\rightarrow$ 교환 $\rightarrow$ `[6, 3, 7, 5, 9]`
    * $7$와 $5$ 비교 $\rightarrow$ 교환 $\rightarrow$ `[6, 3, 5, 7, 9]` $\rightarrow$ **두 번째 최대치 7이 끝에서 두 번째에 고정됨.**
* **PASS 2 완료 결과**: `[6, 3, 5, 7, 9]`

### (2) 선택 정렬 (Selection Sort) 회전별 추적
* **초기 데이터**: `[37, 14, 17, 40, 35]` (오름차순 정렬 대상)
* **정밀 추적 과정**:
  * **1회전**:
    * 전체 영역 `[37, 14, 17, 40, 35]`에서 최소 원소 탐색 $\rightarrow$ 최솟값은 `14` (인덱스 1).
    * `14`와 정렬 미완료 영역의 첫 번째 값인 `37` (인덱스 0)을 맞교환.
    * 결과: `[14, 37, 17, 40, 35]`
  * **2회전**:
    * 인덱스 1부터 끝까지 `[37, 17, 40, 35]` 영역에서 최소 원소 탐색 $\rightarrow$ 최솟값은 `17` (인덱스 2).
    * `17`과 미완료 영역의 첫 번째 값인 `37` (인덱스 1)을 맞교환.
    * 결과: `[14, 17, 37, 40, 35]`
  * **3회전**:
    * 인덱스 2부터 끝까지 `[37, 40, 35]` 영역에서 최소 원소 탐색 $\rightarrow$ 최솟값은 `35` (인덱스 4).
    * `35`와 미완료 영역의 첫 번째 값인 `37` (인덱스 2)을 맞교환.
    * 결과: `[14, 17, 35, 40, 37]`
* **3회전 완료 결과**: `[14, 17, 35, 40, 37]`

### (3) 삽입 정렬 (Insertion Sort) 회전별 추적
* **초기 데이터**: `[8, 3, 4, 9, 7]` (오름차순 정렬 대상)
* **정밀 추적 과정**:
  * **1회전**:
    * 정렬 완료된 앞의 영역인 `[8]` 뒤에 위치한 두 번째 원소인 키값 `3`을 올바른 자리에 삽입하는 과정입니다.
    * 키 `3`을 앞의 `8`과 비교합니다. $3 < 8$이므로 `8`을 뒤로 한 칸 밀어내고, 빈자리인 맨 앞에 `3`을 안착시킵니다.
    * 결과: `[3, 8, 4, 9, 7]`
* **1회전 완료 결과**: `[3, 8, 4, 9, 7]`

### (4) 힙 정렬 (Heap Sort)의 초기 Max Heap 구축 과정
* **초기 완전 이진트리 원소 상태**: `[5, 3, 8, 4, 9, 1, 6, 2, 7]` (총 $N=9$개 노드)
* **인덱스 기하학적 구조**:
  ```
              5 (index 1)
            /   \
           3     8
          / \   / \
         4   9 1   6
        / \
       2   7
  ```
* **초기 최대 힙(Max Heap) 구축(Heapify) 절차**:
  자식이 존재하는 마지막 부모 노드 번호인 $N // 2 = 9 // 2 = 4$번째 노드부터 역순으로 루트 노드(1번째)까지 Heapify 조건(부모 노드의 값이 자식 노드의 값보다 항상 커야 함)을 적용하여 올립니다.
  1. **4번째 부모 노드 (값 4, 인덱스 4)**: 
     자식은 8번째(값 2)와 9번째(값 7)입니다. 자식 중 최댓값은 `7`이므로 부모 `4`와 교환합니다.
     * 트리 상태: `4`번 노드의 값이 `7`로, `9`번 노드의 값이 `4`로 교환됨.
  2. **3번째 부모 노드 (값 8, 인덱스 3)**:
     자식은 6번째(값 1)와 7번째(값 6)입니다. 부모인 `8`이 자식들보다 이미 크므로 변화 없이 유지됩니다.
  3. **2번째 부모 노드 (값 3, 인덱스 2)**:
     자식은 4번째(값 7로 업데이트됨)와 5번째(값 9)입니다. 자식 중 최댓값은 `9`이므로 부모 `3`과 교환합니다.
     * 트리 상태: `2`번 노드의 값이 `9`로, `5`번 노드의 값이 `3`으로 교환됨.
  4. **1번째 루트 노드 (값 5, 인덱스 1)**:
     자식은 2번째(값 9로 업데이트됨)와 3번째(값 8)입니다. 자식 중 최댓값은 `9`이므로 루트 `5`와 교환합니다.
     * 교환 후 루트 노드 값은 `9`가 되고, 2번째 자식 노드 값은 `5`가 됩니다.
     * **자식 서브트리로 내려가며 연속 Heapify 수행**: 내려간 2번째 노드(값 5)의 자식인 4번째(값 7)와 5번째(값 3)를 비교합니다. 최댓값인 `7`이 부모 `5`보다 크므로 교환합니다.
     * 교환 후 4번째 노드 값은 `5`가 됩니다. 4번째 노드의 자식인 8번째(값 2)와 9번째(값 4)를 비교합니다. `4`가 더 크지만 부모 `5`가 자식들보다 이미 크므로 최종 멈춥니다.
* **최종 완성된 초기 Max Heap 트리 구조**:
  ```
              9
            /   \
           7     8
          / \   / \
         5   3 1   6
        / \
       2   4
  ```

### (5) 퀵 정렬 (Quick Sort) low/high 포인터 정밀 교환 추적
* **초기 데이터**: `[71, 49, 92, 55, 38, 82, 72, 53]`
* **피벗 (Pivot)**: 리스트의 첫 번째 원소인 `71`
* **low/high 제어 알고리즘 세부 추적 과정**:
  * **초기 포인터 설정**: `low`는 피벗 바로 우측인 `49` (인덱스 1)에 위치, `high`는 맨 끝 원소인 `53` (인덱스 7)에 위치합니다.
  * **1단계 포인터 이동**:
    * `low` 포인터는 오른쪽으로 이동하며 피벗 `71`보다 큰 값을 찾습니다. `49`는 작으므로 패스, `92`가 크므로 **`92` (인덱스 2)에서 멈춥니다.**
    * `high` 포인터는 왼쪽으로 이동하며 피벗 `71`보다 작은 값을 찾습니다. `53`이 피벗보다 작으므로 이동하지 않고 즉시 **`53` (인덱스 7)에서 멈춥니다.**
    * `low < high` (인덱스 $2 < 7$)이므로 두 값을 교환합니다.
    * 결과: `[71, 49, 53, 55, 38, 82, 72, 92]`
  * **2단계 포인터 이동**:
    * `low`를 다시 인덱스 3(`55`)부터 출발시킵니다. `55` 패스, `38` 패스, `82`가 피벗보다 크므로 **`82` (인덱스 5)에서 멈춥니다.**
    * `high`를 다시 인덱스 6(`72`)부터 출발시킵니다. `72`는 크므로 패스, `82` 패스, `38`이 피벗보다 작으므로 **`38` (인덱스 4)에서 멈춥니다.**
    * 이제 두 포인터가 서로 교차하여 엇갈렸습니다 (`low` 인덱스 5 > `high` 인덱스 4).
    * 조건에 따라 교환 루프를 중단하고, 피벗 `71`과 `high` 포인터 위치의 값인 `38`을 맞교환합니다.
    * 결과: `[38, 49, 53, 55, 71, 82, 72, 92]`
* **1회전 퀵 분할 완료 결과**: `[38, 49, 53, 55, 71, 82, 72, 92]` (피벗 `71`은 중앙에 완전히 정렬 고정됨)
