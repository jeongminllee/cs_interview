# 3. 비선형구조 (Non-Linear Data Structures)

## 3.1 트리 (Tree)

### (1) 이진 트리의 수학적 특성 증명
모든 노드가 최대 2개의 자식만을 가질 수 있도록 제한된 **이진 트리(Binary Tree)**는 다음과 같은 엄밀한 수학적 정리들이 항상 만족됩니다.

#### 정리 3.1 (단말 노드 수와 2도 노드 수의 일치 관계)
이진 트리 내의 전체 단말 노드(Leaf node, 차수가 0인 노드)의 개수를 $n_0$라 하고, 자식 노드를 2개 가진 노드의 개수를 $n_2$라 할 때, 항상 다음 관계식이 성립합니다.
$$n_0 = n_2 + 1$$

#### 대수적 증명
트리 내의 전체 노드 개수를 $N$, 전체 간선(Edge)의 개수를 $B$라고 선언합시다. 
1. **노드와 간선 수의 기본적인 위상 관계**:
   루트 노드를 제외한 트리의 모든 노드는 자신으로 들어오는 단 하나의 부모 간선을 가집니다. 따라서 전체 노드 수와 간선 수 사이에는 다음 관계가 항상 만족됩니다.
   $$N = B + 1 \tag{3.1}$$
2. **노드의 차수(자식 수)를 기준으로 한 총 노드 수의 합산**:
   트리의 노드들은 자식의 수(차수)에 따라 $0$도 노드($n_0$), $1$도 노($n_1$), $2$도 노드($n_2$)의 3가지 범주로 나뉘므로, 전체 노드 수는 다음과 같습니다.
   $$N = n_0 + n_1 + n_2 \tag{3.2}$$
3. **간선 수를 자식 수 관점에서 합산**:
   각 노드에서 뻗어 나가는 자식 간선 수의 총합이 전체 간선의 총합 $B$가 됩니다. 즉, 2도 노드는 각각 2개의 간선을 뻗치고, 1도 노드는 1개의 간선을 뻗치며, 단말 노드는 간선을 뻗치지 않습니다.
   $$B = 2n_2 + 1n_1 + 0n_0 = 2n_2 + n_1 \tag{3.3}$$
4. **대입을 통한 일치식 도출**:
   식 (3.3)을 식 (3.1)에 대입합니다.
   $$N = (2n_2 + n_1) + 1 \tag{3.4}$$
   식 (3.2)의 $N$과 식 (3.4)의 $N$은 서로 같아야 하므로 연립하여 정리합니다.
   $$n_0 + n_1 + n_2 = 2n_2 + n_1 + 1$$
   양변에서 공통 항 $n_1$을 소거하고 좌변의 $n_2$를 우변으로 이항합니다.
   $$n_0 = n_2 + 1$$
   이로써 임의의 모든 이진 트리에서 단말 노드의 개수가 2도 노드의 개수보다 항상 1개 더 많음이 수학적으로 엄밀히 증명됩니다.

### (2) 수식 표기법 변환 문제 정밀 풀이

#### [문제] 중위 표기식(Infix) `X = A / B * (C + D) + E`를 전위(Prefix) 및 후위(Postfix)로 변환
* **대수적 연산 우선순위가 반영된 전체 괄호화**:
  1. 가장 높은 우선순위 괄호 계산: `(C + D)`
  2. 곱셈과 나눗셈은 왼쪽에서 오른쪽 순으로 연산: `(A / B)`를 먼저 계산하고, 이 결과와 앞선 괄호의 곱셈 `((A / B) * (C + D))`을 처리.
  3. 덧셈 연산 처리: `(((A / B) * (C + D)) + E)`
  4. 대입 연산 처리: `(X = (((A / B) * (C + D)) + E))`

#### ① 전위 표기법 (Prefix) 변환 과정 (연산자를 괄호 앞으로 이동)
* 대상 식: `(X = (((A / B) * (C + D)) + E))`
* 내부부터 연산자 이동:
  * `(A / B)` $\rightarrow$ `/ A B`
  * `(C + D)` $\rightarrow$ `+ C D`
  * `((/ A B) * (+ C D))` $\rightarrow$ `* / A B + C D`
  * `((* / A B + C D) + E)` $\rightarrow$ `+ * / A B + C D E`
  * `(X = (+ * / A B + C D E))` $\rightarrow$ `= X + * / A B + C D E`
* **최종 전위식 결과**: `= X + * / A B + C D E`

#### ② 후위 표기법 (Postfix) 변환 과정 (연산자를 괄호 뒤로 이동)
* 대상 식: `(X = (((A / B) * (C + D)) + E))`
* 내부부터 연산자 이동:
  * `(A / B)` $\rightarrow$ `A B /`
  * `(C + D)` $\rightarrow$ `C D +`
  * `((A B /) * (C D +))` $\rightarrow$ `A B / C D + *`
  * `((A B / C D + *) + E)` $\rightarrow$ `A B / C D + * E +`
  * `(X = (A B / C D + * E +))` $\rightarrow$ `X A B / C D + * E + =`
* **최종 후위식 결과**: `X A B / C D + * E + =`

## 3.2 그래프 (Graph)

### (1) 완전 그래프(Complete Graph)의 간선 수 공식 유도
* **완전 그래프**: 그래프 내의 모든 정점 쌍 사이에 직접 연결된 간선이 존재하는 연결 형태.

```
       [무방향 완전 그래프 (정점 n=4)]
       
                 A ─────── B
                 │ ╲     ╱ │
                 │   ╲ ╱   │
                 │   ╱ ╲   │
                 │ ╱     ╲ │
                 D ─────── C
                 
       * 간선 수 = 4 * 3 / 2 = 6개
```

* **무방향 완전 그래프 (Undirected Complete Graph)**:
  각 정점 $n$은 자신을 제외한 나머지 $n-1$개의 정점과 직접 간선으로 연결됩니다. 따라서 전체 정점에서 뻗어 나가는 가지의 총합은 $n(n-1)$입니다. 무방향 그래프에서는 임의의 정점 $A$에서 $B$로 가는 간선과 $B$에서 $A$로 오는 간선이 동일한 하나의 간선이므로 2로 나누어 중복을 해제합니다.
  $$\text{총 간선 수} = \frac{n(n-1)}{2}$$
* **방향 완전 그래프 (Directed Complete Graph)**:
  방향 그래프에서는 $\langle A, B \rangle$와 $\langle B, A \rangle$를 서로 독립된 서로 다른 간선으로 구별하므로, 중복 제거를 위한 나누기 2 연산을 수행하지 않습니다.
  $$\text{총 간선 수} = n(n-1)$$

### (2) 그래프 탐색 (DFS 및 BFS) 작동 메커니즘
* **깊이 우선 탐색 (DFS - Depth First Search)**:
  임의의 정점에서 출발하여 갈 수 있는 한 깊게 경로를 타고 들어가다가, 더 이상 갈 곳이 없는 단말 정점에 도달하면 가장 최근의 분기점으로 되돌아와(Backtracking) 다른 방향을 탐색하는 기법.
  * **핵심 도구**: **스택 (Stack)**을 통해 경로 복귀 점들을 관리하거나, 컴파일러의 함수 호출 재귀(Recursion)를 활용하여 자연스럽게 구현합니다.
* **너비 우선 탐색 (BFS - Breadth First Search)**:
  시작 정점과 인접해 있는 가까운 모든 이웃 정점들을 최우선으로 다 방문한 뒤, 다음 깊이 레벨의 정점들로 점진적으로 탐색 범위를 원형으로 확장해 나가는 기법.
  * **핵심 도구**: **큐 (Queue)**를 사용하여 다음에 방문할 정점 목록을 선입선출 대기 행렬로 적재 관리합니다. 최단 경로 탐색에 탁월한 특징이 있습니다.
