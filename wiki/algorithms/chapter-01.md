# 1. 자료구조 개념과 알고리즘 (Introduction to Data Structures and Algorithms)

## 1.1 자료구조의 기본 개념

### (1) 자료와 정보의 정의
* **자료 (Data)**: 현실 세계에서 단순한 관찰이나 측정, 조사를 통해 수집한 가공되지 않은 사실(Fact)이나 값(Value).
* **정보 (Information)**: 수집된 자료를 특정 목적에 맞게 정리, 처리, 가공하여 의사결정에 직간접적인 도움을 주는 유용한 지식.
* **정보 처리의 수학적 모델**:
  $$P(\text{Data}) = \text{Information} \quad (\text{단, } P\text{는 컴퓨터의 연산/가공 처리 과정})$$

### (2) 자료구조 (Data Structure)의 필요성
자료구조는 컴퓨터 내에서 대량의 데이터를 체계적이고 효율적으로 표현하고 저장하기 위한 수학적/대수학적 모델입니다. 자료구조의 선택은 프로그램의 메모리 사용 효율성뿐 아니라, 실행 속도 및 성능에 직접적인 오버헤드를 발생시키므로 문제의 성격에 맞게 정밀하게 설계되어야 합니다.

### (3) 자료구조의 형태 분류
```
                      자료구조 (Data Structure)
                                  │
      ┌───────────────────────────┼───────────────────────────┐
      ▼                           ▼                           ▼
 단순 자료구조               선형 자료구조               비선형 자료구조
 정수, 실수,                 자료 간 1:1 관계            자료 간 1:N / N:N
 문자, 문자열                (리스트, 스택,              (트리, 그래프)
                             큐, 데크)
```
* **단순 자료구조**: 프로그래밍 언어에서 기본으로 제공하는 정수(Integer), 실수(Real), 문자(Character), 문자열(String) 등.
* **선형 자료구조 (Linear Data Structure)**: 자료들 간의 관계가 1대 1로 인접해 있는 구조 (배열/리스트, 스택, 큐, 데크).
* **비선형 자료구조 (Non-Linear Data Structure)**: 자료 간의 관계가 1대 다 혹은 다대 다의 계층/망 구조를 형성 (트리, 그래프).
* **파일 자료구조 (File Structure)**: 컴퓨터의 주기억장치(RAM)가 아닌 물리적 보조기억장치(하드디스크, SSD)에 보관되는 레코드들의 집합 구조 (순차 파일, 색인 파일, 직접 파일).

### (4) 자료의 구성 단위 (물리적/논리적 크기 계층)
1. **비트 (Bit)**: 이진수 $0$ 또는 $1$ 한 자리를 기록할 수 있는 물리적 최소 단위.
2. **바이트 (Byte)**: 8개의 비트가 모여 형성된 단위. 영문자나 숫자 1자를 표현하는 기본 단위로, 총 $2^8 = 256$가지의 상태를 구별하여 표현함. (4비트는 **니블 (Nibble)**이라고 함)
3. **단어 (Word)**: CPU가 버스를 통해 한 번에 가져와 연산 제어할 수 있는 명령 처리 단위.
   * 반단어 (Half Word): 2 Byte
   * 전단어 (Full Word): 4 Byte
   * 배단어 (Double Word): 8 Byte
4. **필드 (Field / Item)**: 의미 있는 정보의 최소 표현 단위. 데이터베이스나 스프레드시트의 **열 (Column)**에 해당함.
5. **레코드 (Record)**: 연관된 필드들의 집합체. 자료를 처리하고 연산하는 논리적 기본 단위이며, 스프레드시트의 **행 (Row)**에 해당함.
   * *논리 레코드 (Logical Record)*: 프로그래머가 코드 상에서 선언하여 사용하는 레코드 단위.
   * *물리 레코드 (Physical Record / Block)*: 입출력 장치와 디스크 간에 실제로 한 번에 데이터를 전송하는 물리적 입출력 단위.
6. **파일 (File)**: 연관된 레코드들이 보조기억장치 상에 모여 있는 최종 집합체.

## 1.2 자료의 컴퓨터 내부 표현

### (1) 정수의 표현 방식 (Integer Representation)

#### 10진 표현법 (Decimal Format)
* **존 형식 (Zone / Unpack Format)**: 1바이트(8비트) 공간에 10진수 숫자 한 자리를 저장하는 방식.
  * 상위 4비트는 **존 비트(Zone Bit)**로 항상 `1111`($16$진수 `F`)로 채웁니다.
  * 하위 4비트는 **디지트 비트(Digit Bit)**로 숫자의 2진 표현을 담습니다.
  * 최우측 바이트의 존 영역 4비트 자리는 **부호 비트**로 활용됩니다.
    * 양수($+$): `1100` ($16$진수 `C`)
    * 음수($-$): `1101` ($16$진수 `D`)
  * *존 형식 변환 예시*:
    * $+543 \rightarrow \text{F5 F4 C3}$
    * $-543 \rightarrow \text{F5 F4 D3}$
* **팩 형식 (Pack Format)**: 1바이트 공간에 10진수 숫자 두 자리를 모아 팩킹하여 효율적으로 저장하는 방식.
  * 1바이트의 상위 4비트와 하위 4비트에 각각 숫자의 2진 표현을 밀어 넣습니다.
  * 숫자의 가장 끝인 최우측 하위 4비트(마지막 니블) 공간에만 부호를 기록합니다 (양수: `C`, 음수: `D`).
  * *팩 형식 변환 예시*:
    * $+543 \rightarrow \text{00 00 54 3C}$ (빈 앞 공간은 0으로 패딩)
    * $-543 \rightarrow \text{00 00 54 3D}$

#### 고정 소수점 표현법 (Fixed Point - 2진 정수 표현)
주어진 $n$비트 컴퓨터 레지스터 공간에서 고정 소수점을 표현할 때, 최상위 1비트(MSB)는 **부호 비트 (Sign)**(양수 0, 음수 1)로 강제 고정하고 나머지 $n-1$비트로 정수 값을 채웁니다.
* **부호와 절대치 방식**: 단순히 부호 비트만 음수 1로 가동하고 값 부분은 양수와 동일하게 두는 방식.
  * 표현 범위: $-(2^{n-1}-1) \sim 2^{n-1}-1$
  * 단점: $+0$(`000...0`)과 $-0$(`100...0`) 두 개의 0이 물리적으로 존재하여 메모리가 낭비되며 연산 장치 제어가 복잡함.
* **1의 보수 방식**: 음수 값을 표현하기 위해 양수의 모든 비트를 반전시키는 방식.
  * 표현 범위: $-(2^{n-1}-1) \sim 2^{n-1}-1$
  * 단점: 여전히 $+0$과 $-0$이 공존함.
* **2의 보수 방식**: 1의 보수 계산 결과값에 물리적으로 1을 가산하는 방식.
  * 표현 범위: $-2^{n-1} \sim 2^{n-1}-1$
  * 장점: **0이 단 하나만 존재**하며, 가산기 회로(Adder) 하나만으로 뺄셈 연산까지 완벽히 수행 가능하여 현대 컴퓨터 아키텍처의 표준으로 군림함.

### (2) 실수의 표현 방식 (부동 소수점)

실수는 가수부(Fraction)와 지수부(Exponent)를 나누어 표현하는 **부동 소수점(Floating Point)** 기법을 사용하여 넓은 정밀도 영역을 커버합니다.

#### IEEE 754 단정도(Single-Precision) 32비트 표준 포맷
```
 ┌───┬──────────────┬────────────────────────┐
 │ S │  Exponent(E) │        Fraction(M)     │
 └───┴──────────────┴────────────────────────┘
 1bit     8bits              23bits
```
1. **부호 비트 (S)**: 1비트 (양수 0, 음수 1)
2. **지수부 (E)**: 8비트. 음수 지수 표현을 허용하기 위해 실제 지수값 $e$에 바이어스(Bias) 상수 $127$을 더한 $E = e + 127$ 값을 저장함.
3. **가수부 (M)**: 23비트. 이진수를 항상 $1.XXXX_{(2)}$ 형태로 정규화(Normalization)하여 소수점 아래 $XXXX$ 부분만 23비트로 채워 저장함.

#### IEEE 754 실수 변환 연습문제 풀이 과정
* **대상 정밀 실수**: $+13.625$
* **수학적 전개 단계**:
  1. **정수부와 소수부의 2진수 개별 변환**:
     * 정수부: $13 = 8 + 4 + 1 = 1101_{(2)}$
     * 소수부: $0.625 = 0.5 + 0.125 = 2^{-1} + 2^{-3} = 0.101_{(2)}$
     * 결합된 이진 표현: $1101.101_{(2)}$
  2. **이진수의 과학적 정규화**:
     * 정규화 공식: 소수점을 첫 번째 '1' 바로 뒤로 이동시킴.
     * $1101.101_{(2)} = 1.101101_{(2)} \times 2^3 \implies$ 실제 지수 $e = 3$ 확보.
  3. **지수부(Exponent) 계산**:
     * 바이어스 $127$ 적용: $E = 3 + 127 = 130$
     * 130을 8비트 이진수로 변환: $130 = 128 + 2 = 10000010_{(2)}$
  4. **가수부(Fraction) 채우기**:
     * 정규화된 소수점 하위 비트: `101101`
     * 23비트 길이를 맞추기 위해 오른쪽에 0을 가득 채움: `10110100000000000000000`
  5. **부호 비트 결정**: 양수이므로 `0`
  6. **최종 32비트 비트 배열 조립 결과**:
     * `0 | 10000010 | 10110100000000000000000`

### (3) 문자 자료 표현 방식
* **BCD 코드 (Binary Coded Decimal)**: 6비트를 사용하여 $2^6 = 64$개 문자 표현 (존 2비트 + 디지트 4비트).
* **ASCII 코드 (American Standard Code for Information Interchange)**: 7비트를 사용하여 $2^7 = 128$개 문자 표현 (존 3비트 + 디지트 4비트). 데이터 통신의 전역 표준.
* **EBCDIC 코드**: 8비트를 사용하여 $2^8 = 256$개 문자 표현 (존 4비트 + 디지트 4비트). 대형 mainframe 컴퓨터 시스템 환경에서 사용.

## 1.3 알고리즘 및 복잡도 분석

### (1) 알고리즘의 정의 및 5대 필수 요건
* **정의**: 어떠한 문제를 해결하기 위해 명확하게 지시된 유한개의 명령 단계들의 모임.
* **5대 필수 요건**:
  1. **입력 (Input)**: 외부에서 제공되는 데이터가 0개 이상 반드시 존재해야 함.
  2. **출력 (Output)**: 최소한 1개 이상의 유의미한 가공 결과를 출력해야 함.
  3. **명확성 (Definiteness)**: 모든 단계의 명령어는 애매모호하지 않고 직관적이며 명확해야 함.
  4. **유한성 (Finiteness)**: 명령어 단계들은 반드시 유한한 횟수만큼 실행된 후 종료되어 정상 정지해야 함.
  5. **효과성 (Effectiveness)**: 제시된 모든 명령은 종이와 연필만으로도 실행 가능한 원초적 수준의 명령이어야 함.

### (2) 알고리즘의 시간 및 공간 성능 분석
* **시간 복잡도 (Time Complexity)**: 알고리즘의 수행 시간 효율을 물리적 초 단위로 측정하는 것이 아니라, 입력 크기 $n$이 증가함에 따라 알고리즘 내부에서 수행해야 하는 **기본 연산(비교, 이동, 산술 등) 횟수**의 비례 성장을 수학적 함수로 모형화하는 것.
* **공간 복잡도 (Space Complexity)**: 알고리즘이 가동되어 정지할 때까지 추가적으로 필요한 기억 공간(메모리)의 크기.
* **빅오 표기법 (Big-O Notation)**: 시간 복잡도의 수학적 상한선(Worst-case Upper bound)을 점근적으로 평가하는 기법.
  * 정식 수학적 정의: 
    $$f(n) = \mathcal{O}(g(n)) \iff \exists c > 0, n_0 \ge 1 \text{ s.t. } \forall n \ge n_0, f(n) \le c \cdot g(n)$$

#### 빅오 시간복잡도 속도 크기 서열 (빠른 속도 순)
$$\mathcal{O}(1) < \mathcal{O}(\log n) < \mathcal{O}(n) < \mathcal{O}(n \log n) < \mathcal{O}(n^2) < \mathcal{O}(2^n) < \mathcal{O}(n!)$$
