# 6. 파일 편성 (File Structures and Access Methods)

## 6.1 파일편성 개요

* **정의**: 많은 양의 자료가 보관된 파일을 하드디스크나 SSD 등 각종 보조기억장치에 물리적으로 저장할 때, 기억 공간의 효율성을 높이고 필요한 레코드를 최단 시간 내에 찾아내기 위해 레코드들을 배치 구성하는 기법.
* **OS와의 조화**: 모든 파일은 고유한 파일 편성 방식에 따라 규칙적으로 정렬 조직화되어 하드디스크에 물리 저장되며, 운영체제(OS)의 파일시스템(FAT32, NTFS 등)이 이를 기반으로 입출력 버퍼를 통해 데이터를 관리합니다.
* **주요 편성 방식**:
  * 순차 파일 (Sequential File)
  * 색인 순차 파일 (Indexed Sequential File)
  * 직접 파일 (Direct File)

---

## 6.2 순차 파일 (Sequential File)

* **개요**: 입력되는 레코드들의 논리적인 물리 순서대로 보조기억장치의 연속적인 공간에 순차적으로 차례차례 정렬 기록하는 구조.
* **접근 매체**: 주로 앞부분부터 차례대로 감아 읽어야 하는 순차 접근(Sequential Access)만 지원되는 **자기테이프 (Magnetic Tape)** 장치에서 주로 활용됩니다.

```
       [순차 파일의 물리 구조 (자기테이프)]
       
       ┌──────────┬──────────┬──────────┬──────────┐
       │ Record 1 │ Record 2 │ Record 3 │ Record 4 │ ───► 순차적 순서 접근만 가능
       └──────────┴──────────┴──────────┴──────────┘
```

* **장점**:
  * 빈 여백 공간을 두지 않고 연속 기록하므로 기록밀도가 극도로 높아 **기억 공간 활용도가 극대화**됩니다.
  * 레코드가 키 순서대로 고정 배치되어 파일 일괄 처리(Batch processing) 작업 시 타의 추종을 불허하는 가장 빠른 속도를 보장합니다.
  * 물리 구조가 극도로 단순하여 매체 변환이 용이하고 거의 모든 저장 매체에 범용적으로 적용할 수 있습니다.
* **단점**:
  * 중간에 새로운 레코드를 삽입하려면 빈 공간을 만들기 위해 파일 전체를 복사하여 신규 파일을 재구성해야 하므로, 삽입/삭제 오버헤드 시간이 극도로 큽니다.
  * 특정 레코드를 임의로 찾기 위해 무조건 파일의 처음부터 순차 검색해야 하므로, 특정 데이터 조회 검색 효율이 매우 불리합니다.

## 6.3 색인 순차 파일 (Indexed Sequential File / ISAM)

* **개요**: 일괄 처리를 위한 **순차 처리**와 실시간 조회를 위한 **랜덤(임의) 처리**를 모두 효율적으로 달성하기 위해 설계된 정교한 파일 구조.
* **원리**: 
  데이터 레코드들을 키(Key) 항목을 기준으로 물리적으로 미리 정렬하여 정렬 파일로 기록하고, 이들의 탐색을 도울 키 항목들의 주소만 따로 뽑아낸 별도의 **색인(Index) 구역**을 연계 구축합니다. 
  기억장치의 포인터 주소를 사용하므로 **자기디스크(Disk)** 장치에 널리 사용되며, 자기테이프 매체에서는 구조 설계 상 사용할 수 없습니다. 대표적으로 **ISAM(Index Sequential Access Method)**이라고 부릅니다.

### 색인 순차 파일의 3대 핵심 구역 구성 체계
색인 순차 파일은 기하학적으로 **기본 구역, 색인 구역, 오버플로우 구역**의 3가지 물리 공간으로 구획을 나누어 관리합니다.

```
                  색인 순차 파일 (Indexed Sequential File)
                                      │
        ┌─────────────────────────────┼─────────────────────────────┐
        ▼                             ▼                             ▼
 기본 구역 (Prime Area)          색인 구역 (Index Area)      오버플로우 구역 (Overflow)
 실제 레코드 저장               마스터 ➔ 실린더 ➔ 트랙       새 삽입 레코드 대피 공간
 (Key 순 정렬 상태)             색인 계층 트리 구조          (실린더 / 독립 구역 분리)
```

#### ① 기본 구역 (Prime Area)
* 실제 데이터 레코드들이 저장되는 주기억 공간.
* 각 레코드는 설계 단계에서 지정된 키값 순서에 맞춰 물리적으로 엄밀히 정렬된 순서대로 저장되어 순차 처리를 도모합니다.

#### ② 색인 구역 (Index Area)
* 기본 구역에 실린더와 트랙 단위로 저장되어 있는 데이터들의 특정 주소를 가리키는 포인터 색인을 계층(Tree)화하여 관리하는 구역.
  * **트랙 색인 구역 (Track Index Area)**: 
    각 실린더 내의 개별 트랙 상에 기록된 데이터 레코드들 중 **최대 키값**과 해당 트랙의 물리 주소를 매핑하여 기록. 처리할 타겟 레코드가 구체적으로 어느 트랙에 있는지 최종 확인하게 해줍니다.
  * **실린더 색인 구역 (Cylinder Index Area)**:
    각 실린더별 최대 키값과 해당 실린더의 첫 트랙 색인 주소를 기록. 타겟 레코드가 어느 실린더 구역에 도달해야 하는지 확인하게 해줍니다.
  * **마스터 색인 구역 (Master Index Area)**:
    실린더 색인의 양이 방대할 때, 실린더 색인들을 일정 블록으로 나누고 이들의 최대 키값과 주소를 다시 상위 인덱스로 보관하는 최상위 계층 색인 구역.

#### ③ 오버플로우 구역 (Overflow Area)
* 기본 구역에 빈 여백 슬롯이 없어 새로운 레코드를 물리 삽입할 수 없을 때, 이를 임시 대피시켜 저장하기 위해 특별 배정된 여분 공간.
  * **실린더 오버플로우 구역 (Cylinder Overflow Area)**:
    각 실린더 내에 기본 구역의 연장으로 배치해 둔 예비 오버플로우 구역.
  * **독립 오버플로우 구역 (Independent Overflow Area)**:
    실린더 내의 오버플로우 구역까지 모두 소진되었을 때, 전체 파일 수준에서 별도의 공용 디스크 영역에 할당해 둔 백업 예비 구역.

* **장점**:
  * 순차 접근과 직접 접근(임의 탐색)을 모두 완벽하게 보장하여 범용성이 높습니다.
  * 새로운 레코드 삽입 시 전체 파일을 통째로 복사할 필요가 없어 데이터 갱신이 용이합니다.
* **단점**:
  * 색인 구역과 임시 대피 오버플로우 구역을 상시 유지해야 하므로 추가적인 디스크 기억 공간 낭비가 큽니다.
  * 오버플로우 영역으로 대피한 임시 레코드들의 체인이 지나치게 길어지면, 탐색 연산 시 디스크 탐색 오버헤드가 급증하여 전체 시스템의 파일 액세스 성능이 나빠집니다. 이 경우 주기적으로 파일을 완전 재편성(Reorganization)하는 유지 보수가 강제됩니다.

## 6.4 직접 파일 (Direct File / DAM)

* **개요**: 레코드의 물리적인 배치 순서 조건 없이, 임의의 물리적 디스크 공간에 레코드를 독립적으로 분산하여 기록하는 방식.
* **원리**: 
  각 레코드의 고유 키를 해시 함수(Hash Function)에 투입하여 보조기억장치의 물리적 상대 레코드 주소(Relative Record Address)를 계산하여 도출하고, 그 주소에 레코드를 즉각 직접 기록하고 읽어 들입니다. **DAM (Direct Access Method)** 혹은 **랜덤 파일 (Random File)**로 지칭합니다.
* **저장 매체**: 직접 접근 기억장치인 **DASD (Direct Access Storage Device, 예: 하드디스크, SSD)**가 반드시 활용되어야 합니다.

* **장점**:
  * 물리 주소를 직접 연산하므로 중간 매핑 단계를 건너뛰어 **가장 빠른 레코드 접근 시간**을 제공합니다.
  * 레코드의 물리 순서 제약이 없어 임의의 삽입, 삭제, 수정 갱신 연산이 극도로 자유롭고 단순합니다.
* **단점**:
  * 해시 연산을 거치므로 주소 변환 과정 오버헤드가 동반되며, 해시 성능에 따라 공간 활용 효율성이 크게 좌우됩니다.
  * 서로 다른 키가 동일 주소로 매핑되는 충돌(Collision)의 우려가 상존하므로, 임시 슬롯 및 대피 주소 공간 확보를 위한 물리 관리가 요구됩니다.
